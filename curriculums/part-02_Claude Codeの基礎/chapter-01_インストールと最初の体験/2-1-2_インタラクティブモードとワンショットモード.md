# 2-1-2 インタラクティブモードとワンショットモード

## 概要

前のセクションで Claude Code のインストールと認証が完了しました。このセクションでは、Claude Code の2つの起動モードを理解し、実際にファイルを生成して diff を承認する体験をします。解説しながら手を動かしていきますので、ターミナルを開いて一緒に進めましょう。

Claude Code には**インタラクティブモード**と**ワンショットモード** の2つがあります。この2つを使い分けられるようになること、そして Claude Code が提案する変更（diff）を自分で読んで承認する流れを身につけることが、このセクションのゴールです。

## 2つのモード

Claude Code には、用途に応じた2つの起動モードがあります。

### インタラクティブモード

ターミナルで `claude` と入力すると、インタラクティブモードが起動します。

```bash
claude
```

このモードでは、Claude Code と対話的にやり取りします。質問を入力すると Claude が回答し、続けて別の質問や指示を出せます。会話の文脈が保持されるため、「さっきのコードを修正して」のように前の内容を踏まえた指示ができます。

開発中に継続的に作業する場面（機能実装、バグ修正、コードの調査など）では、このモードを使います。Part 2 のハンズオンでも、主にこのモードで作業を進めます。

セッションを終了するには、`exit` と入力するか、`Ctrl+D` を押します。`Ctrl+C` を押すと、現在の生成を中断できます（セッション自体は終了しません）。

### ワンショットモード

1回だけの質問やタスクには、ワンショットモード（`-p` フラグ）が便利です。

```bash
claude -p "このプロジェクトの技術スタックを教えて"
```

このモードでは、指定した質問に対して Claude が回答を返し、そのまま終了します。対話的なやり取りは行いません。

ワンショットモードはパイプ（`|`）と組み合わせて使うこともできます。ファイルの内容やコマンドの出力を Claude に渡して処理させるときに便利です。

```bash
cat README.md | claude -p "このファイルの内容を要約して"
```

この例では、`README.md` の内容を Claude に渡し、要約を依頼しています。

**2つのモードの使い分け**:

| モード | 起動コマンド | 用途 |
|---|---|---|
| インタラクティブモード | `claude` | 継続的な開発作業（機能実装、バグ修正、コード調査） |
| ワンショットモード | `claude -p "質問"` | 1回限りの質問、パイプでのデータ処理 |

💡 `claude "質問"` のように `-p` なしで質問を渡すと、その質問をきっかけにインタラクティブモードが始まります。`-p` 付きは回答後に終了、`-p` なしは回答後も対話を続けられるという違いです。

実務では、開発作業の大半はインタラクティブモードで行います。ワンショットモードは「ちょっと聞きたいだけ」のときや、CI/CD パイプラインなどの自動化に組み込むときに使います。

## 入力テクニック

インタラクティブモードでは、いくつかの入力テクニックを知っておくと効率的に作業できます。

### 複数行入力・ファイル添付・Bash モード

**複数行入力**: 長い指示を書くときは、改行して複数行で入力できます。改行するには `\` に続けて Enter を押します。iTerm2、WezTerm、Ghostty、Kitty では Shift+Enter でも改行できます。VS Code のターミナルでは `/terminal-setup` を実行すると Shift+Enter が使えるようになります。

```
> 以下の要件でユーザーモデルを作成してください。
> 1. name: string
> 2. email: string, unique
> 3. role: enum(admin, author)
```

**ファイル添付（`@`）**: `@` に続けてファイル名を入力すると、そのファイルの内容を Claude に共有できます。Claude Code がファイルの中身を読み取り、質問や指示に活かします。

```
> @app/Models/User.php このモデルのリレーションを説明して
```

💡 Claude Code はプロジェクト内のファイルを必要に応じて自動的に読み取ります。`@` を使わなくても「User モデルを見て」と指示すれば Claude Code が自分でファイルを探します。`@` は「このファイルを確実に見てほしい」ときに使う明示的な指定です。

**Bash モード（`!`）**: 入力の先頭に `!` を付けると、Claude を介さずにシェルコマンドを直接実行できます。実行結果は会話の文脈に追加されるため、Claude がその結果を踏まえて回答できるようになります。

```
> ! git status
```

これは Claude Code のセッションを抜けずに、素早くコマンドを実行したいときに便利です。

### キーボードショートカット

よく使うショートカットを紹介します。すべてを今すぐ覚える必要はありません。使いながら少しずつ身につけていきましょう。

| ショートカット | 動作 |
|---|---|
| `Ctrl+C` | 現在の生成を中断 |
| `Ctrl+D` | セッションを終了 |
| `↑` / `↓` | 入力履歴を表示 |
| `Shift+Tab` | 権限モードの切り替え |
| `?` | 利用可能なショートカット一覧を表示 |

💡 **Prompt Suggestions**: Claude が回答した後、次に入力する内容の候補がグレーの文字で表示されることがあります。Tab を押すと候補を受け入れられます。会話の流れに沿った次のステップを提案してくれるので、何を聞けばいいか迷ったときに参考になります。

💡 **Background Bash**（`Ctrl+B`）: Claude Code がコマンドを実行しているとき、`Ctrl+B` を押すとそのコマンドをバックグラウンドに移動できます。テストの実行やビルドなど、時間のかかる処理を待たずに次の指示を出せます。今は「そういう機能がある」と知っておくだけで十分です。

## ファイル生成と diff 承認

ここからは実際に Claude Code にファイルを作ってもらい、その変更内容（diff）を確認して承認する流れを体験します。

### diff の読み方と承認

Claude Code にコードの生成や編集を依頼すると、いきなりファイルを書き換えるのではなく、**変更内容の差分（diff）を表示して皆さんの承認を求めます**。これは Claude Code の重要な特徴です。

diff が表示されたとき、以下の3つの選択肢があります。

| キー | 動作 |
|---|---|
| `y`（Yes） | 変更を承認し、ファイルに反映する |
| `n`（No） | 変更を却下する |
| `e`（Edit） | 変更内容を編集してから反映する |

diff には、追加される行（緑色で `+` 表示）と削除される行（赤色で `-` 表示）が表示されます。新規ファイルの場合はすべて追加行として表示されます。

この「diff を読んで承認する」流れは、Claude Code を使う上で最も基本的な操作です。どんなに優秀な AI でも、生成されたコードを確認せずに受け入れることは避けるべきです。Part 1 で学んだ「AI 生成コードの責任は自分にある」という原則を、ここから実践していきます。

💡 **権限モードの予告**: Claude Code には、ファイル編集やコマンド実行の際に毎回許可を求めるモード（デフォルト）の他に、一定の操作を自動で承認するモードもあります。権限モードの詳細は 2-9 で体系的に学びます。今はデフォルトのまま、1つずつ確認しながら進めましょう。

### エージェントループ

Claude Code は単に質問に答えるだけではありません。指示を受け取ると、以下のようなステップを自律的に繰り返して作業を進めます。

1. 指示を解釈する
2. 必要なファイルを読み取る
3. コードを生成・編集する（diff を表示して承認を求める）
4. コマンドを実行する（実行許可を求める）
5. 結果を確認し、必要があれば追加の作業を行う

この「考える → 行動する → 確認する」のサイクルを**エージェントループ** と呼びます。Claude Code が作業している間、ターミナルにはどのファイルを読んでいるか、何を実行しようとしているかが表示されます。この動きを見ながら「今 Claude はこう動いているな」と把握する習慣をつけましょう。

### やってみよう: Markdown ライブプレビューエディタを作る

では、実際に Claude Code にアプリを作ってもらいましょう。適当なディレクトリに移動してから、Claude Code を起動してください。

```bash
mkdir -p ~/claude-test && cd ~/claude-test
claude
```

Claude Code が起動したら、以下のように入力してください。たった一言です。

```
> PHPの組み込みサーバーで動くMarkdownライブプレビューエディタを作ってください。左にエディタ、右にリアルタイムプレビューの2ペイン構成で、ダークモード切り替えボタン付きにしてください
```

Claude Code が動き始めます。ターミナルに表示される動作をよく見てください。Claude Code がファイル構成を考え、HTML・CSS・JavaScript のコードを生成し、diff を表示します。これがエージェントループです。

diff が表示されたら、内容を確認しましょう。皆さんの環境では異なるコードが生成されますが、たとえば以下のようなファイルが提案されます。

```html
<!-- index.html -->
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>Markdown Live Preview</title>
    <style>
        /* 2ペインレイアウト、ダークモードの CSS ... */
    </style>
</head>
<body>
    <div class="editor-pane">
        <textarea id="editor"></textarea>
    </div>
    <div class="preview-pane">
        <div id="preview"></div>
    </div>
    <script>
        // Markdown パース、リアルタイムプレビューの JavaScript ...
    </script>
</body>
</html>
```

一言の指示から、HTML の構造、CSS のレイアウトとダークモード、JavaScript のリアルタイム処理まで一気に生成されます。コードの中身を見てみると、`textarea` の `input` イベントでプレビューを更新する仕組みや、CSS 変数でダークモードを切り替える仕組みなど、皆さんが学んできた Web の知識で読み解ける技術の組み合わせで作られているはずです。

確認できたら `y` を押して承認します。続けて、サーバーを起動しましょう。

```
> PHPの組み込みサーバーで起動して
```

Claude Code が `php -S localhost:8000` を実行しようとしたら、`y` で承認してください。ブラウザで `http://localhost:8000` を開くと、Markdown エディタが動いています。左側に Markdown を入力すると、右側にリアルタイムでプレビューが表示されます。ダークモードボタンも試してみてください。

皆さんが普段使う Zenn や Qiita のエディタの基本的な仕組みが、一言で再現されました。

サーバーを停止するには、ターミナルで `Ctrl+C` を押してください。

### 機能を追加してみる

アプリが動いたところで、もう一言。今度は既存のコードを修正する体験をします。

```
> 文字数カウンターと、入力した Markdown をHTMLファイルとしてダウンロードできるボタンを追加してください
```

今回の diff は先ほどと少し違います。新規作成のときはすべて緑色の追加行（`+`）でしたが、既存ファイルの変更では、削除される行（赤色、`-`）と追加される行（緑色、`+`）が混在します。

- 赤い行: 元のコードから消える部分
- 緑の行: 新しく追加・変更される部分

この diff を読んで、意図した変更だけが含まれているかを確認するのが大切です。確認できたら `y` で承認し、ブラウザをリロードして新しい機能が反映されていることを確かめましょう。

ここまでで、以下の流れを体験しました。

1. Claude Code に一言で指示を出す
2. HTML・CSS・JavaScript にわたるコードが生成される
3. diff を読んで確認し、承認する
4. ブラウザで動作を確認する
5. 追加の指示で既存コードを修正する
6. 追加・削除が混在する diff を確認し、承認する

一言の指示から動くアプリが生まれ、もう一言で機能が追加される。これが Claude Code との開発の基本サイクルです。

### ワンショットモードでも試す

最後に、ワンショットモードも試しておきましょう。`exit` でインタラクティブモードを終了してから、以下を実行します。

```bash
cat index.html | claude -p "このコードの構造を日本語で簡潔に説明して"
```

`cat` でファイルの内容をパイプで渡し、Claude にコードの解説を依頼しています。ワンショットモードでは結果が表示され、そのまま終了します。インタラクティブモードとの違いは、続けて追加の指示ができないことです。

このように、ワンショットモードはパイプと組み合わせて「ファイルの内容を渡して処理させる」使い方が特に便利です。

💡 **Voice Input**: Claude Code にはプッシュトゥトーク機能があり、スペースキーを長押しすると音声で指示を入力できます。キーボード入力が面倒なときや、考えながら指示を出したいときに試してみてください。この機能を使うには `/config` から Voice Dictation を有効にする必要があります。

## まとめ

- **インタラクティブモード**（`claude`）は会話の文脈を保持しながら継続的に作業するモードで、開発の主力です
- **ワンショットモード**（`claude -p`）は1回限りの質問やパイプ処理に使います
- 複数行入力（`\` + Enter）、ファイル添付（`@`）、Bash モード（`!`）で効率的に入力できます
- Claude Code はファイル変更時に diff を表示し、`y`（承認）/ `n`（却下）/ `e`（編集）で皆さんが判断します
- 「指示 → diff 確認 → 承認」の流れが、Claude Code での作業の基本サイクルです
