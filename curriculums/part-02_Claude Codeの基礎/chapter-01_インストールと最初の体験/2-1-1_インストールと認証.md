# 2-1-1 インストールと認証

## 概要

Part 1 では、Claude Code がなぜ皆さんの実務に役立つのか、そしてどんな準備が必要かを学びました。ここからは Part 2 です。いよいよ Claude Code をインストールし、実際に動かしていきます。

このセクションでは、Claude Code のインストールと認証を完了し、開発環境を整えます。ターミナルで `claude` と入力して Claude Code が起動する状態を目指しましょう。

## インストール

Claude Code はターミナルで動作する CLI ツールです。公式が推奨するネイティブインストールを使えば、Node.js などの追加依存なしにインストールできます。

### macOS / WSL（Windows Subsystem for Linux）（推奨）

ターミナルを開き、以下のコマンドを実行してください。

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

macOS の方はそのままターミナル（Terminal.app や iTerm2）で、Windows の方は WSL のターミナルで実行します。

インストールが完了すると、`claude` コマンドが使えるようになります。ネイティブインストールはバックグラウンドで自動更新されるため、常に最新バージョンが維持されます。

### 代替インストール方法

macOS で Homebrew を使っている方は、以下でもインストールできます。

```bash
brew install --cask claude-code
```

Windows で WinGet を使う場合は、PowerShell で以下を実行します。

```bash
winget install Anthropic.ClaudeCode
```

⚠️ Homebrew や WinGet でインストールした場合、自動更新されません。定期的に `brew upgrade claude-code` や `winget upgrade Anthropic.ClaudeCode` を実行して手動で更新してください。ネイティブインストールの場合は `claude update` で手動更新もできます。

💡 npm によるインストール（`npm install -g @anthropic-ai/claude-code`）は非推奨（deprecated）です。1-1-2 の前提条件で「Claude Code のためにホストマシンへ Node.js をインストールする必要はない」と説明したのは、このネイティブインストールが推奨されているためです。

### インストールの確認

インストールが成功したか確認しましょう。

```bash
claude --version
```

バージョン番号が表示されれば成功です。

```
1.0.58
```

表示されるバージョン番号は、皆さんがインストールした時期によって異なります。

## 認証

Claude Code を使うには、Claude のアカウントで認証する必要があります。1-2-2 でアカウント登録とプラン契約を済ませているので、ここではその認証を Claude Code に通します。

### アカウントと認証フロー

ターミナルで `claude` と入力して Claude Code を起動してください。

```bash
claude
```

初回起動時は、自動的にブラウザが開いて認証画面が表示されます。ブラウザが自動で開かない場合は、`c` キーを押すとログイン URL がクリップボードにコピーされるので、ブラウザに貼り付けてください。

ブラウザ上で Claude のアカウント（1-2-2 で登録したもの）にログインすると、認証が完了します。ターミナルに戻ると、Claude Code のウェルカム画面が表示されます。

せっかく起動したので、何か話しかけてみましょう。以下のように入力してみてください。

```
> あなたは何ができますか？
```

Claude Code が自分の機能を説明してくれます。これが皆さんと Claude Code の最初の会話です。回答の内容は毎回異なりますが、ファイルの読み書き、コマンドの実行、コードの生成や修正など、さまざまなことができると教えてくれるはずです。

Claude Code の回答を読んでみてください。この教材を進めるうちに、ここで挙げられた機能の一つひとつを実際に使っていくことになります。「こんなにできるのか」と思った方は、次のセクションでその片鱗を体験しましょう。

確認できたら、`exit` と入力してセッションを終了します。認証情報はマシンに保存されるため、次回以降は `claude` を実行するだけですぐに使えます。

💡 認証に関連するコマンドを覚えておきましょう。

- **セッション内**で `/logout` と入力するとログアウトできます
- **ターミナルから** `claude auth login` でアカウントを切り替えられます
- **ターミナルから** `claude auth status` で現在の認証状態を確認できます

### プランの確認

認証が完了したら、自分のプランが正しく認識されているか確認しましょう。ターミナルで以下を実行すると、現在の認証状態やアカウント情報を確認できます。

```bash
claude auth status --text
```

1-2-2 で選択したプラン（Pro または Max）のアカウントで認証されていれば、準備完了です。

⚠️ Claude の無料プランでは Claude Code を利用できません。1-2-2 で Pro プラン以上を契約済みであることを確認してください。

⚠️ よくあるエラー: 認証後に Claude Code が反応しない

```
Authentication failed
```

**原因**: ブラウザでのログインが完了していない、またはネットワーク接続に問題がある場合に発生します。

**対処法**: ブラウザで Claude（claude.ai）にログインできることを確認してから、再度 `claude` を実行してください。

## 環境確認

インストールと認証が終わったら、開発環境が正しくセットアップされているか確認しましょう。

### /doctor による環境診断

Claude Code には環境診断コマンドが用意されています。ターミナルで以下を実行してください。

```bash
claude doctor
```

このコマンドは、Claude Code の動作に必要な環境（ネットワーク接続、認証状態、依存ツールなど）を自動的にチェックし、問題があれば報告してくれます。すべて正常であれば、そのまま次に進みましょう。問題が報告された場合は、表示されるメッセージに従って対処してください。

### VS Code ターミナルでの利用

皆さんは普段 VS Code を使って開発しています。Claude Code も VS Code のターミナル（統合ターミナル）から利用できます。

VS Code のターミナルを開き（`` Ctrl+` `` または `` Cmd+` ``）、`claude` と入力すれば起動します。プロジェクトのディレクトリで起動すると、Claude Code はそのプロジェクトのファイルを認識した状態で動作します。

💡 ターミナルの推奨設定として、Shift+Enter で改行入力ができるようにしておくと便利です。Claude Code のセッション内で `/terminal-setup` と入力すると、VS Code のターミナル向けに Shift+Enter のキーバインドを自動設定してくれます。この設定は必須ではありませんが、後のセクションで複数行の入力をする際に役立ちます。

### VS Code 拡張機能

Claude Code には VS Code 拡張機能も用意されています。VS Code の拡張機能マーケットプレイスで「Claude Code」を検索し、インストールできます。

この拡張機能では、以下のような機能が使えます。

- **インライン diff**: Claude が提案するコード変更を、VS Code のネイティブな差分表示で確認できます
- **会話パネル**: ターミナルではなく、VS Code のサイドバーやタブ上で Claude と対話できます
- **`@` メンション**: ファイル名を `@` で参照して、特定のファイルについて質問できます

拡張機能はターミナル CLI と併用できます。拡張機能のパネルで対話しながら、必要に応じてターミナルの CLI に切り替えることも可能です。

この教材では主にターミナル CLI を使って進めますが、拡張機能の方が使いやすいと感じた場合はそちらを使っても構いません。どちらも同じ Claude Code です。

💡 Claude Code は VS Code 以外の環境でも利用できます。Desktop アプリ（macOS / Windows）、Web（claude.ai/code）、JetBrains IDE 向けプラグイン、さらに claude.ai から遠隔操作する Remote Control など、さまざまなインターフェースが提供されています。この教材ではターミナル CLI を基本としますが、他の選択肢があることを知っておくと、今後の開発環境に合わせて使い分けられます。

## まとめ

- Claude Code は `curl` コマンドによるネイティブインストールが推奨されており、Node.js は不要です
- 初回起動時にブラウザで認証を行い、以降は `claude` コマンドですぐに使えます
- `claude doctor` で環境診断ができます
- VS Code のターミナルから使うのが、普段の開発フローに最も自然に馴染みます
- VS Code 拡張機能を併用すると、インライン diff や会話パネルも利用できます

次のセクションでは、Claude Code の2つの起動モード（インタラクティブモードとワンショットモード）を学び、実際にファイルを生成する初めての体験をします。
