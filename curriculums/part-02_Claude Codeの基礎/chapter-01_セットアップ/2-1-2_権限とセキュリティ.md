## なぜ権限を理解する必要があるのか

前のセクションで、Claude Code にファイルを作成してもらったとき、権限の確認ダイアログが表示されました。「Allow」を選んで先に進みましたが、あのとき何を許可したのか、正確に理解していたでしょうか。

Claude Code はエージェント型ツールです。あなたの指示に基づいて、ファイルの読み書き、コマンドの実行、Git の操作など、実際の操作を行います。これは強力な能力ですが、同時にリスクも伴います。意図しないファイルの変更、危険なコマンドの実行、機密情報の漏洩といった事故は、権限の仕組みを理解していないと起こりえます。

実務では、本番環境のコードや顧客データに影響を与えるような操作を Claude Code に任せる場面も出てきます。そのとき「よくわからないけど Allow を押す」という習慣がついていると、取り返しのつかないミスにつながります。このセクションで、何を許可していて何がブロックされているのかを正確に理解しましょう。

## 🧠 先輩エンジニアはこう考える

権限の確認ダイアログが出るたびに毎回「Allow」を押すのが面倒に感じる気持ちはわかる。自分も最初はそうだった。でも、あのダイアログは「Claude Code が何をしようとしているか」を知るための重要な情報源でもある。特に慣れないうちは、ダイアログの内容を読む習慣をつけた方がいい。何を実行しようとしているのかを確認することで、Claude Code の動きを理解する力もつく。慣れてきたら権限モードを調整して、信頼できる操作は自動で許可するようにすればいい。

## Claude Code の権限モデル

Claude Code が行う操作は、大きく3つのカテゴリに分かれます。それぞれ、許可の必要性が異なります。

### 読み取り操作（許可不要）

ファイルの内容を読む、ディレクトリの構造を確認するといった操作は、許可なく実行されます。読み取りだけでは環境に変更が加わらないため、リスクが低いからです。

- ファイルの読み取り（Read）
- ファイルの検索（Glob、Grep）
- ディレクトリの一覧取得

### Bash コマンドの実行（許可が必要）

`ls` や `cat` のような安全なコマンドであっても、Bash コマンドの実行には許可が必要です。一度許可すると、**そのプロジェクトで永続的に** 許可されます（次回のセッションでも有効です）。

- テストの実行（`php artisan test` など）
- パッケージのインストール（`composer require` など）
- Git コマンド（`git status`、`git commit` など）

> 📝 許可したコマンドはプロジェクト単位で記憶されます。プロジェクト A で `npm test` を許可しても、プロジェクト B では改めて許可が必要です。

### ファイルの変更（許可が必要）

ファイルの作成、編集、削除には許可が必要です。Bash コマンドとは異なり、ファイル変更の許可は**セッション終了まで** 有効です。Claude Code を再起動すると、改めて許可が必要になります。

- ファイルの作成（Write）
- ファイルの編集（Edit）

> 💡 Bash コマンドの許可が永続的なのに対し、ファイル変更の許可がセッション限定なのは、ファイル変更の方がリスクが高いためです。セッションごとに改めて許可を求めることで、意図しない変更を防いでいます。

まとめると、以下のようになります。

```
操作の種類          許可      有効期間
─────────────────────────────────────
読み取り            不要      ―
Bash コマンド       必要      プロジェクトで永続
ファイル変更        必要      セッション終了まで
```

## 5つの権限モード

Claude Code には、許可の求め方を制御する5つの権限モードがあります。作業内容やリスクに応じて使い分けます。

### Default モード

デフォルトの動作です。上で説明した権限モデルの通り、Bash コマンドの実行とファイルの変更に対して個別に許可を求めます。Claude Code を使い始めたばかりの段階では、このモードで操作に慣れることを推奨します。

### Accept Edits モード

ファイルの変更を自動で許可するモードです。Bash コマンドの実行には引き続き許可が必要です。Claude Code がどんなコードを書くかはある程度信頼するが、コマンドの実行は確認したい、という場合に使います。

### Plan モード

Claude Code が計画の提示だけを行い、実際の操作（ファイル変更・コマンド実行）を一切行わないモードです。「まず計画を見てから判断したい」という場面で使います。Plan Mode の詳細は Chapter 2-3 で学びます。

### Don't Ask モード

ファイルの変更も Bash コマンドの実行も自動で許可するモードです。権限の確認ダイアログが表示されなくなるため、作業が高速になります。ただし、Claude Code の操作を逐一確認できなくなるため、**Claude Code の動作を十分に理解してから使ってください。**

> ⚠️ Don't Ask モードでは、Claude Code が判断したすべての操作が即座に実行されます。意図しないファイルの変更やコマンドの実行が起きても、確認の機会がありません。このモードを使う場合は、Git でこまめにコミットする習慣をつけ、いつでも変更を戻せる状態を保ちましょう。

### Bypass Permissions モード

権限チェックを完全に無効化するモードです。Don't Ask モードよりもさらに制限が少なく、通常ブロックされる操作も実行されます。**このモードは、学習中およびチーム開発では使用しないでください。** 個人の実験環境で、リスクを十分に理解したうえで使用する場合に限ります。

### モードの比較

| モード | ファイル変更 | Bash コマンド | おすすめの場面 |
|---|---|---|---|
| **Default** | 都度確認 | 都度確認 | 学習中、初めてのプロジェクト |
| **Accept Edits** | 自動許可 | 都度確認 | コード生成は信頼、コマンドは確認したい |
| **Plan** | 実行しない | 実行しない | 計画立案、影響範囲の確認 |
| **Don't Ask** | 自動許可 | 自動許可 | 動作を理解したうえでの高速作業 |
| **Bypass Permissions** | 全許可 | 全許可 | 個人の実験環境のみ |

### モードの切り替え方

権限モードは、セッション中にいつでも切り替えられます。

| 方法 | 操作 |
|---|---|
| キーボードショートカット | Shift + Tab |
| 起動時に指定 | `claude --mode plan` のように `--mode` オプションを付ける |
| 設定ファイルで指定 | `.claude/settings.json` の `defaultMode` に記載 |

> 💡 Shift + Tab を押すと、Default → Accept Edits → Plan の3つのモードが順に切り替わります。Don't Ask と Bypass Permissions は、起動時のオプション（`--mode dontAsk`）または設定ファイルで指定します。現在のモードはプロンプトの表示で確認できます。

## 権限ルールによるカスタマイズ

Default モードで毎回許可を求められるのが煩わしくなってきたら、特定の操作だけ自動で許可する「権限ルール」を設定できます。

権限ルールは `.claude/settings.json` に記述します。

```json
// .claude/settings.json
{
  "permissions": {
    "allow": [
      "Bash(git status)",
      "Bash(git diff)",
      "Bash(php artisan test)"
    ]
  }
}
```

この例では、`git status`、`git diff`、`php artisan test` の3つのコマンドを自動で許可しています。それ以外のコマンドは引き続き許可を求められます。

権限ルールには、`allow`（自動許可）、`deny`（常にブロック）、`ask`（毎回確認）の3種類があります。

```json
// .claude/settings.json
{
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(php artisan test *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)"
    ]
  }
}
```

`*` はワイルドカードで、任意の引数にマッチします。`Bash(git *)` とすれば、`git` で始まるすべてのコマンドが自動で許可されます。

> 📝 権限ルールは、実行しようとしているコマンドの先頭と一致するかどうかで判定されます。`Bash(git)` と書くと `git status` にも `git push --force` にもマッチします。特定のコマンドだけを許可したい場合は、`Bash(git status)` のように具体的に書きましょう。

> 💡 今はまだ権限ルールを設定する必要はありません。Default モードで Claude Code の動きに慣れてから、よく使うコマンドを `allow` に追加していく流れが自然です。

## セキュリティの意識

権限モデルの仕組みを理解したうえで、Claude Code を安全に使うためのポイントを押さえておきましょう。

### Claude Code に渡される情報

Claude Code に指示を出すと、あなたのプロンプトに加えて、プロジェクトのファイル内容やディレクトリ構造なども AI モデルに送信されます。つまり、プロジェクトに含まれる情報は Claude Code を通じて Anthropic のサーバーに送信される可能性があります。

### 気をつけるべきこと

- **機密情報をプロジェクトに含めない**: API キー、パスワード、個人情報などの機密データは、`.env` ファイルなどに分離し、`.gitignore` に追加してください。Claude Code は `.gitignore` に記載されたファイルを読み取り対象から除外する傾向がありますが、明示的に除外する方が安全です
- **権限ダイアログの内容を確認する**: 特に `rm` や `git push` のような破壊的な操作は、実行前に内容を確認しましょう
- **新しいプロジェクトでは Default モードから始める**: 未知のコードベースでいきなり Don't Ask モードを使うのは避けてください
- **生成されたコードを確認する**: Claude Code が生成したコードが安全かどうかの判断は、最終的にあなたの責任です。この「見極め」については、Chapter 2-1 の最後のセクションから実践していきます

> 📝 Claude Code のプライバシーポリシーについて、詳しくは「1-2-1 AI ツールを使う前の確認事項」で学んだ内容を参照してください。データの取り扱いやテレメトリの設定については、そちらで確認しています。

## 🏃 実践: 権限モードを切り替えてみる

学習用プロジェクトで Claude Code を起動し、権限モードの違いを体験してみましょう。

### Step 1: 現在の権限モードを確認する

Claude Code を起動します。

```bash
claude
```

起動直後は Default モードです。画面のプロンプト表示を確認してください。

### Step 2: Accept Edits モードに切り替える

Shift + Tab を押して、Accept Edits モードに切り替えます。プロンプトの表示が変わったことを確認してください。

この状態で、Claude Code に何かファイルを編集する指示を出してみましょう。

```
> README.md の概要に「Part 2 で学習中」という一文を追加して
```

Default モードではファイル変更の許可ダイアログが表示されましたが、Accept Edits モードでは確認なしにファイルが変更されます。

### Step 3: Plan モードに切り替える

もう一度 Shift + Tab を押して、Plan モードに切り替えます。

この状態で指示を出すと、Claude Code は計画を提示するだけで、実際の操作は行いません。

```
> このプロジェクトに .gitignore ファイルを作成して
```

Claude Code は「.gitignore ファイルを作成して、こういう内容にします」という計画を提示しますが、実際にファイルは作成されません。計画の内容を確認してから実行するかどうかを判断できます。

### Step 4: Default モードに戻す

Shift + Tab を何度か押して、Default モードに戻してください。この教材を進める間は、基本的に Default モードを使用します。

> 💡 今後の学習では、必要に応じてモードを切り替えていきます。特に Plan モードは、Chapter 2-3 で改めて詳しく学びます。

## ✨ まとめ

- Claude Code の操作は「読み取り（許可不要）」「Bash コマンド（許可必要・永続）」「ファイル変更（許可必要・セッション限定）」の3カテゴリに分かれる
- 権限モードは5つ（Default / Accept Edits / Plan / Don't Ask / Bypass Permissions）。Shift + Tab で Default / Accept Edits / Plan の3モードを切り替えられる
- 学習中は Default モードを推奨。操作に慣れてから、必要に応じてモードを調整する
- 権限ルール（`.claude/settings.json`）で、特定のコマンドを自動許可または常にブロックできる
- 機密情報の管理と権限ダイアログの確認を習慣にすることが、安全な利用の基本
