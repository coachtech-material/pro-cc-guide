## いよいよ Claude Code を動かす

Part 1 では、Claude Code を学ぶ意義の理解からプランの契約まで、使い始めるための準備を整えました。ここからは実際に手を動かしていきます。

新しいツールを導入するとき、最初のハードルになるのが「インストールしたけど動かない」「認証で詰まった」といったセットアップのトラブルです。特にコマンドラインツールは、環境の違いによって同じ手順でもエラーが出ることがあります。このセクションでは、つまずきやすいポイントを先回りしながら、確実にセットアップを完了させます。

## 🧠 先輩エンジニアはこう考える

新しいツールを導入するとき、自分はまず「最短で動くところまで持っていく」ことを意識している。ドキュメントを全部読んでから始めるのではなく、まずインストールして動かしてみる。動く状態を作ってから細かい設定を詰めていく方が、ツールの全体像を早く掴める。Claude Code も同じで、まずは起動して何か聞いてみるところから始めよう。

## ターミナルを準備する

Claude Code はターミナル上で動作する CLI（コマンドラインインターフェース）ツールです。まず、この教材を通して使うターミナルアプリケーションを決めましょう。

### VS Code のターミナルではだめなのか

普段 VS Code の統合ターミナルを使っている方も多いと思います。Claude Code は VS Code のターミナルでも動作しますが、この教材では **独立したターミナルアプリケーション**（VS Code やエディタに組み込まれていない、単体で動くターミナル）の使用を推奨します。

理由は、Claude Code の動きを正確に理解するためです。Claude Code はファイルの読み書き、コマンドの実行、Git の操作をすべて自分で行います。VS Code を横に開いていると、「VS Code で手動編集しつつ Claude Code にも指示を出す」という混在が起きやすく、学習初期は「どちらが何を変更したか」が曖昧になりがちです。独立したターミナルで Claude Code だけに向き合う方が、エージェント型ツールの動きをクリアに把握できます。

慣れてきたら VS Code のターミナルや、IDE 連携機能（VS Code 拡張など）を使うのはまったく問題ありません。最初の学習段階では、Claude Code の振る舞いに集中できる環境を用意しましょう。

### おすすめのターミナルアプリケーション

**macOS の場合:**

| アプリケーション | 特徴 |
|---|---|
| **iTerm2** | macOS で最も広く使われているターミナル。機能が豊富で、日本語の情報も多い。迷ったらこれ |
| **Ghostty** | 高速で動作が軽い。シンプルな操作感を好む方に |
| Terminal.app | macOS に標準搭載。最低限の動作は可能だが、機能は限定的 |

**Windows の場合:**

| アプリケーション | 特徴 |
|---|---|
| **Windows Terminal** | Windows 11 に標準搭載。WSL との相性が良く、Windows ユーザーはこれを使用 |

> 💡 他にも WezTerm（クロスプラットフォーム対応、高いカスタマイズ性）や Kitty（GPU 描画による高速表示）といった選択肢もあります。ターミナルにこだわりがある方は試してみてください。

この教材のスクリーンショットや操作例は、特定のターミナルに依存しない形で記載しています。どのターミナルを使っても手順は同じです。

## 学習用プロジェクトを作成する

Claude Code はプロジェクトのディレクトリで起動して使います。この Chapter を通して使う学習用プロジェクトを作成しましょう。

ターミナルアプリケーションを開き、以下のコマンドを実行してください。

```bash
mkdir cc-practice
cd cc-practice
git init
```

`cc-practice` という名前は一例です。好きな名前で構いません。

> 📝 **なぜ Git リポジトリにするのか**: Claude Code は Git と密接に連携しています。変更の差分表示、チェックポイントによる自動バックアップ、コミットやプルリクエストの作成など、多くの機能が Git リポジトリであることを前提としています。Git で初期化されていないディレクトリでも起動はできますが、利用できる機能が制限されます。

## Claude Code をインストールする

### システム要件

Claude Code を動かすために必要な環境です。

| 項目 | 要件 |
|---|---|
| OS | macOS 13.0 以降 / Windows 10 1809 以降 / Ubuntu 20.04 以降 / Debian 10 以降 / Alpine Linux 3.19 以降 |
| メモリ | 4 GB 以上 |
| ネットワーク | インターネット接続が必要 |
| シェル | Bash、Zsh、PowerShell、CMD のいずれか |

> 📝 Windows の場合、Git for Windows のインストールが別途必要です。ネイティブの Windows（Git Bash）でも、WSL（Windows Subsystem for Linux）内でも利用できます。WSL を使う場合は、WSL 内で macOS / Linux と同じインストールコマンドを実行します。

### インストール

Claude Code のインストール方法はいくつかありますが、**ネイティブインストール** が推奨されています。ネイティブインストールはバックグラウンドで自動アップデートが行われるため、常に最新バージョンを利用できます。

**macOS / Linux / WSL の場合:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows（PowerShell）の場合:**

```bash
irm https://claude.ai/install.ps1 | iex
```

**macOS で Homebrew を使う場合（代替手段）:**

```bash
brew install --cask claude-code
```

**Windows で WinGet を使う場合（代替手段）:**

```bash
winget install Anthropic.ClaudeCode
```

> 💡 以前は npm（`npm install -g @anthropic-ai/claude-code`）でインストールする方法が主流でしたが、現在は非推奨です。npm 経由でインストール済みの場合は、ネイティブインストールに切り替えることを推奨します。

### インストールの確認

インストールが完了したら、バージョンを確認します。

```bash
claude --version
```

バージョン番号が表示されれば、インストールは成功です。

> ⚠️ **よくあるエラー**: `claude: command not found`
>
> ```
> zsh: command not found: claude
> ```
>
> **原因**: インストール先のディレクトリ（`~/.local/bin`）にパスが通っていない可能性があります。
>
> **対処法**: ターミナルを再起動してください。それでも解決しない場合は、`~/.zshrc`（Bash の場合は `~/.bashrc`）に以下を追加し、ターミナルを再起動します。
>
> ```bash
> export PATH="$HOME/.local/bin:$PATH"
> ```

何か問題が起きた場合は、`claude doctor` コマンドで環境の診断ができます。

```bash
claude doctor
```

このコマンドは、システム要件の充足状況や認証の状態などを自動でチェックし、問題があれば対処法を提示してくれます。

### 自動アップデート

ネイティブインストールで導入した場合、Claude Code はバックグラウンドで自動的にアップデートされます。手動でアップデートしたい場合は、以下のコマンドを実行します。

```bash
claude update
```

> 💡 リリースチャンネルには `latest`（デフォルト）と `stable`（約1週間遅延）の2つがあります。安定性を重視する場合は `stable` チャンネルに切り替えることもできますが、通常は `latest` で問題ありません。

## 認証する

Claude Code を使うには、Part 1 で契約した Claude のアカウントで認証を行う必要があります。

先ほど作成した学習用プロジェクトのディレクトリに移動し（すでにいる場合はそのまま）、以下のコマンドを実行します。

```bash
claude
```

初回起動時には、以下のような流れで認証が行われます。

```
┌──────────────────────────────────────┐
│ ターミナルで `claude` を実行          │
└──────────┬───────────────────────────┘
           ↓
┌──────────────────────────────────────┐
│ ブラウザが自動で開く                  │
│ （Claude.ai のログイン画面）          │
└──────────┬───────────────────────────┘
           ↓
┌──────────────────────────────────────┐
│ ログインして認証を許可               │
└──────────┬───────────────────────────┘
           ↓
┌──────────────────────────────────────┐
│ ターミナルに戻ると認証完了           │
│ Claude Code が使える状態に           │
└──────────────────────────────────────┘
```

<!-- TODO: 画像追加 - ブラウザの認証画面 -->

ブラウザが自動で開かない場合は、ターミナルに表示される URL を手動でブラウザに貼り付けてください。

> ⚠️ **よくあるエラー**: 認証後もターミナルに反応がない
>
> **原因**: ブラウザで認証を完了しても、ターミナルに自動で戻らない場合があります。
>
> **対処法**: ブラウザで認証を完了した後、ターミナルのウィンドウに戻ってください。認証が正常に完了していれば、Claude Code のインターフェースが表示されます。

> 📝 認証に使用するアカウントは、「1-2-2 プランの選択と費用」で契約した Claude.ai のアカウント（Pro または Max プラン）です。Anthropic Console（API）のアカウントとは別物なので注意してください。

## Claude Code のインターフェース

認証が完了すると、ターミナルに Claude Code のインターフェースが表示されます。

<!-- TODO: 画像追加 - Claude Code 起動直後の画面 -->

画面の下部に入力欄があり、ここにテキストを入力して Claude Code に指示を出します。チャット型 AI と同じように自然言語で話しかけることができますが、Claude Code はエージェント型ツールです。テキストの応答だけでなく、ファイルの読み書きやコマンドの実行など、実際の操作を行います。

いくつかの基本操作を覚えておきましょう。

| 操作 | 方法 |
|---|---|
| メッセージを送信 | テキストを入力して Enter |
| 複数行を入力 | `\` + Enter（全ターミナル共通）、または Shift + Enter（iTerm2、WezTerm、Ghostty、Kitty で対応） |
| 会話を終了 | Ctrl + C でキャンセル、Ctrl + D で終了 |
| Bash コマンドを実行 | `!` に続けてコマンドを入力（例: `! ls`） |
| スラッシュコマンド | `/` に続けてコマンド名（例: `/help`） |
| ファイルを参照 | `@` に続けてファイル名（オートコンプリート対応） |

> 💡 操作に迷ったら `/help` と入力してみてください。利用可能なコマンドの一覧が表示されます。

## 🏃 実践: 最初のやり取り

学習用プロジェクトで Claude Code を起動した状態で、実際にやり取りしてみましょう。

### Step 1: 簡単な質問をする

まずは気軽に話しかけてみましょう。入力欄に以下のように入力して Enter を押します。

```
> Claude Code でどんなことができるか、簡潔に教えて
```

Claude Code が自然言語で応答します。チャット型 AI と同じような体験ですが、これは Claude Code の機能のほんの一部です。

### Step 2: プロジェクトの状態を聞く

次に、プロジェクトに関する質問をしてみましょう。

```
> このプロジェクトのディレクトリ構成を確認して
```

Claude Code はファイルシステムを読み取り、現在のディレクトリの状態を報告します。作成したばかりのプロジェクトなので `.git` ディレクトリしかない状態が表示されるはずです。ここで注目してほしいのは、Claude Code がターミナルコマンド（`ls` など）を実行してファイルの情報を取得しているという点です。単にテキストを返しているのではなく、実際にあなたの環境を操作しています。

### Step 3: ファイルを作成してもらう

Claude Code のエージェントとしての能力を体験してみましょう。

```
> このプロジェクトの README.md を作成して。プロジェクト名は「Claude Code 学習用プロジェクト」で、概要として「Claude Code の基本機能を学ぶための練習用リポジトリ」と書いて
```

このとき、**権限の確認ダイアログ** が表示されるはずです。Claude Code がファイルを作成しようとしているため、あなたの許可を求めています。ここでは「Allow」を選択して進めてください。

<!-- TODO: 画像追加 - 権限確認ダイアログの画面 -->

許可すると、Claude Code が README.md を作成します。ファイルが作成されたことを確認しましょう。

```bash
! cat README.md
```

Claude Code 上で `!` に続けてコマンドを入力すると、Bash コマンドを直接実行できます。作成された README.md の内容が表示されるはずです。

> 📝 ここで表示された「権限の確認ダイアログ」は、Claude Code のセキュリティ機能の一部です。Claude Code は勝手にファイルを変更したりコマンドを実行したりはしません。必ずあなたの許可を得てから操作を行います。次のセクションで、この権限の仕組みを詳しく学びます。

### Step 4: 会話を終了する

ここまでの体験で、Claude Code の基本的な操作感が掴めたと思います。Ctrl + C を押すと現在の応答をキャンセルでき、Ctrl + D を押すと Claude Code を終了できます。

> 💡 Claude Code を終了しても、ここで作成したファイルや Git リポジトリはそのまま残ります。次のセクションでも同じプロジェクトを使い続けます。再度起動するには、プロジェクトのディレクトリで `claude` と入力するだけです。

## ✨ まとめ

- Claude Code はネイティブインストールが推奨。`curl` コマンド一発でインストールでき、自動アップデートで常に最新版が使える
- 認証は初回起動時にブラウザ経由で行う。Claude.ai のアカウント（Pro / Max プラン）を使用する
- Claude Code は自然言語で指示を出すだけでなく、ファイルの読み書きやコマンド実行など実際の操作を行うエージェント型ツール
- ファイルの変更やコマンドの実行には権限の確認が入る。Claude Code は勝手に操作を行わない
