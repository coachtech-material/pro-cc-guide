# 2-2-0 Chapter 2 の地図

> Claude Code を効果的に使うには、その仕組みを理解することが不可欠です。この Chapter では、Claude Code の動作原理、コンテキスト管理、そしてプロンプト設計という3つの「考え方」を学びます。Chapter 2-3 で各機能を実践するための土台になる内容です。

## セクション一覧

### 2-2-1 エージェントループ ｜ 📖🏃 読んで手を動かす

- 動作原理: 収集 → 実行 → 検証 の3フェーズの繰り返し
- モデルとツールの2要素、ツールの5カテゴリ
- ツール実行ログの読み方と Esc による中断介入

### 2-2-2 コンテキストとセッション管理 ｜ 📖🏃 読んで手を動かす

- コンテキストウィンドウの構成と自動コンパクションの仕組み
- 主要管理コマンド（`/clear`・`/compact`・`/btw`）の使い分け
- セッションの継続・分岐・チェックポイントによる Rewind

### 2-2-3 プロンプト設計 ｜ 📖🏃 読んで手を動かす

- 4つの設計原則（具体的コンテキスト・検証手段・探索と実装の分離・委譲）
- `@filename` でのファイル直接参照と `ultrathink` キーワード
- よくある失敗パターン（キッチンシンク・過修正ループ・無限探索・検証の空白）と対処

## 📖 この Chapter の進め方

3つのセクションすべてが 📖🏃 読んで手を動かす構成です。概念を読んで理解した後、各セクションの末尾にある 🏃 実践で cc-practice を使って手を動かして体感します。Chapter 2-3 の機能実践に入る前の土台になる内容です。上から順に進めてください。
