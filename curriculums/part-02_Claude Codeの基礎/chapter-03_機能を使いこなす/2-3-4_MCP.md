## 導入: Claude Code の「できること」には壁がある

ここまで学んできた通り、Claude Code はファイルの読み書き、コマンドの実行、Web 検索など、多くのことができます。しかし、「できないこと」もあります。

たとえば、ブラウザを操作して Web アプリの動作確認をする。データベースに直接クエリを投げてデータを確認する。Figma のデザインデータを参照する。これらは Claude Code の標準機能には含まれていません。

「じゃあ諦めるしかないのか」というと、そうではありません。**MCP（Model Context Protocol）** を使えば、Claude Code に新しい能力を追加できます。ブラウザを操作する能力、データベースにアクセスする能力、外部サービスと連携する能力を、プラグインのように後付けできるのです。

### 🧠 先輩エンジニアはこう考える

> MCP は「Claude Code に新しい手足を生やす」仕組みだと思えばいい。たとえば Playwright の MCP サーバーを追加すれば、Claude Code はブラウザを操作できるようになる。「このページのスクリーンショットを撮って」「ログインフォームに入力して動作確認して」といった指示に応えられるようになる。自分のプロジェクトで何が必要かを考えて、必要な MCP サーバーを追加していくのが実務での使い方だ。初めて MCP でブラウザが動くのを見たときは、ちょっと感動した。

## MCP とは

MCP は **Model Context Protocol** の略で、Claude Code と外部ツール・サービスを接続するための標準プロトコルです。

```
┌─────────────┐      MCP プロトコル      ┌─────────────────┐
│             │ ◀──────────────────────▶ │ MCP サーバー     │
│ Claude Code │                          │（Playwright）    │
│（クライアント）│                       │                  │
│             │ ◀──────────────────────▶ │ MCP サーバー     │
│             │                          │（GitHub）        │
└─────────────┘                          └─────────────────┘
```

Claude Code が **クライアント** として動作し、各種 **MCP サーバー** に接続します。MCP サーバーは、特定の能力（ブラウザ操作、データベースアクセス、外部 API 連携など）を提供するプログラムです。

MCP サーバーが提供するものは主に2つです。

| 提供するもの | 説明 | 例 |
|---|---|---|
| **ツール** | Claude Code が実行できる操作 | スクリーンショットの撮影、ファイルのアップロード |
| **リソース** | Claude Code が参照できる情報 | データベースのスキーマ、API のドキュメント |

MCP サーバーを追加すると、Claude Code のツール一覧に新しいツールが追加されます。Claude Code はこれらのツールを、標準ツール（ファイル読み取り、コマンド実行など）と同じように使えるようになります。

> 📝 MCP は Claude Code 固有の仕組みではありません。Anthropic が策定したオープンな標準プロトコルで、Claude Desktop や他の AI ツールでも利用できます。

## MCP サーバーの設定方法

MCP サーバーの追加方法は2つあります。

### 方法1: CLI コマンドで追加する（推奨）

`claude mcp add` コマンドを使って MCP サーバーを追加します。

```bash
claude mcp add playwright -- npx -y @anthropic-ai/mcp-playwright
```

この1行で、Playwright MCP サーバーがプロジェクトに追加されます。設定ファイルの手動編集は不要です。

```bash
# HTTP 接続タイプのサーバーを追加する場合
claude mcp add my-server --transport http --url https://example.com/mcp

# 追加済みのサーバーを一覧表示
claude mcp list

# サーバーを削除
claude mcp remove playwright
```

### 方法2: .mcp.json に直接記述する

プロジェクトルートの `.mcp.json` ファイルに JSON 形式で記述する方法もあります。

```json
{
  "mcpServers": {
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-playwright"]
    }
  }
}
```

チームでリポジトリに設定を共有する場合、`.mcp.json` をコミットしておくと、メンバー全員が同じ MCP サーバーを使えます。

### 接続タイプ

MCP サーバーとの接続方式は主に2つあります。

| タイプ | 説明 | 用途 |
|---|---|---|
| **stdio** | ローカルでプロセスを起動して標準入出力で通信 | ほとんどの MCP サーバー |
| **http** | HTTP(S) 経由でリモートサーバーに接続 | クラウドサービスとの連携 |

どちらを選ぶかは、MCP サーバーがどこで動くかで決まります。

- **ローカルで動く MCP サーバー**: `stdio` を使う。Playwright、ファイルシステム操作、LSP など、あなたのマシン上でプロセスとして起動するタイプ。ほとんどの MCP サーバーはこちら
- **リモートで動く MCP サーバー**: `http` を使う。クラウド上の API サービス（外部の SaaS が MCP エンドポイントを提供している場合など）に接続するタイプ

迷ったら `stdio` です。MCP サーバーの README に `npx` や `node` で起動するコマンドが書かれていれば `stdio`、URL が指定されていれば `http` と判断できます。

> 📝 この他に `sse`（Server-Sent Events）や `ws`（WebSocket）といった接続タイプもありますが、よく使うのは `stdio` と `http` の2つです。

### サーバーの管理

MCP サーバーの状態は Claude Code 内で確認・管理できます。

```
> /mcp
```

このコマンドで、現在接続されている MCP サーバーの一覧と、各サーバーが提供するツールの一覧が表示されます。

## どんな MCP サーバーがあるか

MCP サーバーは多くの開発者やコミュニティによって公開されています。代表的なものを紹介します。

| サーバー | 提供する能力 |
|---|---|
| **Playwright** | ブラウザの操作（ページ遷移、クリック、入力、スクリーンショット） |
| **GitHub** | GitHub の操作（リポジトリ検索、Issue 作成、PR 操作） |
| **Sentry** | エラー監視サービスとの連携（エラー情報の取得） |
| **Slack** | Slack との連携（メッセージの送受信） |
| **Figma** | Figma のデザインデータの参照 |

> 💡 利用可能な MCP サーバーの一覧は、Claude Code 内で `/mcp` コマンドを実行するか、MCP の公式レジストリで確認できます。

## MCP サーバーを追加するときの注意点

MCP サーバーは強力ですが、注意すべき点もあります。

**セキュリティ**

MCP サーバーはあなたの環境で動作するプログラムです。信頼できるソースから提供されたサーバーのみを使用してください。特に、認証情報（API キーなど）を扱うサーバーは、その情報がどのように管理されるかを確認しましょう。

**パフォーマンス**

MCP サーバーを多数追加すると、Claude Code の起動時間が長くなる場合があります。使わないサーバーは設定から削除するか、後述する Sub-agents のスコープ機能で必要なときだけ有効にしましょう。

**コンテキストの消費**

MCP サーバーが提供するツールの説明は、コンテキストウィンドウの一部を消費します。多数のサーバーを追加すると、ツール定義だけでコンテキストを圧迫する可能性があります。

## 🏃 実践: Playwright MCP サーバーを追加する

Claude Code にブラウザ操作の能力を追加してみましょう。Playwright MCP サーバーを使うと、Claude Code がブラウザを操作してスクリーンショットを撮影できるようになります。

> 📌 この実践には Node.js がインストールされている必要があります。`node --version` でバージョンが表示されることを確認してください。

### Step 1: MCP サーバーを設定する

Claude Code が起動している場合は、一度終了（Ctrl + D）してください。ターミナルで以下のコマンドを実行します。

```bash
claude mcp add playwright -- npx -y @anthropic-ai/mcp-playwright
```

MCP サーバーが追加されます。設定内容を確認してみましょう。

```bash
claude mcp list
```

Playwright サーバーが一覧に表示されるはずです。確認したら、Claude Code を再度起動します。

```bash
claude
```

### Step 2: MCP サーバーの接続を確認する

MCP サーバーが正しく接続されているか確認します。

```
> /mcp
```

Playwright サーバーが一覧に表示され、提供するツール（`browser_navigate`、`browser_take_screenshot` など）が確認できるはずです。

> ⚠️ **よくあるエラー**: MCP サーバーが「disconnected」と表示される
>
> ```
> playwright: disconnected
> ```
>
> **原因**: `npx` コマンドが見つからない、または Node.js がインストールされていない可能性があります。
>
> **対処法**: ターミナルで `node --version` と `npx --version` を実行し、両方がインストールされていることを確認してください。問題がなければ、Claude Code を再起動（`/quit` → `claude`）してみてください。

### Step 3: 日報管理画面のスクリーンショットを撮影する

MCP サーバーが接続できたら、2-3-1 で作成した日報管理画面を撮影してみましょう。Sail が起動していることを確認してから実行してください。

> 📌 日報のテストデータが入っていない場合は、先にブラウザで `http://localhost/daily-reports/create` から数件登録しておくとスクリーンショットが見やすくなります。

```
> Playwright で http://localhost/daily-reports にアクセスして、スクリーンショットを screenshot.png として保存して
```

Claude Code が Playwright MCP サーバーを使ってブラウザを起動し、日報の一覧画面にアクセスし、スクリーンショットを撮影してファイルに保存します。

### Step 4: MCP サーバーの削除方法を確認する

MCP サーバーが不要になった場合の削除方法も確認しておきましょう。今回は削除せず、コマンドの確認だけ行います。

```bash
claude mcp list
```

一覧に表示されるサーバー名を確認してください。削除する場合は `claude mcp remove <サーバー名>` を実行します。今回は Playwright サーバーをそのまま残しておきます。

**ここで起きたことを振り返りましょう。** あなたは Claude Code に「スクリーンショットを撮って」と指示しただけです。Claude Code は MCP サーバー経由で Playwright を操作し、ブラウザの起動、ページ遷移、スクリーンショット撮影、ファイル保存までを自律的に行いました。MCP がなければ不可能だった操作が、コマンド1つの追加で実現しています。

> 🧠 先輩エンジニアの思考: 「MCP サーバーは外部プログラムを自分の環境で動かすわけだから、信頼できるソースかどうかは必ず確認する。npm パッケージと同じで、作者やダウンロード数、メンテナンス状況を見る習慣をつけたい。」

## ✨ まとめ

- **MCP（Model Context Protocol）** は Claude Code と外部ツール・サービスを接続する標準プロトコル
- Claude Code がクライアント、外部プログラムが MCP サーバーとして動作する
- MCP サーバーは **ツール**（操作）と **リソース**（情報）を提供する
- `claude mcp add` コマンド（推奨）または `.mcp.json` ファイルで設定し、`/mcp` コマンドで管理する
- Playwright、GitHub、Sentry など、多様な MCP サーバーが公開されている
- セキュリティ、パフォーマンス、コンテキスト消費に注意する
