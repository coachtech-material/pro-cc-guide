# 2-8-1 MCP で外部ツールに接続する

## 概要

これまでの Chapter で、Claude Code はファイルの読み書き、コマンドの実行、Git 操作など、多くのことをこなしてきました。しかし、できないこともあります。たとえば、データベースの中身を直接見ることです。

「users テーブルに何件データが入っている？」と聞いても、Claude Code はデータベースに接続する手段を持っていないため、マイグレーションファイルやシーダーから推測するしかありません。実際のデータを確認するには、皆さんが `sail mysql` でログインして SQL を叩き、その結果を伝える必要がありました。

MCP（Model Context Protocol）を使えば、この制約を取り払えます。Claude Code がデータベースに直接接続し、テーブル構造やデータを確認しながら作業できるようになります。さらに、ブラウザの画面を見せることも可能です。

このセクションでは、MCP の概念を理解し、MySQL とブラウザの2つの接続を設定します。

## MCP とは

### AI ツールと外部をつなぐオープンスタンダード

MCP（Model Context Protocol）は、AI ツールと外部のデータソースやサービスを接続するためのオープンスタンダードです。Claude Code に限らず、さまざまな AI ツールが MCP をサポートしています。

MCP を使うと、Claude Code に「新しい能力」を追加できます。

- **データベース**: テーブル構造の確認、データの検索・集計
- **ブラウザ**: Web ページのスクリーンショット撮影、UI の確認
- **GitHub**: Issue の作成、PR のレビュー
- **Slack**: メッセージの送受信、チャンネルの操作

これらは MCP サーバーとして提供されます。Claude Code が MCP クライアントとなり、MCP サーバーを通じて外部サービスにアクセスする仕組みです。

### `claude mcp add` コマンド

MCP サーバーの追加は `claude mcp add` コマンドで行います。接続方式によって構文が異なります。

ローカルプロセス（stdio）の場合:

```bash
claude mcp add --transport stdio [オプション] <サーバー名> -- <コマンド> [引数...]
```

リモートサーバー（http）の場合:

```bash
claude mcp add --transport http [オプション] <サーバー名> <URL>
```

ポイントは2つです。

1. **オプションはサーバー名の前に置く**: `--transport`、`--env`、`--scope`、`--header` などのオプションは、サーバー名より前に指定します
2. **stdio では `--` でコマンドを分離する**: サーバー名の後に `--` を置き、その後に実行コマンドを書きます（http では `--` は不要で、URL を直接指定します）

## MySQL に接続する

### 接続設定

テック記事プラットフォームのディレクトリで、MySQL の MCP サーバーを追加します。Laravel Sail を起動した状態で以下のコマンドを実行してください。

```bash
claude mcp add --transport stdio --env MYSQL_HOST=127.0.0.1 --env MYSQL_PORT=3306 --env MYSQL_USER=sail --env MYSQL_PASSWORD=password --env MYSQL_DATABASE=laravel mysql -- npx -y @benborla29/mcp-server-mysql
```

コマンドが長いので分解して見てみましょう。

- `--transport stdio`: ローカルプロセスとして起動する方式です。MCP サーバーの標準入出力を通じて通信します
- `--env MYSQL_HOST=127.0.0.1 ...`: MySQL への接続情報を環境変数で渡しています。Laravel Sail のデフォルト設定に合わせています
- `mysql`: このサーバーの名前です。Claude Code 内で識別するために使います
- `-- npx -y @benborla29/mcp-server-mysql`: `--` の後が実際に実行されるコマンドです。npm パッケージを `npx` で実行しています

💡 `--env` で渡している接続情報は、プロジェクトの `.env` ファイルに書かれている `DB_HOST`、`DB_PORT`、`DB_USERNAME`、`DB_PASSWORD`、`DB_DATABASE` と同じ値です。Laravel Sail のデフォルト設定をそのまま使っています。

> ⚠️ よくあるエラー: 接続できない
>
> ```
> MCP server "mysql" failed to start
> ```
>
> **原因**: Laravel Sail が起動していないか、MySQL コンテナがまだ準備中です。
>
> **対処法**: `sail up -d` で Sail を起動し、`sail mysql -e "SELECT 1"` で接続確認してから再試行してください。

### 接続を確認する

Claude Code を起動し、`/mcp` コマンドで MCP サーバーの接続状態を確認します。

```bash
claude
```

```
> /mcp
```

`mysql` サーバーが「connected」と表示されれば接続成功です。利用可能なツール（`query`、`list_tables` など）も表示されます。

### データベースに質問する

接続できたら、Claude Code にデータベースの内容を聞いてみましょう。

```
> 現在のデータベースにあるテーブル一覧と、各テーブルのレコード数を教えてください
```

Claude Code が MCP サーバーを通じて SQL を実行し、テーブル一覧とレコード数を返してくれます。これまではマイグレーションファイルを読んで推測するしかなかった情報が、実際のデータベースから取得されています。

もう少し踏み込んだ質問もしてみましょう。

```
> articles テーブルの status カラムの値ごとの件数を集計してください
```

Claude Code が `SELECT status, COUNT(*) FROM articles GROUP BY status` のような SQL を実行し、draft / review / published ごとの件数を教えてくれます。シーダーで投入したデータの分布を確認できます。

💡 MCP で実行される SQL は Claude Code のツール実行ログに表示されます。2-3-1 で学んだエージェントループの一部として、どんなクエリが実行されたかを確認できます。意図しないクエリ（大量データの取得や更新系 SQL）が実行されていないかを意識する習慣をつけましょう。

> ⚠️ MCP サーバーはサードパーティ製のパッケージです。Anthropic がすべてのサーバーの正確性やセキュリティを検証しているわけではありません。信頼できるソースのサーバーを選び、接続する環境（本番データベースへの直接接続は避けるなど）に注意してください。

### Transport タイプ

先ほど `--transport stdio` を使いましたが、MCP サーバーの接続方式（Transport タイプ）は複数あります。

| Transport | 用途 | 例 |
|---|---|---|
| `stdio` | ローカルで動くサーバー | MySQL、ファイルシステム操作 |
| `http` | リモートサーバー（推奨） | クラウドサービスの API |
| `sse` | リモートサーバー（非推奨） | `http` に移行が推奨されている |
| `ws` | 双方向リアルタイム通信 | WebSocket 対応サーバー |

ローカル開発では `stdio` を使うことがほとんどです。チームで共有するリモートの MCP サーバーに接続する場合は `http` を使います。

### MCP スコープ

MCP サーバーの設定は、3つのスコープで管理できます。

| スコープ | 保存先 | 共有範囲 |
|---|---|---|
| `local`（デフォルト） | `~/.claude.json`（プロジェクトパス別） | 自分のみ、このプロジェクト |
| `project` | プロジェクトルートの `.mcp.json` | チーム全体（Git で共有） |
| `user` | `~/.claude.json` | 自分のみ、全プロジェクト |

複数のスコープに同名のサーバーが定義されている場合、**local > project > user** の優先順位で解決されます。

先ほどの `claude mcp add` はスコープを指定しなかったため `local`（デフォルト）で保存されました。チームで同じ MCP サーバーを使う場合は `--scope project` で `.mcp.json` に保存し、Git で共有します。

```bash
claude mcp add --scope project --transport stdio --env MYSQL_HOST=127.0.0.1 mysql -- npx -y @benborla29/mcp-server-mysql
```

💡 `project` スコープの `.mcp.json` では、環境変数の展開（`${DB_PASSWORD}` のような記法）が使えます。パスワードを直接書く代わりに環境変数を参照させることで、機密情報を Git にコミットせずに済みます。

## ブラウザに接続する

### Playwright MCP でアプリの画面を見せる

データベースだけでなく、ブラウザも MCP で接続できます。Playwright MCP を追加すると、Claude Code にアプリの画面を見せながら UI の確認や修正を指示できるようになります。

```bash
claude mcp add --transport stdio playwright -- npx -y @playwright/mcp@latest
```

Claude Code を再起動し、`/mcp` で `playwright` サーバーが接続されていることを確認してください。

### 画面を見せて修正を指示する

Sail でアプリを起動した状態（`sail up -d` + `sail npm run dev`）で、Claude Code にアプリの画面を見せてみましょう。

```
> http://localhost で記事一覧ページを開いて、スクリーンショットを撮ってください
```

Claude Code が Playwright MCP を通じてブラウザを操作し、スクリーンショットを撮影します。画面の状態を「見た」上で、修正の指示を出せます。

```
> 記事一覧ページの UI を確認して、改善点があれば提案してください
```

Claude Code が画面のレイアウト・配色・情報の見せ方を分析し、具体的な改善提案をしてくれます。「提案」であることがポイントです。実際に修正するかどうかは皆さんが判断してください。

提案の中で気に入ったものがあれば、そのまま指示できます。

```
> その提案のうち、カードレイアウトの改善を適用してください
```

Claude Code がブラウザで現在の画面を見ながらコードを修正し、再度スクリーンショットを撮って結果を確認する。このサイクルが MCP の力です。

💡 Playwright MCP はスクリーンショットの撮影だけでなく、ボタンのクリック、フォームへの入力、ページ遷移など、ブラウザの操作全般をサポートしています。「ログインページからログインして、ダッシュボードの状態を確認して」のような一連の操作も指示できます。

### MCP サーバーの管理

追加した MCP サーバーは以下のコマンドで管理できます。

```bash
claude mcp list          # 登録済みサーバーの一覧
claude mcp get mysql     # 特定サーバーの詳細
claude mcp remove mysql  # サーバーの削除
```

## 見極めチェック

- [ ] **正しさ**: `/mcp` コマンドで `mysql` サーバーと `playwright` サーバーが「connected」と表示されているか
- [ ] **正しさ**: Claude Code にデータベースの質問をしたとき、マイグレーションファイルからの推測ではなく、実際のデータベースからデータが取得されているか
- [ ] **正しさ**: Playwright MCP でスクリーンショットが撮影でき、画面の状態を正しく認識しているか
- [ ] **品質**: MCP スコープが目的に合っているか（個人利用なら `local`、チーム共有なら `project`）
- [ ] **品質**: `project` スコープの `.mcp.json` でパスワードを直接記載せず、環境変数の展開（`${DB_PASSWORD}`）を使っているか
- [ ] **安全性**: MCP で接続しているデータベースが開発環境であり、本番データベースに接続していないか
- [ ] **安全性**: MCP サーバーのパッケージが信頼できるソースから提供されているか（npm パッケージのダウンロード数、メンテナンス状況を確認する）
- [ ] **安全性**: MCP 経由で実行された SQL がツール実行ログに表示されていることを確認し、意図しない更新系クエリが実行されていないか

> この Section では特に「安全性」に注目してください。MCP サーバーはサードパーティ製のパッケージであり、Anthropic が検証しているわけではありません。接続先の環境（開発か本番か）とパッケージの信頼性を必ず確認しましょう。

## まとめ

- MCP（Model Context Protocol）は、AI ツールと外部サービスを接続するオープンスタンダードです。Claude Code に「新しい能力」を追加できます
- `claude mcp add` コマンドで MCP サーバーを追加します。オプションはサーバー名の前、実行コマンドは `--` の後に置きます
- MySQL MCP を使えば、Claude Code がデータベースに直接接続し、テーブル構造やデータを確認しながら作業できます
- Playwright MCP を使えば、アプリの画面をスクリーンショットで撮影し、UI を見ながら修正を指示できます
- MCP スコープ（local / project / user）で設定の共有範囲を制御できます。チーム共有は `project` スコープが便利です
