# 2-3-1 エージェントループとツール実行ログ

## 概要

前の Chapter では、Claude Code にモデルや CRUD を生成させ、テック記事プラットフォームの土台を作りました。プロンプトを入力すると、マイグレーション、モデル、コントローラ、Blade テンプレート、ルート定義、シーダーが次々と生成されました。

このセクションでは、あのとき裏側で何が起きていたのかを理解します。Claude Code がどのように考え、どのようにファイルを読み書きし、どのように結果を確認しているのか。この仕組みを知ることで、うまくいかないときに「何が原因か」「どう対処するか」を自分で判断できるようになります。

## エージェントループ

### Claude Code は3つのフェーズを繰り返す

Claude Code に指示を出すと、内部で **エージェントループ** と呼ばれるサイクルが動き始めます。このサイクルは3つのフェーズで構成されています。

1. **Gather Context（情報を集める）**: プロジェクトのファイルを読み、コードの構造やリレーションを把握する
2. **Take Action（行動する）**: ファイルの作成・編集、コマンドの実行など、実際の変更を行う
3. **Verify Results（結果を確認する）**: 変更が正しく動くかをテスト実行やファイル確認で検証する

この3フェーズは一度で終わるとは限りません。Verify Results の結果次第で、Claude Code は再び Gather Context に戻り、追加の情報を集めて次のアクションを取ります。テストが失敗すればエラーメッセージを読み、原因を探り、コードを修正し、再度テストを実行します。この繰り返しが、1つのプロンプトから複数ファイルにわたる変更を生み出す仕組みです。

### 2-2 の振り返り: あのとき裏で何が起きていたか

前の Chapter で「3つのモデルと CRUD を作成してください」と指示したとき、Claude Code は以下のように動いていました。

**Gather Context**: まず、プロジェクトの状態を把握します。CLAUDE.md を読んで Sail コマンド体系を理解し、既存のファイル構造を確認し、`composer.json` から Laravel のバージョンを読み取ります。

**Take Action**: 情報を集めた上で、マイグレーションファイルを作成し、モデルにリレーションを定義し、コントローラに検索・フィルタ・ページネーションのロジックを書き、Blade テンプレートを生成し、ルート定義を追加しました。

**Verify Results**: 生成したファイルの整合性を確認し、ルート定義とコントローラのメソッドが対応しているか、モデルのリレーションが正しい方向で設定されているかを検証しました。

皆さんが見ていた「次々とファイルが生成される」という体験の裏で、このサイクルが何度も回っていたのです。

### ループが適応する

エージェントループの動き方は、指示の内容によって変わります。

- **質問だけの場合**: Gather Context だけで完了することもあります。「このプロジェクトの認証方式は？」と聞けば、ファイルを読んで回答するだけです
- **バグ修正の場合**: 3フェーズを何度も繰り返します。テストを実行し、エラーを読み、原因を特定し、修正し、再テストします
- **大規模な機能追加の場合**: 多数のファイルにわたって Take Action と Verify Results を繰り返し、数十ステップのアクションを連鎖させます

皆さんはこのループの途中でいつでも介入できます。Claude Code が間違った方向に進んでいると感じたら、入力欄にメッセージを打って Enter を押せば、Claude Code は作業を中断して方向を修正します。

## ツールと実行ログ

### Claude Code が使うツール

エージェントループの各フェーズで、Claude Code は **ツール** を使って実際の作業を行います。ツールがなければ、Claude Code はテキストで応答することしかできません。ツールがあるからこそ、ファイルを読み、コードを書き、コマンドを実行できるのです。

ツールは大きく5つのカテゴリに分かれています。

| カテゴリ | Claude Code ができること |
|---|---|
| **File operations** | ファイルの読み取り、コードの編集、新規ファイルの作成 |
| **Search** | パターンでのファイル検索、正規表現での内容検索 |
| **Execution** | シェルコマンドの実行、テスト実行、Git 操作 |
| **Web** | Web 検索、ドキュメントの取得、エラーメッセージの調査 |
| **Code intelligence** | 型エラーや警告の検出、定義へのジャンプ、参照の検索 |

前の Chapter で Claude Code が `./vendor/bin/sail artisan migrate` を実行したのは **Execution** ツール、マイグレーションファイルを読んだのは **File operations** ツール、ファイル名を探したのは **Search** ツールです。

### ツール実行ログを読む

Claude Code が作業している最中、ターミナルにはツールの実行ログが表示されます。どのファイルを読んでいるか、どのコマンドを実行しているか、リアルタイムで確認できます。

ログを注意深く見る必要は普段ありません。しかし、Claude Code の動きがおかしいと感じたときや、想定外の変更が行われたときに、ログを振り返ることで「何が起きたか」を追跡できます。

💡 より詳細なログを見たい場合は、`Ctrl+O` で verbose 出力を切り替えられます。ツールの実行引数や戻り値など、通常は省略される詳細情報が表示されます。

### Task List で進捗を確認する

複数ファイルにまたがる大きなタスクでは、Claude Code が自動的に **Task List** （タスクリスト）を作成することがあります。タスクリストは、Claude Code が自分の作業を整理するために生成する進捗リストです。

`Ctrl+T` を押すと、ターミナルのステータスエリアにタスクリストが表示されます。各タスクには「待機中」「進行中」「完了」のインジケータがつき、Claude Code がどこまで進んでいるかを把握できます。

タスクリストはコンテキストのコンパクション（次のセクションで学びます）をまたいでも保持されるので、長い作業でも Claude Code が方向を見失いにくくなります。

## うまくいかないときの対処

エージェントループは万能ではありません。Claude Code が同じエラーを繰り返したり、的外れなファイルを編集し続けたりすることがあります。そんなときの対処法を知っておくことは、日々の作業で必ず役立ちます。

### 判断フロー

以下の順番で対処します。

**ステップ 1: 修正指示を出す**

まずは会話の中で修正を指示します。「そのファイルではなく、ArticleController を修正してください」のように、具体的にどこが違うかを伝えます。

**ステップ 2: `/rewind` で巻き戻す**

修正指示を出しても同じ問題が2回以上続く場合は、Claude Code がコンテキスト内で混乱している可能性があります。`/rewind` コマンドで問題が起きる前の状態に巻き戻しましょう。`/rewind` はコードや会話を過去の時点に戻せます（チェックポイントの詳細は 2-7 で学びます）。

```
> /rewind
```

巻き戻したら、別のアプローチで指示を出し直します。

**ステップ 3: `/clear` でやり直す**

`/rewind` でも解決しない場合は、コンテキストが根本的に汚れています。`/clear` で会話をリセットし、まっさらな状態から始めましょう。

```
> /clear
```

`/clear` を実行すると会話履歴がすべて消えますが、ファイルの変更は残ります。新しいセッションとして、改めて指示を出し直します。

### 無限ループのサイン

特に注意が必要なのは、Claude Code が **無限探索ループ** に入るパターンです。同じエラーメッセージが繰り返し表示されたり、同じファイルを何度も編集しては元に戻したりしている場合、Claude Code は自力では問題を解決できない状態です。

このサインを見つけたら、ただ待つのではなく、上の判断フローに従って介入してください。放置すると、コンテキストが無駄に消費されるだけでなく、意図しない変更が積み重なるリスクがあります。

💡 この判断フロー（修正指示 → `/rewind` → `/clear`）は、この教材の残りの Chapter でも繰り返し使います。「Claude Code がうまく動かない」と感じたら、まずこのフローを思い出してください。

## まとめ

- Claude Code は **エージェントループ** （Gather Context → Take Action → Verify Results）を繰り返してタスクを遂行します
- ツールは5つのカテゴリ（File operations / Search / Execution / Web / Code intelligence）に分かれ、これらがエージェントループの各フェーズで使われます
- `Ctrl+T` で **Task List** を表示し、大きなタスクの進捗を確認できます
- うまくいかないときは **修正指示 → `/rewind` → `/clear`** の順で対処します。Claude Code が同じエラーを繰り返す無限ループは介入のサインです

次のセクションでは、エージェントループを支える「コンテキスト」の管理方法と、作業を中断・再開するためのセッション管理を学びます。長時間の開発を品質を保って進めるための実用知識です。
