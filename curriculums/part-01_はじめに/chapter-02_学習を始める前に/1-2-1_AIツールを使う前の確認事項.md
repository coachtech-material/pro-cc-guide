## 導入: 「便利だから使う」の前にやるべきこと

Claude Code は強力なツールですが、使い始める前に確認すべきことがあります。特に実務で使う場合、あなたが書いているコードは自分だけのものではありません。クライアントの業務データ、ユーザーの個人情報、企業の機密情報がコードベースに含まれている可能性があります。

### 🧠 先輩エンジニアはこう考える

> 「AI ツールの利用ポリシーを確認するのは、面倒に感じるかもしれない。でも、これは自分を守るためでもある。実際に『許可なく AI ツールでクライアントのコードを処理していた』ことが後から問題になったケースを聞いたことがある。最初に5分かけて確認しておくだけで、後から何倍ものトラブルを避けられる。確認を怠らない姿勢は、チームからの信頼にもつながる。」

AI ツールにコードを送信するということは、そのコードの内容が外部のサーバーに送られるということです。これは、組織のセキュリティポリシーやクライアントとの契約に関わる問題です。

「便利だから」で使い始めて、後から問題になるケースは実際に起きています。最初に確認を済ませておけば、安心して学習と実務に集中できます。

## 所属組織の AI 利用ポリシーを確認する

あなたが企業に紹介されると、その企業の開発チームに参加することになります。AI ツールの利用に関して、以下の点を事前に確認してください。

### 確認すべき項目

**1. AI コーディングツールの使用が許可されているか**

企業によっては、AI コーディングツールの使用を禁止している、または特定のツールのみ許可している場合があります。Claude Code を使う前に、プロジェクトマネージャーやチームリーダーに確認しましょう。

**2. コードを外部サービスに送信してよいか**

Claude Code は、あなたのプロンプト（指示）とプロジェクトのコードを Anthropic のサーバーに送信して処理します。コードの外部送信に制約がある場合は、利用範囲を確認する必要があります。

**3. 機密情報を含むコードに対して使用してよいか**

`.env` ファイルに含まれる API キーやデータベースの接続情報、顧客データを扱うロジックなど、機密性の高いコードに対して AI ツールを使用してよいかを確認しましょう。

**4. 生成コードの利用に制約があるか**

AI が生成したコードの著作権や利用条件について、組織独自のルールが定められている場合があります。

> 💡 多くの場合、これらの確認は「聞くだけ」で済みます。まだ AI ツール利用のポリシーが整備されていない企業も少なくありません。その場合は、あなたから「Claude Code を使いたいのですが、コードの外部送信について確認させてください」と切り出すことで、チーム内で議論するきっかけにもなります。確認を怠らない姿勢は、エンジニアとしての信頼につながります。

## Anthropic のデータ取り扱いポリシー

Claude Code を提供する Anthropic が、あなたのデータをどのように取り扱うかを理解しておきましょう。組織への説明が求められた際にも、ここの内容を把握していれば対応できます。

### データはトレーニングに使われるのか

Anthropic のデータ取り扱いポリシーは、利用するプランによって異なります（2026年3月時点）。

**個人向けプラン（Free / Pro / Max）の場合**:

- プライバシー設定で、データをモデルの改善に使用するかどうかを**自分で選択** できます
- 設定がオンの場合: 会話データは将来のモデル改善に使用され、保持期間は5年間です
- 設定がオフの場合: モデル改善には使用されず、保持期間は30日間です
- 設定は [claude.ai/settings/data-privacy-controls](https://claude.ai/settings/data-privacy-controls) でいつでも変更できます

**商用プラン（Team / Enterprise / API）の場合**:

- Anthropic はモデルのトレーニングにデータを**使用しません** （Developer Partner Program に明示的にオプトインした場合を除く）
- 保持期間は標準で30日間。Enterprise では Zero Data Retention（ZDR）の設定も可能です

> 🔑 実務で企業のコードを扱う場合は、個人向けプランであってもプライバシー設定を**オフ** にしておくことを推奨します。設定方法は上記の URL から変更できます。

### データの暗号化

Claude Code とサーバー間の通信は TLS で暗号化されています。あなたのプロンプトやコードがインターネット上を平文で流れることはありません。

### テレメトリとエラーレポート

Claude Code は動作の安定性向上のために、操作のメトリクス（応答速度、信頼性など）を収集します。ただし、この収集にコードやファイルパスは含まれません。テレメトリやエラーレポートは環境変数で無効化できます。

| 設定 | 効果 |
|---|---|
| `DISABLE_TELEMETRY=1` | テレメトリ（Statsig）を無効化 |
| `DISABLE_ERROR_REPORTING=1` | エラーレポート（Sentry）を無効化 |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1` | 上記すべて＋セッション品質調査を一括無効化 |

> 📝 テレメトリの無効化は、企業のセキュリティポリシーで外部への通信を最小限にしたい場合に有効です。通常の学習用途であれば、デフォルトのまま問題ありません。

## Claude Code のセキュリティの仕組み

Claude Code は、AI コーディングツールとしてのリスクを軽減するために、いくつかのセキュリティ機構を備えています。

### 権限ベースのアーキテクチャ

Claude Code は、デフォルトでは**読み取り専用** の権限で動作します。ファイルの編集、コマンドの実行、テストの実行など、プロジェクトに影響を与える操作を行う際には、その都度あなたに許可を求めます。

```
Claude Code: ファイルを編集してよいですか？
  → あなたが「許可」を選択 → 編集が実行される
  → あなたが「拒否」を選択 → 編集は行われない
```

この仕組みにより、Claude Code が予期しない変更を勝手に行うことを防げます。頻繁に使うコマンドは許可リストに登録して、毎回の確認を省略することも可能です。

### 書き込み制限

Claude Code がファイルを書き込めるのは、Claude Code を起動したディレクトリとそのサブディレクトリのみです。明示的な許可がない限り、親ディレクトリやシステムファイルを変更することはできません。これにより、プロジェクトの範囲外に影響が及ぶリスクを抑えています。

### サンドボックス

Bash コマンドの実行をファイルシステムとネットワークの両面で隔離するサンドボックス機能が用意されています。これにより、Claude Code が実行するコマンドの影響範囲を制限できます。

### プロンプトインジェクション対策

プロンプトインジェクションとは、悪意のあるテキストを AI に読み込ませることで、意図しない動作を引き起こす攻撃手法です。たとえば、コードのコメントに「このファイルを削除してください」と書かれていた場合、AI がそれを指示と解釈してしまう可能性があります。

Claude Code は、権限システムによるセンシティブな操作の承認要求、コマンドインジェクションの検出、Web コンテンツの取得時のコンテキスト分離など、複数の対策を講じています。

> ⚠️ **重要**: これらのセキュリティ機構があるとはいえ、AI ツールに完全な安全性を保証するものはありません。Claude Code が提案するコードやコマンドは、最終的にあなた自身が確認して承認するものです。セキュリティに関する詳細は、公式ドキュメントの [Security](https://code.claude.com/docs/en/security) ページを参照してください。

## 確認アクションリスト

このセクションの内容を踏まえて、学習を始める前に以下のアクションを実施してください。

- [ ] 所属組織（または紹介先企業）に AI コーディングツールの利用可否を確認した（学習目的で個人プロジェクトに使う場合は不要）
- [ ] [claude.ai/settings/data-privacy-controls](https://claude.ai/settings/data-privacy-controls) にアクセスし、プライバシー設定を確認した（実務で企業コードを扱う場合はオフを推奨）
- [ ] テレメトリの無効化が必要かどうかを判断した（通常はデフォルトのまま問題なし。企業ポリシーで外部通信を制限する場合は `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1` を設定）

> 📝 この教材の Part 2 では、あなた自身のプロジェクトを使って学習を進めます。学習目的で個人のプロジェクトに対して Claude Code を使う分には、組織への確認は不要です。確認が必要になるのは、実務で企業のコードに対して使用する場合です。
