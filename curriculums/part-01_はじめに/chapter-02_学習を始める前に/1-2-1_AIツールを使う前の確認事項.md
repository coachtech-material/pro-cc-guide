# 1-2-1 AI ツールを使う前の確認事項

## 🎯 このセクションで学ぶこと

- 実務で AI コーディングツールを使う際に、なぜ事前確認が必要かを理解する
- 所属組織の AI ツール利用ポリシーの確認ポイントを把握する
- Claude Code におけるデータの取り扱い（学習利用・保持期間）を正確に理解する
- Claude Code のセキュリティの仕組みを概観する

---

## なぜ「確認」から始めるのか

「早く Claude Code を使ってみたい」という気持ちはよくわかります。でも、実務で AI コーディングツールを使い始める前に、必ず確認しておくべきことがあります。

実務のコードベースには、企業の機密情報やユーザーの個人情報が含まれています。Claude Code にプロジェクトのコードを読ませるということは、そのコードが外部サービスに送信されるということです。これが許可されていない環境で使ってしまうと、最悪の場合、契約違反や情報漏洩の問題に発展します。

「知らなかった」では済まされない。だからこそ、最初に確認するのです。

---

## 🧠 先輩エンジニアの思考プロセス

自分が初めて AI コーディングツールを実務に導入しようとしたとき、チームリーダーに「まずセキュリティポリシーを確認しろ」と言われました。正直、面倒だと思いました。でも実際に確認してみると、クライアント企業との NDA（秘密保持契約）に「外部サービスへのソースコードの送信禁止」という条項があったのです。もし確認せずに使い始めていたら、契約違反になるところでした。

AI ツールの導入は、技術の問題ではなくビジネスの問題でもあります。特にフリーランスとして企業に参画するあなたは、「自分の判断で勝手に使い始めた」と言われないよう、事前の確認と合意が大切です。

---

## 所属組織の AI 利用ポリシーを確認する

最初に確認すべきは、あなたが所属する組織（紹介先の企業やエージェント）の AI ツール利用ポリシーです。

### 確認すべきポイント

**1. AI コーディングツールの利用が許可されているか**

そもそも AI コーディングツールの利用が許可されていない企業もあります。許可されている場合でも、「承認済みのツールのみ」「特定のプロジェクトのみ」といった条件がつくことがあります。

**2. 外部サービスへのコード送信が許可されているか**

Claude Code は、あなたのコードをクラウド上の AI モデルに送信して処理します。NDA や契約書に「ソースコードの外部送信禁止」といった条項がないか確認してください。

**3. 利用にあたっての手続き**

利用開始前に上長の承認が必要か、セキュリティ部門への申請が必要か、利用規約への同意が必要かなどを確認します。

**4. 生成コードの扱い**

AI が生成したコードの著作権や知的財産権について、組織としての方針が定められている場合があります。

> 💡 確認先がわからない場合は、まず COACHTECH のメンターやエージェントの担当者に相談してください。「AI コーディングツールを使いたいのですが、利用ポリシーはありますか？」と聞くだけで十分です。ポリシーが整備されていない場合でも、「確認した上で使っている」という事実が、後からあなたを守ります。

---

## Claude Code のデータの取り扱い

組織の許可を確認したら、次は Claude Code 側のデータの取り扱いを理解しましょう。「自分のコードがどう扱われるのか」を正確に把握しておくことは、組織への説明責任を果たす上でも重要です。

### AI モデルの学習に使われるか

Claude Code に送信したコードやプロンプトが、AI モデルの学習（トレーニング）に使われるかどうかは、利用するプランによって異なります。

| プラン種別 | 学習利用 | 備考 |
|---|---|---|
| **個人プラン（Pro、Max）** | ユーザーが選択可能 | claude.ai の設定画面（Data Privacy Controls）でオプトイン/オプトアウトを切り替えられる |
| **商用プラン（Team、Enterprise、API）** | デフォルトで学習に使わない | ユーザーが明示的に同意しない限り、学習に利用されない |

> 🔑 個人プラン（Pro、Max）を使う場合は、設定画面で学習利用をオフ（オプトアウト）にすることをおすすめします。特に実務のコードを扱う場合は、必ず確認してください。

### データの保持期間

Claude Code に送信したデータがサーバーにどのくらい保持されるかも、プランと設定によって異なります。

| 条件 | 保持期間 |
|---|---|
| 個人プランで学習利用をオン（オプトイン） | 最大5年 |
| 個人プランで学習利用をオフ（オプトアウト） | 30日 |
| 商用プラン（Team、Enterprise） | 30日（標準） |
| Enterprise（Zero Data Retention 設定） | 即時削除も可能 |

また、Claude Code はローカルにもセッションデータをキャッシュします。これはセッションの再開（途中から続きを行う）に使われるもので、最大30日間保持されます。

### テレメトリ（利用状況データ）

Claude Code は、ツールの改善を目的として匿名の利用状況データ（テレメトリ）を送信します。コードの中身やファイルパスは含まれません。

テレメトリの送信が気になる場合は、環境変数で無効化できます。

```bash
# すべての非必須テレメトリを無効化
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
```

> 📝 テレメトリの無効化は Part 2 のセットアップ時に設定できます。今は「無効化できる」ということだけ覚えておいてください。

## Claude Code のセキュリティの仕組み

データの取り扱いと合わせて、Claude Code がどのようなセキュリティ対策を備えているかも把握しておきましょう。ここでは概要のみ紹介します。詳細は Part 2 で実際に設定しながら学びます。

### 権限の制御

Claude Code は、デフォルトでは **読み取り専用** で動作します。ファイルの編集やコマンドの実行には、あなたの明示的な許可が必要です。

```
あなたの指示
    ↓
Claude Code が計画を立てる
    ↓
ファイル編集やコマンド実行が必要
    ↓
あなたに許可を求める ← ここで確認できる
    ↓
許可された操作のみ実行
```

一度許可した操作を自動的に許可する設定（許可リスト）もありますが、最初は都度確認する運用がおすすめです。

### ファイルシステムの制限

Claude Code は、起動したディレクトリとそのサブディレクトリにのみ書き込みができます。プロジェクトの外のファイルを書き換えてしまう心配はありません。読み取りについてはプロジェクト外（システムライブラリ等）も可能ですが、書き込みは制限されています。

### コマンドのブロックリスト

危険なコマンド（`curl`、`wget` など外部からのダウンロードに使われるコマンド）は、デフォルトでブロックされています。また、コマンドインジェクション（悪意のある入力によるコマンド実行）の検出機能も備えています。

### 通信の暗号化

Claude Code とクラウド上の AI モデルとの通信は、TLS（Transport Layer Security）で暗号化されています。VPN 環境でも利用可能です。

## 確認チェックリスト

以下のチェックリストで、AI ツールを使い始める準備が整っているか確認してください。

### 組織への確認

- [ ] 所属組織の AI ツール利用ポリシーを確認した（またはポリシーがないことを確認した）
- [ ] 外部サービスへのコード送信が許可されていることを確認した
- [ ] 必要な申請・承認手続きがあれば完了した（または手続き不要であることを確認した）
- [ ] クライアント企業との NDA に AI ツール利用を制限する条項がないことを確認した

### データの取り扱いの理解

- [ ] 自分が使うプランでのデータの学習利用ポリシーを理解した
- [ ] 個人プランの場合、学習利用のオプトアウト設定の場所を把握した
- [ ] データの保持期間を理解した

### セキュリティの理解

- [ ] Claude Code がデフォルトで読み取り専用であることを理解した
- [ ] ファイル書き込みの範囲が制限されていることを理解した
- [ ] 権限の許可は自分で制御できることを理解した

> ⚠️ **学習目的で個人プロジェクトに対して使う場合** は、組織への確認は不要です。ただし、後から実務プロジェクトで使い始める際には、必ず上記の確認を行ってください。

---

## ✨ まとめ

- 実務で AI コーディングツールを使う前に、所属組織の利用ポリシーを必ず確認する
- Claude Code に送信したコードの学習利用は、プランと設定によって制御できる。個人プランではオプトアウト設定を推奨
- データの保持期間は、個人プラン（オプトアウト時）で30日、商用プランで30日が標準
- Claude Code はデフォルトで読み取り専用。ファイル編集やコマンド実行にはあなたの許可が必要
- 「確認した上で使っている」という事実が、あなたを守る最大の盾になる
