# 1-2-1 AI ツールを使う前の確認事項

## 概要

Claude Code は強力なツールですが、業務で使い始める前に確認すべきことがあります。所属する企業・チームの AI ツール利用ポリシー、コードや機密情報の取り扱い、そして AI が生成したコードに対する自分自身の責任です。

フリーランスや副業で企業に参画する皆さんにとって、これらの確認を怠ることは信頼問題につながりかねません。このセクションでは、Claude Code を安心して使い始めるために知っておくべきことを整理します。

## 企業・チームの AI ツール利用ポリシー

### 確認すべきポリシー項目

企業に所属して開発を行う場合、AI ツールの利用に関するポリシーが定められていることがあります。Claude Code を使い始める前に、以下の項目を確認してください。

- **AI ツールの利用可否**: そもそも業務で AI コーディングツールを使ってよいか。許可されているツールの種類（Claude Code、GitHub Copilot 等）に制限があるか
- **コードの外部送信**: 業務コードを AI サービスに送信してよいか。機密性の高いプロジェクトでは禁止されている場合がある
- **生成コードの取り扱い**: AI が生成したコードをそのままコミットしてよいか。レビュープロセスに追加の要件があるか
- **利用するプラン**: 企業が契約しているプラン（Teams や Enterprise）があるか。個人の Pro / Max プランで業務コードを扱ってよいか

ポリシーが明文化されていない場合もあります。その場合は、チームリーダーやマネージャーに確認してから使い始めるのが安全です。

💡 COACHTECH のカリキュラムやポートフォリオ開発で使う分には、個人の Pro / Max プランで問題ありません。企業の業務コードを扱う場合に、上記の確認が必要になります。

### Managed settings による組織的なポリシー強制

企業が Claude Code の Teams プランや Enterprise プランを契約している場合、管理者が Managed settings（管理された設定）を使って組織全体にポリシーを強制できます。

たとえば、管理者が以下のような制御を行えます。

- 特定のコマンドの実行を許可または禁止する
- 外部ツール（MCP サーバー）への接続を制限する
- テレメトリ（利用状況データ）の送信設定を組織で統一する

Managed settings が適用されている環境では、皆さんが個別に設定を変更できない項目があります。設定を変更しようとしてエラーが出た場合は、組織の管理者が制御している可能性があるため確認してください。

## コードと機密情報の取り扱い

Claude Code を使うと、皆さんが入力したプロンプトやプロジェクトのコードが Anthropic のサーバーに送信されます。ここでは、そのデータがどのように扱われるかを理解しておきましょう。

### データ訓練ポリシーの違い

Claude のプランは大きく2つのカテゴリに分かれ、データの訓練利用に関するポリシーが異なります。

**Consumer プラン（Free / Pro / Max）**

ユーザーが自分のデータを将来のモデル改善に使うことを許可するかどうかを選択できます。許可した場合、Claude Code で送信したコードやプロンプトがモデルの訓練に使われる可能性があります。この設定は [claude.ai のプライバシー設定](https://claude.ai/settings/data-privacy-controls) からいつでも変更できます。

**Commercial プラン（Teams / Enterprise / API / Claude Gov / サードパーティプラットフォーム）**

Anthropic は、商用プランで送信されたコードやプロンプトをモデルの訓練に使用しません（ユーザーが明示的に Developer Partner Program にオプトインした場合を除く）。企業の業務コードを扱う場合は、この違いを理解しておくことが重要です。

⚠️ 個人の Pro / Max プランで業務コードを扱う場合、プライバシー設定でデータの訓練利用を「許可しない」に設定することを強く推奨します。企業の機密コードがモデルの訓練データに含まれるリスクを避けるためです。

### データ保持期間

Anthropic はプランと設定に応じて、送信されたデータを一定期間保持します。

| プランカテゴリ | 訓練利用の設定 | 保持期間 |
|---|---|---|
| Consumer（Free / Pro / Max） | 許可あり | 5年 |
| Consumer（Free / Pro / Max） | 許可なし | 30日 |
| Commercial（Teams / Enterprise / API） | — | 30日 |

Commercial プランでは、Enterprise 向けに Zero Data Retention（データ保持なし）のオプションも用意されています。

### テレメトリ・エラーレポートのオプトアウト

Claude Code は、利用状況のメトリクス（テレメトリ）とエラーログを Anthropic に送信します。これらにはコードやファイルパスは含まれませんが、必要に応じてオプトアウト（送信停止）できます。

| 送信内容 | 説明 | オプトアウト方法 |
|---|---|---|
| テレメトリ（利用状況） | レイテンシ、信頼性、利用パターン等の指標 | 環境変数 `DISABLE_TELEMETRY=1` を設定 |
| エラーレポート | 運用上のエラーログ | 環境変数 `DISABLE_ERROR_REPORTING=1` を設定 |

これらの環境変数は、Claude Code の設定ファイル（`settings.json`）に記述することもできます。設定ファイルの詳細は Part 2 で学びます。

💡 テレメトリとエラーレポートは、ツールの品質向上に役立つデータです。特別な理由がなければオプトアウトする必要はありません。企業のセキュリティポリシーで外部へのデータ送信が制限されている場合に、この設定が必要になります。

### MCP サーバー経由のデータ漏洩リスク

Claude Code は MCP（Model Context Protocol）を通じて外部ツールと連携できます。MCP の詳細は Part 2 の Chapter 2-8 で学びますが、セキュリティの観点で1つ重要なことを先にお伝えします。

Anthropic は MCP サーバーの管理・監査を行いません。MCP サーバーは誰でも開発・公開できるため、信頼できないサーバーを接続すると、プロジェクトのコードや機密情報が意図しない外部サービスに送信されるリスクがあります。MCP サーバーを導入する際は、自分で開発したもの、または信頼できる提供元のものを使用してください。

## AI 生成コードに対する責任

ここまで、組織のポリシーとデータの取り扱いについて確認しました。最後に、AI が生成したコードに対する「責任」について考えます。

Claude Code が生成するコードの品質は高いですが、完璧ではありません。生成されたコードをレビューせずにコミットし、本番環境で問題が発生した場合、その責任は Claude Code ではなく皆さん自身にあります。これは AI コーディングツール全般に共通する原則です。

### 見極める力の3観点を具体化する

セクション 1-1-1 で、見極める力（Output）の3つの観点を紹介しました。

- **正しさ**: 要件通りに動くか
- **品質**: 保守性・パフォーマンス・既存設計との一貫性は適切か
- **安全性**: 脆弱性がないか。認可・入力検証・機密情報の漏洩がないか

確認の手段はコードリーディング、ブラウザでの動作確認、テスト、セキュリティレビューなど場面によって異なります。ここでは、それぞれの観点で「何を確認するか」をもう少し具体的に見てみましょう。

**正しさの確認例**:
- 生成されたマイグレーションのカラム型・制約が要件と一致しているか
- Controller のバリデーションルールが仕様書の入力条件を網羅しているか
- テストケースが正常系だけでなく、異常系（空文字、上限値超過、未認証アクセス等）もカバーしているか

**品質の確認例**:
- 既存プロジェクトの命名規則（テーブル名、変数名、メソッド名）に従っているか
- N+1 クエリのような非効率なデータ取得がないか
- 既存の共通処理（Helper、Trait、Service クラス等）を重複して再実装していないか

**安全性の確認例**:
- ユーザー入力がバリデーションなしで直接クエリに渡されていないか（SQL インジェクション）
- Blade テンプレートで `{!! !!}`（エスケープなし出力）が不必要に使われていないか（XSS）
- API エンドポイントに適切な認可チェック（Policy、Middleware）が設定されているか

これらの確認項目は、皆さんが COACHTECH で学んだ知識そのものです。Part 2 以降のハンズオンでは、各セクションの末尾に「見極めチェック」として具体的なチェックリストを配置します。生成されたコードを自分の目で読み、理解し、判断する習慣をここから身につけていきましょう。

## まとめ

- 企業の業務で使う場合は、AI ツール利用ポリシー（利用可否、コードの外部送信、生成コードの取り扱い）を事前に確認する
- Consumer プラン（Pro / Max）と Commercial プラン（Teams / Enterprise / API）ではデータ訓練ポリシーが異なる。業務コードを扱う場合はプライバシー設定を確認する
- MCP サーバー経由のデータ漏洩リスクに注意し、信頼できるサーバーのみを使用する
- AI が生成したコードの責任は自分自身にある。正しさ・品質・安全性の3観点で検証する習慣を持つ
