# 1-1-2 対象読者と前提条件

## 概要

このセクションでは、この教材がどのような方を対象としているか、学習を始めるために必要な技術スキルと開発環境を確認します。自分のスキルレベルと照らし合わせて、この教材が適切かを判断してください。

## この教材の対象読者

この教材は、COACHTECH を卒業し、副業・フリーランス・転職などの形で Laravel を使った実務開発に携わる皆さんを対象としています。

基礎力はあるが実務はこれから、という段階にいる方を想定しています。「現場で通用するか」という不安を持ちつつも、「学んだことを活かしたい」という意欲がある方にとって、Claude Code は強力な武器になります。

## 前提となる技術スキル

この教材では、以下の技術を前提知識として扱います。これらの基本的な説明は省略しますが、Claude Code と組み合わせたときの注意点は丁寧に解説します。

| 技術 | 前提レベル |
|---|---|
| **Laravel** | MVC、Eloquent ORM、認証（Fortify）、認可（Policy）、バリデーション、ミドルウェア、RESTful API、PHPUnit によるテストを実装できる |
| **フロントエンド** | HTML/CSS、Tailwind CSS の基礎がある |
| **データベース** | RDB 設計（正規化、ER 図）、SQL（JOIN、サブクエリ、集計）を扱える |
| **開発プロセス** | 要件定義 → DB/API 設計 → 実装 → テストの一連の開発フローを経験している |
| **開発環境・ツール** | ターミナル操作、Docker Compose、Git/GitHub（ブランチ運用、PR、コードレビュー）を使える |

AI コーディングツールの使用経験は問いません。ChatGPT を少し触った程度でも、全く使ったことがなくても問題ありません。

## 前提となる開発環境

### OS

この教材は macOS（13.0 以上）と Windows（10 以上）の両方を対象としています。RAM は 4GB 以上が必要です。

Windows の場合は WSL（Windows Subsystem for Linux）を経由して操作します。WSL 環境では Docker Desktop との連携に注意が必要です。Docker Desktop の設定で「Use the WSL 2 based engine」が有効になっていることを確認してください。

💡 Windows では WSL を使わず、Git for Windows をインストールして PowerShell や CMD から直接 Claude Code を利用する方法もあります。この教材では WSL 経由を前提としますが、詳しくは公式ドキュメントの [Setup](https://code.claude.com/docs/en/setup) を参照してください。

### エディタ・ブラウザ

エディタは **VS Code** を前提とします。Claude Code はターミナルで動作しますが、VS Code のターミナル内で使うことも、VS Code 拡張として使うこともできます。

ブラウザは Claude Code の認証フロー（初回ログイン時にブラウザでの認証が必要）と、開発中のアプリケーションの動作確認に使用します。

### Docker Desktop

この教材のハンズオンでは Laravel Sail（Docker ベースの開発環境）を使用します。Docker Desktop がインストール済みであることを前提とします。

### Node.js について

Claude Code の利用にあたって、ホストマシンへの Node.js のインストールは不要です。

Claude Code のインストールは、curl によるネイティブインストールが推奨されています。ネイティブインストールは Node.js に依存せず、バックグラウンドで自動更新されます。Homebrew（`brew install --cask claude-code`）や WinGet（`winget install Anthropic.ClaudeCode`）でもインストールできますが、これらは自動更新されないため手動での更新が必要です。以前提供されていた npm でのインストールは現在非推奨（deprecated）です。

⚠️ Claude Code の利用には Pro、Max、Teams、Enterprise、または Console（API）アカウントが必要です。無料の Free プランでは Claude Code を利用できません。企業に所属している場合は、企業のプラン・ポリシーを確認してください。プランの詳細はセクション 1-2-2 で説明します。

また、ハンズオンで使用するフロントエンドのビルド（Tailwind CSS 等）は Laravel Sail のコンテナ内で実行するため（`sail npm install`、`sail npm run dev`）、ホストマシンに Node.js をインストールする必要はありません。

## まとめ

- この教材は、COACHTECH を卒業し Laravel を使った実務開発に携わる皆さんを対象としている

- 開発環境は macOS または Windows（WSL）、エディタは VS Code、Docker Desktop が必要
- Node.js のホストマシンへのインストールは不要（Claude Code はネイティブインストール推奨、フロントエンドビルドは Sail コンテナ内で実行）
