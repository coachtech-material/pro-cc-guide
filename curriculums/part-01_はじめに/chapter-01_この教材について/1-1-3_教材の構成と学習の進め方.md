# 1-1-3 教材の構成と学習の進め方

## 🎯 このセクションで学ぶこと

- 教材の全体構成（4つの Part）とそれぞれの役割を把握する
- Part / Chapter / Section の3層構造を理解する
- 自分に適した学習の進め方を選択できる

まず教材の全体構成を把握し、次に3層構造と Section の種類を理解し、最後に自分に適した学習の進め方を選択します。

---

## 導入: 全体像を把握してから学ぶ

技術書を最初のページから順番に読み進めて、途中で「今やっていることが全体のどこに位置するのかわからない」と感じた経験はないでしょうか。全体像が見えないまま学習を進めると、各トピックの関連性がつかめず、学んだことが断片的な知識で終わってしまいます。

このセクションでは、教材の全体構成を先に示します。各 Part が何を扱い、どのような順序で能力を積み上げていくのかを把握したうえで学習を始めてください。

### 🧠 先輩エンジニアはこう考える

> 「新しいフレームワークやツールを学ぶとき、自分はまず全体の地図を頭に入れるようにしている。最初に全体像を掴んでおくと、個々のトピックを学ぶときに『今やっていることが全体のどこに位置するか』がわかるし、つまずいたときに『この先で詳しくやるから今は大まかに把握すればいい』という判断ができる。この教材も同じ。まず地図を手に入れよう。」

## 教材の全体構成

この教材は4つの Part で構成されています。それぞれが、前のセクションで紹介した3つの能力（使いこなす力・見極める力・学び続ける力）に対応しています。

### Part 1: はじめに（現在地）

教材の導入パートです。Claude Code を学ぶ意義、対象読者、教材の構成を説明し、AI ツールを使い始めるための確認事項とプラン選択を行います。ハンズオンはありません。

**対応する能力**: なし（学習の準備）

### Part 2: Claude Code の基礎

Claude Code の主要機能を習得するパートです。3つの Chapter で構成されています。

- **Chapter 2-1**: セットアップ。インストールと認証を行い、権限やモデル選択、CLAUDE.md の設定まで、自分のプロジェクトで手を動かしながら基盤を整えます
- **Chapter 2-2**: Claude Code の仕組みと基本概念を学びます。エージェントループ、コンテキスト管理、プロンプト設計など、使いこなすための土台を固めながら、cc-practice（Part 2 で作成する学習用の Laravel Sail プロジェクト）で実際に体験します
- **Chapter 2-3**: cc-practice で Claude Code の各機能（Plan Mode、Skills、Hooks、MCP など）を実践的に体験します

**対応する能力**: 使いこなす力 + 見極める力の型の導入

### Part 3: Claude Code の実践

既存プロジェクトで実務タスクを遂行するパートです。提供するプロジェクトを題材に、バグ修正、機能開発、リファクタリングなどの実務タスクを Claude Code と協働して完了させます。Part 2 で個別に学んだ機能を組み合わせ、タスクの要件理解からコードレビュー、PR 作成まで一気通貫で実践します。

**対応する能力**: 見極める力 + 使いこなす力の総合演習

### Part 4: 継続的な学習

学習を習慣化するためのパートです。公式ドキュメントの読み方、新機能の検証方法、情報収集のチャネルなど、この教材を終えた後も自力で成長を続けるための指針を示します。ハンズオンはありません。

**対応する能力**: 学び続ける力

```
Part 1          Part 2              Part 3            Part 4
はじめに   →   Claude Code の基礎  →  実践         →   継続的な学習
（準備）       （機能の習得）        （総合演習）       （自走の準備）
                    │                    │
              使いこなす力          見極める力         学び続ける力
              ＋型の導入           ＋総合演習
```

## 3層構造: Part / Chapter / Section

教材は **Part > Chapter > Section** の3層で構成されています。

- **Part**: 大きなテーマのまとまり。教材全体で4つ
- **Chapter**: Part 内のトピックの区切り。複数の Section で構成される
- **Section**: 学習の最小単位。1つのセクションで1つのテーマを扱う

Section には3つの種類があります。

| 種類 | 内容 | 手を動かす？ |
|---|---|---|
| **概念** | 機能の意義・仕組み・使い方を解説 | いいえ |
| **ハンズオン** | 概念で学んだ機能を実際に使う | はい |
| **混合** | 概念を学びながらすぐに手を動かす | はい |

ハンズオンセクションと混合セクションでは、コードが生成されるたびに **「見極めチェック」** を行います。これは、生成コードを正しさ・品質・安全性の3つの観点で検証するプロセスです。Part 2 から導入し、繰り返し実践することで、AI の出力を自分の判断で採用する習慣を身につけます。

## 学習の進め方

### 基本: Part 1 から順番に進める

この教材は、前の Part で学んだ知識が次の Part の前提になるように設計されています。特に初めて Claude Code を使う方は、Part 1 から順番に進めることを推奨します。

```
Part 1 → Part 2 → Part 3 → Part 4
（この順番で進めるのが基本）
```

### 応用: 必要な Part から始める

すでに Claude Code を使った経験がある方や、特定のトピックだけを学びたい方は、以下を参考に必要な Part から始めることもできます。

| あなたの状況 | 推奨する開始位置 |
|---|---|
| Claude Code を使ったことがない | Part 1 から順番に |
| インストール済みだが機能をしっかり学びたい | Part 2 Chapter 2-2 から |
| 基本機能は理解していて実践したい | Part 3 から（Part 2 を参照しながら） |
| 実務で使っていて学習の習慣化を図りたい | Part 4 から |

ただし、Part 3 は Part 2 の知識を前提としています。Part 3 で行き詰まった場合は、Part 2 の該当セクションに戻って確認してください。各ハンズオンセクションには、前提となる概念セクションへのリンクが記載されています。

## ✨ まとめ

- 教材は Part 1（はじめに）→ Part 2（基礎）→ Part 3（実践）→ Part 4（継続的な学習）の4部構成
- Part > Chapter > Section の3層構造。Section には概念・ハンズオン・混合の3種類がある
- 初めて Claude Code を使う方は Part 1 から順番に、経験者は必要な Part から始められる
