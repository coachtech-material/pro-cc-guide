# 1-1-3 教材の構成と学習の進め方

## 概要

このセクションでは、教材の全体構成と各 Part の役割、Claude Code のどの機能をどの Chapter で学ぶかの学習マップ、そして皆さんに合った学習の進め方を説明します。全体像を把握することで、今自分がどこにいて、次に何を学ぶのかが見通せるようになります。

## 全体構成と各 Part の概要

この教材は4つの Part で構成されています。

| Part | 対応する能力 | 概要 | ハンズオン |
|---|---|---|---|
| Part 1: はじめに | ー | 教材の導入。Claude Code を学ぶ意義と全体像の理解 | なし |
| Part 2: Claude Code の基礎 | 使いこなす力（主）+ 見極める力（型の導入） | 新規プロジェクトを構築しながら主要機能を習得 | テック記事プラットフォームを段階的に構築 |
| Part 3: Claude Code の実践 | 見極める力（主）+ 使いこなす力（総合演習） | 既存プロジェクトで実務タスクを遂行 | 提供プロジェクトで実践 |
| Part 4: 継続的な学習 | 学び続ける力 | 情報収集・新機能検証・実務適用の習慣化 | なし |

### Part 2〜3 の学習アプローチ

Part 2 と Part 3 は、この教材の中核です。

**Part 2** では、テック記事プラットフォーム（Zenn や Qiita のようなサービス）を Claude Code と協働してゼロから構築します。11 の Chapter を通じて Claude Code の主要機能を一つずつ習得しながら、アプリケーションを段階的に成長させていきます。序盤は具体的な指示（テーブル設計やカラム型を詳細に伝える）から始め、中盤ではビジネス要件レベルの指示（「認証機能を追加して」）へ、終盤では複数機能をまとめて指示するレベルへと進みます。技術レベルは上げず、指示の抽象度と1回の指示で生成されるコード量で成長を促します。

Part 2 では「見極める力」の型を導入します。コードが生成されるたびに、正しさ・品質・安全性の3つの観点でチェックする習慣を身につけます。

**Part 3** では、既存プロジェクト（ProConnect）を使い、Part 2 で学んだ機能を総合的に活用しながら実務タスク（バグ修正・機能開発・リファクタリング）を実践します。ここでは「見極める力」を実践的に鍛えます。既存のコードベースを読み解き、Claude Code の出力が既存の設計方針に沿っているかを自分で判断する力を養います。

### Part 4 の学習アプローチ

**Part 4** では、ツール・技術の継続的なキャッチアップの習慣を身につけます。公式ドキュメントの読み方、変更履歴の追い方、新機能の検証方法、実務への適用判断の基準を学びます。AI コーディングツールは急速に進化するため、学び続ける力は長期的なキャリアにおいて最も重要な能力の一つです。

## Claude Code 機能の学習マップ

Claude Code の各機能をどの Chapter で学ぶかを示します。

### Part 2 で習得する主要機能

Part 2 では 11 の Chapter を通じて、以下のような Claude Code の主要機能を習得します。

| 主要機能 | Chapter | 概要 |
|---|---|---|
| CLAUDE.md・プロンプト設計 | 2-2, 2-4 | プロジェクト固有の指示と効果的な指示の書き方 |
| コンテキスト・セッション管理 | 2-3 | 会話の管理、作業の中断・再開 |
| テスト・Hooks | 2-6 | テストの生成と実行、自動処理 |
| Git・Worktree | 2-7 | バージョン管理と並列作業 |
| MCP・Sub-agents | 2-8 | 外部ツール連携と並列タスク |
| 権限・モデル・コスト管理 | 2-9, 2-10 | セキュリティ設定、モデルの使い分け |

このほかにも、インストール・認証（2-1）、Plan Mode（2-4）、Skills（2-5）、Plugins（2-8）など、各 Chapter で実際に手を動かしながら学んでいきます。今の時点ですべてを覚える必要はありません。

### Part 3 で追加する機能

Part 3 では新規に導入する機能は GitHub Actions（CI/CD での Claude Code 活用）のみです。他はすべて Part 2 で習得済みの機能を、実務タスクの文脈で実践します。

### 見極める力の扱い

見極める力（正しさ・品質・安全性の3観点）は、Part 2 と Part 3 で段階的に鍛えます。

| | Part 2（型の導入） | Part 3（実践） |
|---|---|---|
| **位置づけ** | コード生成のたびにチェックする習慣を作る | 実務タスクの中で自律的に判断する |
| **チェック方法** | 教材が提示する見極めチェックリストに沿って確認 | 自分でチェック観点を考えて確認 |
| **重点** | Chapter ごとに1つの観点を重点的に練習 | タスクの性質に応じて3観点を使い分ける |

> Part 3・4 は現在設計中のため、上記は確定している情報です。詳細は Part 3・4 の冒頭で改めて説明します。

## 学習の進め方

### 推奨順序と依存関係

基本的には **Part 1 → Part 2 → Part 3 → Part 4** の順に進めることを推奨します。

Part 2 の Chapter は線形に依存しています。ハンズオンのアプリケーション状態が引き継がれるため、2-1 から 2-10 まで順番に進めてください。ただし、Chapter 2-11（応用: 知らない技術に Claude Code で挑む）はスキップ可能です。

Part 3 は Part 2 全体の完了が前提ですが、Part 3 内の Chapter 間は非線形の依存関係を持ちます（Chapter 3-1 のコードリーディングが前提で、3-2〜3-4 は独立して進められます）。

Part 4 は他の Part への依存がなく、いつでも参照できます。

### 経験に応じた選択学習

皆さんの経験に応じて、学習の進め方をアレンジできます。

- **AI コーディングツールが初めての方**: Part 1 から順に進めてください。Part 2 の序盤で基本操作を丁寧に解説しています
- **ChatGPT 等の AI ツールを使ったことがある方**: Part 1 を読んだ後、Part 2 から実践に入ってください。AI との対話経験があれば、Claude Code の操作にもスムーズに馴染めます
- **他の AI コーディングツール（GitHub Copilot、Cursor 等）の経験がある方**: Part 1 で教材の全体像を把握した後、Part 2 を通じて Claude Code 固有の機能を重点的に学んでください

いずれの場合も、Part 2 のハンズオンで実際に手を動かすことが最も重要です。読むだけでは身につきません。

💡 ハンズオンの題材（テック記事プラットフォーム）はモデルケースです。別のアプリを作りたい場合はそれでも構いません。Claude Code は同じプロンプトでも生成されるコードが毎回異なるため、「この通りに作れば同じ結果になる」という前提を置いていません。大切なのは各 Chapter で紹介する Claude Code の機能を実際に使い、見極めチェックで検証することです。

## まとめ

- 教材は4つの Part で構成。Part 2（基礎）と Part 3（実践）が中核
- Part 2 ではテック記事プラットフォームを構築しながら Claude Code の主要機能を習得し、見極める力の型を導入する
- Part 3 では既存プロジェクトで実務タスクを実践し、見極める力を鍛える
- 推奨順序は Part 1 → 2 → 3 → 4。Part 2 は線形依存のため順番に進める
- ハンズオンの題材は柔軟に変更可能。各 Chapter の機能を使い、見極めチェックで検証することが重要

次の Chapter では、Claude Code を使い始める前に確認すべき事項とプランの選択について説明します。
