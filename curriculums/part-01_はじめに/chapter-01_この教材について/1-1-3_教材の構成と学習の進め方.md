# 1-1-3 教材の構成と学習の進め方

## 🎯 このセクションで学ぶこと

- 教材全体の構成（Part 1〜4）と各 Part の役割を理解する
- 自分の状況に合った学習ペースを選択できる
- 教材の記法（絵文字、コードブロック、注記）の読み方を知る

---

## カリキュラムの全体像

ここまでで、Claude Code を学ぶ意義と、この教材が想定する読者像を確認しました。このセクションでは、教材の全体像を俯瞰し、学習計画を立てるための情報をお伝えします。

この教材は **4つの Part** で構成されています。Part の中に Chapter があり、Chapter の中に Section がある3層構造です。

```
教材全体
├── Part 1: はじめに          ← いまここ
├── Part 2: Claude Code の基礎
├── Part 3: Claude Code の実践
└── Part 4: 継続的な学習
```

---

## 🧠 先輩エンジニアの思考プロセス

技術を学ぶとき、「とりあえず全部読もう」とするのと、「全体像を掴んでから詳細に入ろう」とするのでは、効率がまったく違います。自分は最初、ドキュメントを頭から順番に読んでいましたが、途中で「今読んでいる部分が全体のどこに位置するのか」がわからなくなって挫折することが多かった。先に目次を読んで「ここからここまでが基礎で、ここから応用」とわかっていると、安心感が違います。

この教材も同じです。まず全体像を把握してから、各 Part に入っていきましょう。

## 各 Part の位置付け

### Part 1: はじめに（現在の Part）

| 対応する能力 | ハンズオン |
|---|---|
| なし（導入） | なし |

Claude Code を学ぶ意義、教材の構成、学習を始める前の準備事項を扱います。コードは書きません。この Part を読み終えれば、Part 2 に進む準備が整います。

### Part 2: Claude Code の基礎（3 Chapter）

| 対応する能力 | ハンズオン |
|---|---|
| 使いこなす力 + 見極める力（型の導入） | あり |

Claude Code のセットアップから始まり、基本的な仕組みを理解し、主要機能を実践するパートです。3つの Chapter で段階的に学びます。

- **Chapter 2-1**: セットアップ。インストールから初回起動まで、自分のプロジェクトで Claude Code を動かします
- **Chapter 2-2**: 基本理解。Claude Code がどのように動くのか、仕組みと設計思想を学びます（座学中心）
- **Chapter 2-3**: 機能実践。スターターキット（練習用プロジェクト）を使い、主要機能を一つずつ体験します

> 💡 Part 2 では「見極める力」の **型** を導入します。生成されたコードを「正しさ・品質・安全性」の3観点でチェックする習慣を、最初のハンズオンから身につけていきます。

### Part 3: Claude Code の実践

| 対応する能力 | ハンズオン |
|---|---|
| 見極める力 + 使いこなす力（総合演習） | あり |

実務に近い既存プロジェクト上で、バグ修正・機能開発・リファクタリングといった実務タスクを Claude Code と協働して遂行するパートです。Part 2 で学んだ個別の機能を組み合わせ、実際の開発フローの中で使いこなします。

Part 2 が「個別の技を覚える」段階だとすれば、Part 3 は「試合で使う」段階です。

### Part 4: 継続的な学習

| 対応する能力 | ハンズオン |
|---|---|
| 学び続ける力 | なし |

Claude Code や AI コーディングツール全般のアップデートを追い、自分のワークフローに取り込み続けるための方法を学ぶパートです。公式ドキュメントの読み方、新機能の検証方法、情報収集の習慣化について扱います。

## 学習ペースの目安

この教材に「このペースで進めてください」という決まりはありません。あなたの状況（実務と並行しているか、学習に集中できるか）に合わせて進めてください。

ただし、目安として以下のようなペースを想定しています。

| パターン | 1日の学習時間 | 全体の目安 |
|---|---|---|
| じっくり型 | 1〜2時間/日 | 2〜3週間 |
| 集中型 | 3〜4時間/日 | 1〜2週間 |

### おすすめの進め方

**Part 1 → Part 2 は順番通りに進めてください。** Part 2 のセットアップを済ませないと、以降のハンズオンが進められません。

**Part 2 の Chapter 内も基本的に順番通りです。** Chapter 2-1（セットアップ）→ Chapter 2-2（基本理解）→ Chapter 2-3（機能実践）の順で進めてください。各 Chapter が前の Chapter の知識を前提としています。

**Part 3 は Part 2 が完了してから取り組んでください。** Part 2 で学んだ機能と見極めの型を、Part 3 で総合的に活用します。

**Part 4 はいつ読んでも構いません。** ただし、Part 2〜3 を終えた後に読むのがもっとも効果的です。

> 💡 「一度にすべてを覚えよう」としないでください。ハンズオンで手を動かしながら体で覚えるのが、もっとも定着率の高い学び方です。わからない箇所があっても、先に進むうちに「ああ、あのことか」とつながる場面が必ずあります。

## 教材の読み方ガイド

この教材では、いくつかの記法ルールを統一しています。先に知っておくと読み進めやすくなります。

### セクションの種類

各セクションは以下の3種類のいずれかです。Chapter の冒頭に一覧表があるので、事前に確認できます。

| 種類 | 内容 |
|---|---|
| **概念** | 機能の意義・仕組み・使い方を解説。コード生成は伴わない |
| **ハンズオン** | 概念セクションで学んだ機能を実際に手を動かして実践 |
| **混合** | 概念を学びながらすぐに手を動かす。セットアップなど |

### 絵文字の意味

教材内で使われる絵文字には、それぞれ決まった意味があります。

| 絵文字 | 意味 |
|---|---|
| 🎯 | 学習目標（セクション冒頭に必ずあります） |
| ✨ | まとめ（セクション末尾に必ずあります） |
| 💡 | TIP・補足情報・実務で役立つ知見 |
| ⚠️ | 注意事項・よくあるエラー |
| 📝 | ノート・定義・補足説明 |
| 🔑 | 重要概念・キーポイント |
| 🏃 | ハンズオン手順・実践パート |
| 🧠 | 先輩エンジニアの思考プロセス |
| 🔍 | 見極めチェック（生成コードの検証） |
| 📖 | Chapter の進め方 |

### コードブロックの見方

コードブロックの言語指定で種類を区別しています。

**ターミナルで実行するコマンド**（`bash` と表記）:

```bash
claude
```

**Claude Code への入力（プロンプト）**（言語指定なし、行頭に `>` ）:

> ユーザー一覧にページネーションを追加して

**ファイルの内容や生成コード**（適切な言語を指定）:

```php
// app/Http/Controllers/UserController.php
public function index()
{
    $users = User::paginate(15);
    return view('users.index', compact('users'));
}
```

**コマンドの実行結果**（言語指定なし）:

```
 Claude Code v1.0.0
 ? How can I help you today?
```

### 見極めチェックの読み方

ハンズオンや混合セクションでコードを生成した後、🔍 **見極めチェック** というブロックが登場します。これは、生成されたコードを3つの観点で検証するためのチェックリストです。

- **正しさ**: ビジネスロジックや要件を満たしているか
- **品質**: コーディング規約、命名規則、設計パターンに沿っているか
- **安全性**: セキュリティホールや情報漏洩のリスクがないか

Part 2 から繰り返し登場しますので、「こういうフォーマットがあるんだな」と覚えておいてください。

## ✨ まとめ

- 教材は Part 1（導入）→ Part 2（基礎）→ Part 3（実践）→ Part 4（継続学習）の4部構成
- Part 2 で「使いこなす力」と「見極める力の型」、Part 3 で総合演習、Part 4 で「学び続ける力」を養う
- 基本的に Part 1 → 2 → 3 の順で進める。Part 4 はいつ読んでもよいが、Part 3 の後が最適
- 絵文字やコードブロックの記法にはルールがある。迷ったらこのセクションに戻って確認する
