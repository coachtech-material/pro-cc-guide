# 1-1-1 なぜ Claude Code を使うのか

## 🎯 このセクションで学ぶこと

- Claude Code がどのようなツールで、何ができるのかを知る
- この教材で養う3つの能力（使いこなす力・見極める力・学び続ける力）を理解する

## AI がエンジニアの仕事を変えている

COACHTECH の選抜試験を突破し、これから実務に臨もうとしているあなた。まず最初に、率直な話をさせてください。

いま、ソフトウェア開発の世界は大きな転換点にあります。AI コーディングツールの登場によって、一人のエンジニアが出せるアウトプットの量と速度が劇的に上がりました。これまで数時間かかっていた実装が数分で終わる。テストコードの雛形を一から書く必要がなくなる。そんな世界がすでに現実になっています。

この変化は、企業の採用にも影響を与えています。一人あたりの生産性が上がるということは、同じ成果を出すのに必要な人数が減るということです。特にジュニアレベルのエンジニアにとって、「基礎ができます」だけでは差別化が難しくなっています。

でも、悲観する必要はありません。この変化は、裏を返せばチャンスでもあります。AI ツールを使いこなせるジュニアエンジニアは、使いこなせないミドルエンジニアよりも高い生産性を発揮できます。つまり、AI ツールとの協働スキルは、経験年数というハンデを一気に縮める武器になるのです。

この教材は、その武器の一つである **Claude Code** を、あなたが実務で使いこなせるようになることを目指して作られました。COACHTECH ではあなたのような方を「Pro生」と呼んでいます。この教材では、以降「あなた」と呼びかけていきます。

## 🧠 先輩エンジニアの思考プロセス

自分が駆け出しのエンジニアだった頃、先輩に「ドキュメントを読め」「コードを読め」と言われて、ひたすら手を動かしていました。わからないことがあれば Stack Overflow を検索し、エラーメッセージをコピペして解決策を探す。それが当たり前の学び方でした。

いま、自分の隣には Claude Code がいます。「このエラーの原因を調べて」と頼めば、コードベース全体を読んだ上で原因を特定してくれる。「このテーブルに対する CRUD のテストを書いて」と言えば、既存のテストコードの書き方に合わせてテストを生成してくれる。正直、最初は「これ、自分の仕事なくなるんじゃないか」と思いました。

でも実際に使い込んでみると、そうではなかった。Claude Code が出すコードをそのまま使えることもあれば、プロジェクトの事情に合わない部分を自分で修正する必要があることもある。「何を作るか」「これで本当に正しいか」「セキュリティ的に大丈夫か」を判断するのは、結局自分です。つまり、AI は「考える」部分を肩代わりしてくれるのではなく、「作業する」部分を高速化してくれる。自分の判断力と AI の実行力を掛け合わせることで、一人で出せる価値が何倍にもなる。そういう道具なんです。

## Claude Code とは何か

Claude Code は、Anthropic が提供する **エージェント型コーディングツール** です。「エージェント型」というのは、単にコードを補完するだけでなく、あなたの指示を受けてコードベースを読み、ファイルを編集し、コマンドを実行し、外部ツールと連携するところまでを自律的に行える、という意味です。

ChatGPT のようなチャット AI を使ったことがあるなら、それとの違いを考えるとわかりやすいでしょう。チャット AI は「質問に答える」ツールです。コードの書き方を聞けば教えてくれますが、実際にファイルを編集したり、テストを実行したりはしません。一方、Claude Code は **あなたの開発環境の中で直接動作します**。ターミナルで起動し、プロジェクトのコードを読み、ファイルを作成・編集し、Git でコミットまでしてくれます。

### Claude Code でできること

Claude Code の主な機能を整理すると、以下のようになります。

- **コードの読解と生成**: プロジェクト全体のコードを読んだ上で、新しいコードを生成する。既存の設計パターンやコーディング規約を踏まえた生成ができる
- **ファイル操作**: ファイルの作成、編集、削除を直接行う
- **コマンド実行**: テストの実行、リンターの実行、ビルドなど、ターミナルコマンドを実行する
- **Git 操作**: コミットの作成、プルリクエストの作成など、Git ワークフローを支援する
- **外部ツール連携**: MCP（Model Context Protocol）を通じて、データベース、Slack、ブラウザなど外部サービスと接続する
- **カスタマイズ**: CLAUDE.md ファイルでプロジェクト固有のルールを伝えたり、カスタムスキル（独自のコマンド）を定義したりできる

### どこで使えるか

Claude Code はさまざまな環境で利用できます。

- **ターミナル（CLI）**: もっとも基本的で高機能なインターフェース。この教材では主にこちらを使います
- **VS Code / Cursor 拡張機能**: エディタ内で直接 Claude Code を使える。差分のインライン表示やファイル参照が便利
- **JetBrains プラグイン**: IntelliJ、PhpStorm など JetBrains 系 IDE で利用可能
- **デスクトップアプリ**: macOS、Windows 向け。複数セッション管理やスケジュール実行が可能
- **Web（claude.ai/code）**: ブラウザから利用。ローカル環境のセットアップ不要で、長時間のタスク実行に向いている

> 💡 この教材ではターミナル（CLI）を基本として解説します。CLI での操作を理解していれば、他の環境への応用は自然にできるようになります。

### エージェントループという仕組み

Claude Code の動作を理解する上で重要なのが「エージェントループ」という仕組みです。あなたが指示を出すと、Claude Code は以下のサイクルを自律的に繰り返します。

```
あなたの指示
    ↓
┌─────────────────────────┐
│  1. 指示を理解する        │
│  2. 必要な情報を集める     │
│     （コードを読む等）     │
│  3. 計画を立てる          │
│  4. 実行する              │
│     （編集・コマンド等）   │
│  5. 結果を確認する        │
│     ↓                    │
│  まだ完了していなければ    │
│  → 2 に戻る              │
└─────────────────────────┘
    ↓
完了を報告
```

たとえば「ユーザー一覧画面にページネーションを追加して」と指示すると、Claude Code は次のように動きます。

1. プロジェクト内のユーザー一覧に関するコードを検索・読解する
2. 既存のページネーション実装やルーティングを確認する
3. コントローラー、ビュー、必要に応じてテストを編集する
4. 変更内容を確認し、不足があれば追加の編集を行う

このループを理解しておくことで、「Claude Code に何を任せられて、どこで自分が判断すべきか」がクリアになります。今は概要だけ把握すれば十分です。Part 2 で実際に手を動かしながら体感していきます。

## この教材で養う3つの能力

Claude Code を「使える」だけでは、実務で価値を出し続けることはできません。この教材では、以下の3つの能力をバランスよく養うことを目指します。

### 1. 使いこなす力（Input）

Claude Code の機能・設定・制約を把握し、目的に応じて的確に活用する力です。

同じ「バグを直して」という指示でも、プロジェクトのコンテキストを正しく伝えられるかどうかで、Claude Code の出力品質は大きく変わります。CLAUDE.md でプロジェクトのルールを伝える、適切な粒度で指示を出す、必要に応じて権限を設定する。こうした「入力の質」を高めるスキルが、使いこなす力です。

### 2. 見極める力（Output）

生成されたコードを **正しさ・品質・安全性** の3つの観点で検証し、責任を持って採用する力です。

Claude Code が生成するコードは、多くの場合そのまま動きます。しかし、「動く」と「正しい」は同じではありません。ビジネスロジックが要件を満たしているか（正しさ）、プロジェクトのコーディング規約に沿っているか（品質）、セキュリティホールがないか（安全性）。これらを判断できるのは、プロジェクトの文脈を理解しているあなただけです。

> 🔑 **コードの最終責任は常にあなたにあります。** Claude Code はあくまで「提案」をしているのであって、そのコードをプロジェクトに取り込む判断をするのはあなたです。これは、先輩エンジニアにレビューを依頼されたときと同じ責任感で臨むべきものです。

### 3. 学び続ける力（Time）

ツール・技術の進化を追い、自ら試し、実務に適応し続ける力です。

Claude Code は日々進化しています。新しい機能が追加され、モデルの性能が向上し、できることの範囲が広がり続けています。この教材で学んだ内容も、半年後には一部が古くなっているかもしれません。だからこそ、「公式ドキュメントを読む」「新機能を試す」「自分のワークフローに組み込む」という習慣を身につけることが、長期的には最も重要です。

### 3つの能力とカリキュラムの対応

```
Part 2: Claude Code の基礎
  → 使いこなす力 + 見極める力（型の導入）

Part 3: Claude Code の実践
  → 見極める力 + 使いこなす力（総合演習）

Part 4: 継続的な学習
  → 学び続ける力
```

Part 2 で基礎を学び、Part 3 で実務に近い形で実践し、Part 4 で自走できるようになる。この流れで、3つの能力を段階的に養っていきます。

## Claude Code が「相棒」になる理由

最後に、なぜ数ある AI コーディングツールの中で Claude Code なのか、という点について触れておきます。

Claude Code を選ぶ理由は、大きく3つあります。

**1. コードベース全体を理解した上で動作する**

Claude Code はプロジェクトのファイル構造、既存のコード、設定ファイルを読んだ上でコードを生成します。「このプロジェクトでは Eloquent の scope を使っている」「テストは Feature ディレクトリに書いている」といった文脈を踏まえた提案ができるため、生成されたコードがプロジェクトに馴染みやすいのです。

**2. ターミナルからすべてを完結できる**

ファイルの編集、テストの実行、Git 操作まで、ターミナルから離れることなく開発を進められます。「コードを生成して、コピーして、エディタに貼り付けて」という手間がありません。開発のフローが途切れないことは、集中力を維持する上で想像以上に大きな価値があります。

**3. カスタマイズ性が高い**

CLAUDE.md ファイルでプロジェクト固有のルールを伝えたり、カスタムスキルで繰り返し行う作業を自動化したり、MCP で外部ツールと連携したりと、自分の開発スタイルに合わせた環境を構築できます。使い込むほどに、あなた専用の開発パートナーに育てていける。これが「相棒」と呼ぶ理由です。
