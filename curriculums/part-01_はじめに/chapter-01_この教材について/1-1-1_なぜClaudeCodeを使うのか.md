# 1-1-1 なぜ Claude Code を使うのか

## 概要

ターミナルに一言伝えるだけで、Controller、Blade テンプレート、マイグレーション、ルーティングが一気に生成される。皆さんが手作業で30分かけていた実装が、数十秒で形になる。Claude Code を使うと、そんな体験が日常になります。

この教材は、皆さんが Claude Code を使いこなし、実務開発の生産性と品質を高めるための教材です。このセクションでは、なぜ今 Claude Code を学ぶ必要があるのか、この教材が目指すこと、そして皆さんが身につける3つの能力について説明します。さらに、Claude Code がどのようなツールなのか、その特徴と強みを紹介します。

## AI 時代のジュニアエンジニアの現在地

### AI がもたらした変化

AI コーディングツールは、もはや一部の先進的なエンジニアだけのものではありません。GitHub の調査によると、開発者の92%が業務内外で AI コーディングツールを使用しています。コードの自動生成、バグの自動検出、テストの自動作成といった作業が AI で可能になり、企業がジュニアエンジニアに求める水準は確実に上がっています。

これは脅威ではなく、チャンスです。AI を使いこなせるジュニアエンジニアは、従来ミドル層が担っていた領域の一部をカバーできます。つまり、AI の台頭はジュニアエンジニアの価値を下げるのではなく、AI を活用できるエンジニアの価値を高めているのです。

ここで重要なのは、AI に的確な指示を出すには、具体的な知識を構造化し適切な抽象度で言語化する必要があるということです。「ユーザー認証を作って」という漠然とした指示と、「Laravel Fortify を使って、admin と author の2ロールでアクセス制御を実装して」という具体的な指示では、生成されるコードの品質が大きく異なります。皆さんが COACHTECH で身につけた Laravel、データベース設計、Git などの知識は、AI を使いこなす土台になります。

### AI の出力には検証が必要

AI が生成するコードは便利ですが、そのまま信頼してはいけません。AI は一見正しく見えるコードを生成しますが、さまざまな問題を含むことがあります。たとえば、認可ロジックに漏れがあり特定のユーザーが本来アクセスできないデータを閲覧できてしまう、テストは通るが境界条件（0件・上限値・空文字列など）を見落としている、既存プロジェクトの設計パターンと異なる実装をしている、といったケースです。

だからこそ、AI の出力を検証する力が必要です。要件が正しく実装に落とし込まれているか、セキュリティに問題はないか、既存の設計方針に沿っているか。これらを判断するのは、まさに皆さんがこれまで学んできたプログラミングの知識そのものです。

## この教材のコンセプト

### 皆さんの知識が AI を使いこなす土台になる

AI に的確な指示を出すには、何を作りたいかを具体的に言語化する必要があります。「Eloquent で多対多リレーションを設定して」と指示するには、リレーションの概念を理解していなければなりません。AI の出力を検証するには、生成されたマイグレーションのカラム型が適切か、Controller のクエリが効率的か、Policy のアクセス制御に漏れがないかを判断する必要があります。

つまり、AI に指示を出す力（Input）と、出力を検証する力（Output）の両方に、プログラミングの知識が不可欠です。皆さんが COACHTECH で身につけた知識は、AI を使いこなす土台そのものです。この教材では、その知識を活かして Claude Code と協働する力を養います。

### 実務タスクを高速かつ高品質に遂行する

この教材のゴールは、Claude Code と協働して実務タスクを高速かつ高品質に遂行できるようになることです。具体的には、以下の3つの実務タスクを対象とします。

- **バグ修正**: 既存コードの不具合を特定し、修正する
- **機能開発**: 新規機能の実装や既存機能の改修を行う
- **リファクタリング**: コードの品質・可読性・パフォーマンスを改善する

いずれのタスクも、既存プロジェクトのコーディング規約と設計方針に従いながら遂行します。コードリーディングが前提であり、生成されたコードを理解せずに採用することはありません。

## この教材で養う3つの能力

この教材では、実務タスクを支える3つの能力を養います。いずれも個人での開発に限らず、チーム開発の文脈でも求められる能力です。

### 使いこなす力（Input）

Claude Code の設定・機能・制約を把握し、目的に応じた機能選択とプロンプト設計で的確に活用する力です。

Claude Code には多くの機能があります。インタラクティブモードでの対話、Plan Mode での計画立案、Skills によるプロンプトの再利用、MCP によるツール統合など、場面に応じて適切な機能を選択する必要があります。この教材の Part 2 で、新規プロジェクトを構築しながら主要機能を一つずつ習得します。

### 見極める力（Output）

Claude Code の出力を検証・判断し、責任を持って採用する力です。AI の出力をそのまま受け入れず、自分で理解・判断する習慣を保ちます。以下の3つの観点で確認します。

- **正しさ**: 要件通りに動くか
- **品質**: 保守性・パフォーマンス・既存設計との一貫性は適切か
- **安全性**: 脆弱性がないか。認可・入力検証・機密情報の漏洩がないか

この3つの観点は、AI 生成コードに対して常に意識すべきものです。Part 2 で型を導入し、Part 3 で実践的に鍛えます。

### 学び続ける力（Time）

ツール・技術の進化を追い、自ら試し、実務に適応し続ける力です。

AI コーディングツールは急速に進化しています。新機能の追加、既存機能の変更、新しいツールの登場が頻繁に起こります。公式ドキュメント・SNS・コミュニティから最新動向をキャッチアップし、新機能を検証し、開発フローへの適用を判断する習慣が必要です。Part 4 でこの習慣の身につけ方を学びます。

## Claude Code の特徴

### 補完型からエージェント型へ

AI コーディングツールは進化を続けており、大きく2つのアプローチに分類できます。

**補完型**は、エディタ上でコードを書いている最中に次の行や関数の候補を提案するアプローチです。GitHub Copilot がその代表例で、コードの「続き」を予測して提案します。開発者が主導し、AI がアシストする形です。

**エージェント型**は、開発者の指示に基づいて AI が自律的にタスクを遂行するアプローチです。コードベース全体を読み取り、複数ファイルを横断して編集し、コマンドの実行やツールとの統合まで行います。開発者がゴールを伝え、AI が計画を立てて実行する形です。Cursor のようにエディタに AI を深く統合したツールも登場し、AI コーディングツールの形態は多様化しています。

Claude Code はターミナルベースのエージェント型 AI コーディングツールです。ファイルの読み書き、ターミナルコマンドの実行、Git 操作、外部ツールとの連携を自律的に行います。「記事一覧に検索機能を追加して」と伝えれば、Claude Code はコードベースを読み取り、必要なファイルを特定し、Controller の修正、Blade テンプレートの追加、Route の設定までを一連の流れで実行します。

### Claude Code の強み

Claude Code には以下の特徴があります。

**コードベースの理解**: プロジェクト全体を読み取り、ファイル間の依存関係やアーキテクチャを把握します。既存の設計パターンに沿ったコードを生成できるため、プロジェクトに馴染むコードが得られます。

**ファイル編集とコマンド実行**: 複数ファイルの作成・編集を行い、マイグレーションの実行やテストの実行といったコマンドも実行します。「マイグレーションを作成して実行して」と伝えれば、ファイルの生成からコマンドの実行まで一貫して行います。

**ツール統合（MCP）**: Model Context Protocol（MCP）というオープンスタンダードを通じて、データベース、Slack、ブラウザなどの外部ツールと連携できます。

**利用環境の多様性**: ターミナル（CLI）、VS Code 拡張、JetBrains プラグイン、デスクトップアプリ、Web ブラウザ、さらには iOS アプリからも利用できます。この教材では主にターミナルと VS Code を使いますが、自分のワークフローに合った環境を選べます。

**カスタマイズ性**: CLAUDE.md によるプロジェクト固有の指示、Skills による再利用可能なワークフロー、Hooks による自動化など、チームや個人の開発スタイルに合わせてカスタマイズできます。
