# 3-2-0 Chapter 2 の地図

> 既存プロジェクトに参画したとき、最初に求められるのは「コードを読んで理解すること」です。この Chapter では、コードを効率的に読むための認知科学に基づく方法論を学び、Claude Code を活用して CourseHub のコードを実際に読み解きます。

## セクション一覧

### 3-2-1 コードリーディングの方法論 ｜ 📖 読んで学ぶ

- 認知負荷の2つの源泉（プログラミング知識・ビジネスドメイン知識）
- 認知負荷を下げる5ステップ（探索 → 検索 → 転写 → 理解 → 増強）
- Claude Code の説明を検証するサイクル（聞く → 読む → 言い換える → 実装で検証）

### 3-2-2 知らない技術への向き合い方 ｜ 📖 読んで学ぶ

- 4ステップで使い分ける理解の深さ（理解 → 評価 → 判断 → 説明できる）
- 「説明できないコードは受け入れない」の原則と AI 時代の学び方の落とし穴
- Laravel 実務でよく出会うパターンの地図（Form Request・Policy・Service・Event/Listener・Observer・Jobs）

### 3-2-3 CourseHub のコードを読む ｜ 📖🏃 読んで手を動かす

- Sub-agents での並列探索と MCP での DB 直接確認
- Course → Chapter → Lesson → Quiz の階層構造とロールベース認可の把握
- 受講登録フロー（`EnrollmentController` → `EnrollmentService` → `Enrollment` / `LessonProgress`）の追跡

## 📖 この Chapter の進め方

まず概念セクション（[3-2-1 コードリーディングの方法論](3-2-1_コードリーディングの方法論.md)）で、なぜコードが読めないのかを認知科学の視点から理解し、認知負荷を下げる5ステップの方法論を学びます。次の概念セクション（[3-2-2 知らない技術への向き合い方](3-2-2_知らない技術への向き合い方.md)）で、知らない技術に出会ったときの向き合い方を学びます。最後に混合セクション（[3-2-3 CourseHubのコードを読む](3-2-3_CourseHubのコードを読む.md)）で、CourseHub のコードを5ステップに沿って実際に読み、Sub-agents や MCP を使って大量のコードを効率的に探索し、Claude の説明を自分の目で検証する体験をします。
