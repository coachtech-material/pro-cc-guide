# 3-3-0 Chapter 3 の地図

> バグ報告を受けたとき、何から手をつけ、どう進め、どう検証するか。この Chapter では、Claude Code と協働してバグを修正するための方法論を学び、CourseHub で実践します。

## セクション一覧

### 3-3-1 バグ修正の方法論 ｜ 📖 読んで学ぶ

- バグ修正の5フェーズ（報告 → 再現 → 調査 → 修正 → 検証）
- Claude Code への調査指示の組み立て方（症状・再現手順・期待値）
- バグの3類型（データの正しさ・アクセス制御・機能不全）と類型別の調査アプローチ
- `/rewind` チェックポイントと段階的テスト実行による安全な修正サイクル

### 3-3-2 CourseHub のバグを修正する ｜ 📖🏃 読んで手を動かす

- 進捗率バグ: `getProgressRate()` のクエリ修正による分子分母の条件統一
- 認可バグ: `CoursePolicy@view` の無条件 `return true` の修正と網羅的テスト
- 500 エラーバグ: 未回答時の null アクセスへのバリデーション追加と例外ハンドリング

## 📖 この Chapter の進め方

まず 3-3-1 でバグ修正の方法論を学びます。バグの報告から修正完了までの全体フローを理解し、バグの種類に応じた調査アプローチの違いを押さえてください。次の 3-3-2 では、CourseHub に仕込まれた3つのバグを実際に修正します。3-3-1 で学んだフローを繰り返し実践することで、バグ修正のプロセスを身につけます。
