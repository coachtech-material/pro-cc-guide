# 3-1-0 Chapter 1 の地図

> Part 3 では、提供プロジェクト「CourseHub」を題材に、実務で発生するタスク（バグ修正・機能開発・リファクタリング）を Claude Code と協働して遂行します。Part 2 で個別に学んだ機能を組み合わせ、タスクの理解からコードレビュー、PR 作成まで一気通貫で実践します。

## セクション一覧

### 3-1-1 Claude Code で実務タスクを遂行する考え方 ｜ 📖 読んで学ぶ

- AI が担う 70% と人間が判断する 30% の役割分担
- AI 生成コードの検証が必要な3つの理由（脆弱性・設計判断・説明責任）
- Claude Code との協働がコードレビュープロセスである、という捉え方
- CourseHub プロジェクトの概要（Laravel 10 / Sail / MySQL）と admin / coach / student のロール構成

### 3-1-2 セットアップと動作確認 ｜ 📖🏃 読んで手を動かす

- Git クローンから Sail 起動までの一連の手順
- `.env` のサービス名設定（`DB_HOST=mysql` など）と Composer インストール、マイグレーション・シーディング
- README とプロジェクトの実態にズレがあった場面で Claude Code を使ったトラブルシュート

### 3-1-3 プロジェクトの規約・設定を確認する ｜ 📖🏃 読んで手を動かす

- README 規約と実コードの乖離（Fat Controller・Form Request 未使用など）の特定
- CLAUDE.md の Sail コマンド統一・PHP 8.2 / MySQL 8.0 明記・コース構造説明の追加
- `.claude/settings.json` への権限追加と既存 Skill（test・lint）の修正

## 📖 この Chapter の進め方

この Chapter はまず概念セクション（3-1-1 Claude Codeで実務タスクを遂行する考え方）で Part 3 全体の考え方を理解し、その後2つの混合セクション（3-1-2 セットアップと動作確認, 3-1-3 プロジェクトの規約・設定を確認する）で提供プロジェクトの環境を整えます。3-1-1 はハンズオンなしの座学です。3-1-2 と 3-1-3 では実際に手を動かします。
