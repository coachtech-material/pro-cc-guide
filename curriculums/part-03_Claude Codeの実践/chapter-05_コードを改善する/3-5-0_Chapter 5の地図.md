# 3-5-0 Chapter 5 の地図

> 動いているコードを「壊さずに良くする」。この Chapter では、リファクタリングの方法論を学び、CourseHub のコードを Claude Code と協働して改善します。

## セクション一覧

### 3-5-1 リファクタリングの方法論 ｜ 📖 読んで学ぶ

- リファクタリングの4フェーズ（対象特定 → テスト確認 → 改善 → テスト再実行）
- 代表的なコードスメル（Fat Controller・N+1 クエリ・重複コード・不統一パターン）と優先順位の付け方
- テストなしリファクタリング禁止の原則とベースライン化
- Claude Code の改善提案を判断する3基準（規約適合性・既存パターンとの一貫性・必要十分性）

### 3-5-2 CourseHub のコードを改善する ｜ 📖🏃 読んで手を動かす

- Fat Controller の責務分離: `CoachCourseController@store` を `StoreCourseRequest` 抽出 + private メソッド分割
- N+1 クエリの解消: `with()` / `withCount()` の Eager Loading と Worktree での並列改善
- Service クラス導入提案を見送る判断（現問題の必要十分性を超える場合）

## 📖 この Chapter の進め方

まず [3-5-1 リファクタリングの方法論](3-5-1_リファクタリングの方法論.md) でリファクタリングの方法論を学びます。「動作を変えずにコードを良くする」ためのフローと、Claude Code の改善提案を判断する基準を理解してください。次の [3-5-2 CourseHubのコードを改善する](3-5-2_CourseHubのコードを改善する.md) では、CourseHub のコードを実際に改善します。1つ目は Fat Controller の責務分離（コード構造の改善）、2つ目は N+1 クエリの解消（パフォーマンスの改善）です。改善の対象が違えばアプローチも変わることを体験します。
