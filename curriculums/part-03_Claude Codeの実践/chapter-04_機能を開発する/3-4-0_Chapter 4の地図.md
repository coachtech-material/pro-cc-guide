# 3-4-0 Chapter 4 の地図

> 機能開発は、バグ修正とは異なる判断力が求められるタスクです。この Chapter では、Claude Code と協働して既存機能の拡張と新規機能の構築を進める方法論を学び、CourseHub で実践します。

## セクション一覧

### 3-4-1 機能開発の方法論 ｜ 📖 読んで学ぶ

- バグ修正との違い（正解の有無・設計判断の有無）と4フェーズ（要件理解 → 設計 → 実装 → 検証）
- 着手前に確認すべき3点（類似機能のコード・コーディング規約の実態・テストパターン）
- 要件分解の3基準（独立して動作確認できる単位・依存関係の順序・既存変更の最小化）
- Claude Code の設計提案を判断する基準（規約適合性・既存パターンとの一貫性・必要十分性）

### 3-4-2 CourseHub に機能を追加する ｜ 📖🏃 読んで手を動かす

- 小テスト再受験（既存機能の拡張）: `SubmissionPolicy` への合格チェック追加と `result.blade.php` での再受験ボタン制御
- コースレビュー（新規機能の構築）: Review モデル・`StoreReviewRequest`・`ReviewPolicy`・`ReviewController` を作成
- 既存パターン（`EnrollmentService`・`SubmissionPolicy`）に合わせた段階的実装とテスト

## 📖 この Chapter の進め方

まず [3-4-1 機能開発の方法論](3-4-1_機能開発の方法論.md) で機能開発の方法論を学びます。バグ修正との違いを押さえ、要件理解から検証までの全体フローと、Claude Code の設計提案を判断する基準を理解してください。次の [3-4-2 CourseHubに機能を追加する](3-4-2_CourseHubに機能を追加する.md) では、CourseHub で2つの機能を実際に開発します。1つ目は既存機能の拡張（小テスト再受験）、2つ目は新規機能の構築（コースレビュー）です。スコープの違いによってアプローチがどう変わるかを体験します。
