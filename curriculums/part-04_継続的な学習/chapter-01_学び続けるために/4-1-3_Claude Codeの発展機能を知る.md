# 4-1-3 Claude Code の発展機能を知る

## 🎯 このセクションで学ぶこと

- 教材で深く扱わなかった Claude Code の発展機能の全体像を把握する
- 各機能が「どんな場面で使えるか」を理解し、次に試す候補を見つける
- 4-1-1 で学んだ学び続けるサイクルを、新しい機能の調査に応用する

教材で扱ってきた機能は Claude Code の一部です。このセクションでは、まだ触れていない機能を「何ができるか」「どんな場面で使うか」の2つの視点で紹介します。すべてを今すぐ使う必要はありません。自分の開発スタイルに合いそうなものを見つけ、4-1-1 のサイクル（情報収集→検証→適用判断）で試してみてください。

> 📝 以下の機能には `Experimental`（実験的）や `Research Preview`（研究プレビュー）のラベルがついているものがあります。仕様が変わる可能性があるため、導入前に公式ドキュメントで最新の状態を確認してください。

## 導入: まだ知らない機能がある

### 🧠 先輩エンジニアはこう考える

> Claude Code の機能一覧を初めて全部見たとき、「こんなにあるのか」と驚いた記憶がある。でも全部を使いこなす必要はない。自分の場合、普段の開発で使っている機能は全体の半分もない。大事なのは「こういうことができる」と知っておくこと。いざ必要になったときに「そういえばあの機能が使えそうだ」と思い出せれば十分。このセクションはそのためのカタログだと思って読んでほしい。

## ローカル環境を拡張する機能

まずは、普段の開発環境をさらに便利にする機能を紹介します。

### Computer Use（コンピュータ操作）

Claude Code が Mac 上のアプリケーションを直接操作する機能です。ブラウザ、スプレッドシート、その他の GUI アプリケーションを Claude Code が見て、クリックし、入力できます。

**どんな場面で使うか**

- Web アプリケーションの動作確認を Claude Code に任せたいとき
- Figma のデザインと実装を比較して差分を見つけたいとき
- スプレッドシートのデータを読み取って処理に使いたいとき

**使い方の概要**

Desktop アプリのセッション開始時に環境セレクターから Computer Use を有効化します。macOS では、アクセシビリティと画面収録の権限を許可する必要があります。

> ⚠️ Computer Use を有効にすると、Claude Code がマウスやキーボードを操作します。機密情報が表示されている画面で使用する際は注意してください。

📖 公式ドキュメント: [Desktop](https://code.claude.com/docs/en/desktop)

### Voice Dictation（音声入力）

スペースキーを長押しして音声でプロンプトを入力する機能です。キーボードで入力しにくい長い指示や、手がふさがっているときに便利です。

**どんな場面で使うか**

- 長い要件や背景情報を伝えたいとき
- コードを読みながら、気づいた点を口頭でメモしたいとき
- 日本語の文章をタイピングするより口頭で伝えた方が速いとき

**使い方の概要**

`/voice` コマンドで有効化します。スペースキーを長押しすると録音が始まり、離すとテキストに変換されます。日本語を含む 20 言語に対応しています。Push-to-Talk キーは `~/.claude/keybindings.json` で変更できます。

> 💡 音声認識はコーディング用語（OAuth、JSON、localhost など）に最適化されています。プロジェクト名や Git ブランチ名も認識ヒントとして自動的に使用されます。

📖 公式ドキュメント: [Voice Dictation](https://code.claude.com/docs/en/voice-dictation)

### Chrome 連携（ブラウザデバッグ）

Claude in Chrome 拡張機能を使って、Chrome ブラウザと Claude Code を接続する機能です。Claude Code が Chrome のタブを操作し、コンソールログを読み取り、ネットワークリクエストを確認できます。

**どんな場面で使うか**

- ローカルで動作する Web アプリのフロントエンドをデバッグしたいとき
- ブラウザのコンソールエラーを見ながらコードを修正したいとき
- フォームの入力テストを自動化したいとき
- 操作手順を GIF で録画して PR やドキュメントに添付したいとき

**使い方の概要**

Chrome ウェブストアから「Claude in Chrome」拡張機能をインストールし、`claude --chrome` または `/chrome` で接続します。Claude Code はあなたのログイン状態を共有するため、認証が必要な Web アプリケーションも操作できます。

> 📝 この機能は beta です。Google Chrome または Microsoft Edge が必要で、Brave や Arc には対応していません。拡張機能 v1.0.36 以上、Claude Code v2.0.73 以上が必要です（2026年3月時点）。

📖 公式ドキュメント: [Chrome](https://code.claude.com/docs/en/chrome)

### Desktop アプリ

macOS と Windows で利用できるスタンドアロンの GUI アプリケーションです。ターミナルの Claude Code と同等の機能に加え、ビジュアル diff のレビュー、開発サーバーのライブプレビュー、Computer Use、並列セッションなどの機能を提供します。

**どんな場面で使うか**

- コードの変更をビジュアル diff で確認してから適用したいとき
- 複数のタスクを並列で実行したいとき（Worktree を自動作成して分離）
- GitHub の PR を監視し、CI 失敗時に自動修正させたいとき
- スケジュール実行（cron のように定期的にタスクを実行）を設定したいとき

**使い方の概要**

[claude.ai/download](https://claude.ai/download) からインストールします。ターミナルの Claude Code と同じアカウントでログインすると、セッションを行き来することもできます。

### Dispatch（モバイルからタスク送信）

Claude のモバイルアプリから Desktop アプリにタスクを送信する機能です。Desktop アプリが自律的にタスクを実行し、完了を通知します。

**どんな場面で使うか**

- 通勤中やミーティング中に思いついたタスクを、すぐに実行開始させたいとき
- 「帰宅したら結果を確認する」というワークフローを組みたいとき

**使い方の概要**

Desktop アプリの Dispatch 機能を有効にしたうえで、Claude のモバイルアプリからタスクを送信します。

📖 公式ドキュメント: [Desktop](https://code.claude.com/docs/en/desktop)

## リモートから使う機能

ローカルのターミナルを離れても Claude Code を使える機能群です。

### Claude Code on the Web（ブラウザ版）

ローカル環境を一切必要とせず、ブラウザから Claude Code を実行する機能です。Anthropic が管理するクラウド VM 上でタスクを実行し、完了したら差分を確認して PR を作成できます。iOS・Android アプリからも利用可能です。

**どんな場面で使うか**

- 外出先や別のマシンから作業を進めたいとき
- ローカル環境のセットアップなしに PR を作成したいとき
- 長時間かかるタスクをクラウドに任せて、ローカルでは別の作業をしたいとき

**使い方の概要**

[claude.ai/code](https://claude.ai/code) にアクセスし、GitHub アカウントを接続してリポジトリを選択します。ターミナルから `claude --remote` で Web セッションを開始することもできます。逆に、Web で作業したセッションを `/teleport` でローカルに引き取ることもできます。

> 📝 クラウド VM には Python、Node.js、PHP、Go、Rust など主要な言語のランタイムがプリインストールされています。Laravel プロジェクトの場合、Setup Script で `composer install` や Sail の起動を設定できます。

📖 公式ドキュメント: [Claude Code on the Web](https://code.claude.com/docs/en/claude-code-on-the-web)

### Remote Control（リモート操作）

ローカルで実行中の Claude Code セッションを、claude.ai やモバイルアプリから遠隔操作する機能です。実行はすべてローカルマシンで行われます。

**どんな場面で使うか**

- 長時間のタスクを自宅のマシンで実行しながら、外出先からモバイルで進捗を確認・操作したいとき
- チームメンバーに自分のセッションを共有して、一緒に作業したいとき

**使い方の概要**

`claude remote-control` でサーバーモードを起動するか、既存のセッションから `/remote-control` で有効化します。QR コードが表示されるので、モバイルで読み取って接続できます。`--spawn same-dir`（同じディレクトリ）または `--spawn worktree`（Git Worktree で分離）オプションで、複数のセッションを同時に受け付けることもできます。

> 💡 通信はすべて HTTPS で暗号化され、受信ポートは開きません。セキュリティ面での心配は不要です。

📖 公式ドキュメント: [Remote Control](https://code.claude.com/docs/en/remote-control)

## チーム・自動化の機能

個人の開発を超えて、チームやシステムと連携する機能群です。

### Agent Teams（チームエージェント）

複数の Claude Code インスタンスがチームとして協調動作する機能です。リーダーセッションがタスクを分配し、チームメイトセッションが並行して作業を進めます。

**どんな場面で使うか**

- 大規模なリファクタリングで、フロントエンド・バックエンド・テストを並行して進めたいとき
- 複数の仮説を同時に検証したいとき（バグの原因調査など）
- 新しいモジュールを複数のファイルにまたがって一度に実装したいとき

**使い方の概要**

環境変数 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` を有効にして利用します。チームリーダーがタスクリストを作成し、チームメイトに割り当てます。各チームメイトは独自のコンテキストウィンドウを持ち、メールボックスを通じて互いにメッセージを送受信できます。

> 📝 この機能は Experimental（実験的）です。推奨サイズは 3〜5 のチームメイト、各チームメイトに 5〜6 タスクです。Sub-agents との違いは、チームメイト同士が直接コミュニケーションできる点です。

📖 公式ドキュメント: [Agent Teams](https://code.claude.com/docs/en/agent-teams)

### Channels（外部通知の受信）

Telegram、Discord、iMessage などの外部サービスから、実行中の Claude Code セッションにイベントを送信する機能です。双方向で、Claude Code から返信することもできます。

**どんな場面で使うか**

- Slack や Discord でバグ報告を受け取ったら、自動的に調査を開始させたいとき
- デプロイの通知をセッションに流し込んで、問題があればすぐに対応させたいとき

**使い方の概要**

`/plugin install` でチャンネルプラグインをインストールし、`--channels` フラグ付きで Claude Code を起動します。セキュリティのため、送信者のペアリング（許可リスト登録）が必要です。

> 📝 この機能は Research Preview です。Team/Enterprise プランではデフォルトで無効になっており、管理者が `channelsEnabled` を有効にする必要があります。

📖 公式ドキュメント: [Channels](https://code.claude.com/docs/en/channels)

### Slack 連携

Slack チャンネルで `@Claude` とメンションするだけで、Claude Code のセッションが自動作成される機能です。コードの変更は claude.ai/code 上で実行され、完了後に PR を作成できます。

**どんな場面で使うか**

- Slack のスレッドで議論している課題を、そのまま Claude Code に作業させたいとき
- 非エンジニアのチームメンバーが、Slack から直接コードの修正を依頼したいとき
- チームの共有チャンネルで、タスクの進捗を透明に共有しながら作業したいとき

**使い方の概要**

Claude for Slack アプリをインストールし、Claude Code on the Web へのアクセスと GitHub アカウントの接続を設定します。チャンネルに `@Claude` を招待し、メンションすると自動的にコーディングタスクを検出してセッションを作成します。完了したら Slack スレッドに結果の要約と「View Session」「Create PR」ボタンが表示されます。

ルーティングモードは2種類あります。「Code only」はすべてのメンションを Claude Code セッションとして処理し、「Code + Chat」はコーディングタスクと一般的な質問を自動で振り分けます。

> ⚠️ Slack 連携はチャンネル（パブリック・プライベート）でのみ動作します。ダイレクトメッセージ（DM）では利用できません。

📖 公式ドキュメント: [Slack](https://code.claude.com/docs/en/slack)

### Agent SDK（旧 Headless Mode）（プログラマティック実行）

Claude Code をプログラムから実行する仕組みです。CLI の `claude -p` コマンド、Python パッケージ、TypeScript パッケージの3つの方法で利用できます。

**どんな場面で使うか**

- CI/CD パイプラインに Claude Code を組み込みたいとき（PR の自動レビュー、テスト生成など）
- pre-commit フックで自動的にコード品質チェックを実行したいとき
- 大量のファイルに対して同じ変更を一括適用したいとき
- 独自のツールやワークフローに Claude Code を統合したいとき

**使い方の概要**

```bash
# 非対話モードで実行（結果をテキストで出力）
claude -p "このプロジェクトのテストカバレッジを確認して"

# JSON 形式で構造化された出力を取得
claude -p "このファイルの問題点をリストアップして" --output-format json

# 最小構成で実行（CLAUDE.md や Skills を読み込まない）
claude -p --bare "Hello"
```

`--bare` フラグを使うと、CLAUDE.md、Skills、Hooks、Plugins、MCP、Auto Memory の読み込みをすべてスキップして起動します。スクリプトから呼び出す場合に、余計な設定の影響を受けずに実行できます。

> 💡 Part 2 で学んだ GitHub Actions での PR レビュー（2-3-8）は、この Agent SDK（旧 Headless Mode）を活用した例です。`claude -p` で非対話的に実行し、結果を PR のコメントとして投稿していました。

📖 公式ドキュメント: [Headless](https://code.claude.com/docs/en/headless)

## 機能の選び方

10 の機能を紹介しましたが、すべてを使う必要はありません。自分の開発スタイルと現在の課題に合わせて、優先度をつけましょう。

| あなたの状況 | おすすめの機能 |
|---|---|
| ターミナルだけの作業に慣れてきた | Desktop アプリ、Voice Dictation |
| Web アプリの開発が多い | Chrome 連携、Claude Code on the Web |
| 外出先でも作業したい | Remote Control、Claude Code on the Web |
| チーム開発の効率を上げたい | Slack 連携、Agent Teams |
| CI/CD を改善したい | Agent SDK（旧 Headless Mode） |

4-1-1 で学んだサイクルを思い出してください。

1. **情報収集**: この一覧や公式ドキュメントで「何ができるか」を把握する
2. **検証**: 使い捨てのプロジェクトや Worktree で試す
3. **適用判断**: 安定性、チームへの影響、コストとメリットで判断する

新しい機能を試すこと自体が「学び続ける力」の実践です。

## ✨ まとめ

- Claude Code にはこの教材で扱った機能以外にも、開発体験を大きく変える発展機能がある
- **ローカル拡張**: Computer Use、Voice Dictation、Chrome 連携、Desktop アプリ（Dispatch を含む）
- **リモート利用**: Claude Code on the Web、Remote Control
- **チーム・自動化**: Agent Teams、Channels、Slack 連携、Agent SDK（旧 Headless Mode）
- すべてを使う必要はない。自分の開発スタイルと課題に合わせて選び、学び続けるサイクルで検証・導入する
- `Experimental` や `Research Preview` の機能は仕様変更の可能性がある。公式ドキュメントで最新情報を確認してから導入する
