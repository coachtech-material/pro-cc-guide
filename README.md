# COACHTECH Pro生向け Claude Code 教材

> ジュニアエンジニア（Pro生）が Claude Code を活用し、実務開発の生産性と品質を高めるための教材。

## 教材の概要

AI で一人の生産性が上がり企業の採用が慎重になる中、Pro生がミドル・シニア相当のバリューを出せるよう Claude Code を使いこなせるようになることを目指します。

### 対象読者

COACHTECH の選抜試験を経てフリーランスエージェントに所属しているジュニアエンジニア（Pro生）。

**前提知識:**

- バックエンド: Laravel（MVC、Eloquent ORM、認証/認可、RESTful API、PHPUnit）
- データベース: RDB 設計、SQL（JOIN、サブクエリ、集計）
- 開発環境・ツール: Docker Compose、Git/GitHub（ブランチ運用、PR、コードレビュー）
- AI 活用経験は問わない

### 身につける3つの能力

1. **使いこなす力** — Claude Code の機能・設定・制約を把握し、目的に応じて的確に活用する力
2. **見極める力** — 生成コードを正しさ・品質・安全性の3観点で検証し、責任を持って採用する力
3. **学び続ける力** — ツール・技術の進化を追い、自ら試し、実務に適応し続ける力

## レビューのお願い

教材本体は **`curriculums/`** ディレクトリにあります。

```
curriculums/
├── part-01_はじめに/
├── part-02_Claude Codeの基礎/
├── part-03_Claude Codeの実践/
└── part-04_継続的な学習/
```

各 Section は Markdown ファイル1つに対応しています。

主に以下のような観点でフィードバックいただけると助かります！

- **実務との整合性**: 実際の現場の進め方やPro生が直面する課題とズレていないか
- **構成と過不足**: Part・Chapter の順序や粒度は適切か。抜けている観点や不要に感じる部分はないか
- **学習体験**: Pro生の前提知識に対して説明が過剰/不足していないか。Part 間の流れに飛躍や重複がないか

GitHub 上で Issue やプルリクエストのコメントでフィードバックをお願いします。
