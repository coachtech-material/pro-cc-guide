## Chapter 1-1 この教材について（3セクション）

**ゴール**: Claude Code を学ぶ意義と教材の全体像を理解し、自分に適した学習計画を立てられるようにする。

- **1-1-1 なぜ Claude Code を使うのか**
  - ゴール: AI コーディングツールとしての Claude Code の特徴と、ジュニアエンジニアが活用する意義を理解する
  - 記述パターン: 概念
  - Subsection:
    - AI 時代のジュニアエンジニアの現在地 — なぜ今 Claude Code を学ぶ必要があるかを理解する
      - 市場の変化（AI がジュニアの作業を代替、企業の採用ハードル上昇）
      - 求められる力の変化（AI を使いこなし、ミドル・シニアの領域をカバーする）
      - AI 生成コードの特性（ロジックエラー率、セキュリティ問題の統計）と、だからこそ出力を検証する力が必要であること
    - この教材のコンセプト — この教材が目指すことを理解する
      - AI活用とプログラミング知識の関係（相反しない。知識がAIを使いこなす土台になる）
      - 生産性の向上と品質の担保（Claude Code と協働して実務タスクを高速かつ高品質に遂行する）
      - 実務タスクの3分類（バグ修正・機能開発・リファクタリング）
    - この教材で養う3つの能力 — 使いこなす力・見極める力・学び続ける力の概要を把握する
      - 使いこなす力（Input）: Claude Code に的確に指示を出す力
      - 見極める力（Output）: 出力を検証・判断し、責任を持って採用する力。3観点（正しさ・品質・安全性）
      - 学び続ける力（Time）: ツール・技術の進化に適応し続ける力
    - Claude Code の特徴 — AI コーディングツールとしての Claude Code を理解する
      - コーディングエージェントの進化（GitHub Copilot → Cursor → Claude Code の流れ。補完型からエージェント型へ）
      - 補完型とエージェント型の違い（機能一覧表は作らず、アプローチの違いに絞る。エージェント型の具体像: コードベースの読み取り・ファイル編集・コマンド実行・ツール統合）
      - Claude Code の特徴と強み。利用環境の多様性（Terminal / VS Code / JetBrains 等）にも概要レベルで触れる
  - 公式ドキュメント: [Claude Code Overview](https://code.claude.com/docs/en/overview)

- **1-1-2 対象読者と前提条件**
  - ゴール: 自分のスキルレベルと照らし合わせて、この教材が適切かを判断できる
  - 記述パターン: 概念
  - Subsection:
    - この教材の対象読者 — 想定する読者像を確認する
    - 前提となる技術スキル — 必要な技術レベルを確認する
    - 前提となる開発環境 — 必要な環境・ツールを確認する
      - OS（macOS / Windows（WSL経由））。WSL 環境での Docker Desktop 連携の注意点
      - エディタ（VS Code）
      - ブラウザ（Claude Code の認証フローで使用）
      - Docker Desktop がインストール済みであること（Laravel Sail の前提）
      - Node.js はホストマシンへのインストール不要（Claude Code はネイティブインストール推奨、npm インストールは非推奨。フロントエンドビルドは Sail コンテナ内で実行）
  - 公式ドキュメント: [Setup](https://code.claude.com/docs/en/setup)

- **1-1-3 教材の構成と学習の進め方**
  - ゴール: 教材の全体構成と学習アプローチを把握し、自分に適した進め方を選択できる
  - 記述パターン: 概念
  - Subsection:
    - 全体構成と各 Part の概要 — 4つの Part の役割と流れを理解する
      - Part 一覧表（Part名、対応する能力、概要、ハンズオン方針）
      - Part 2〜3 の学習アプローチ — Part 2 で新規プロジェクト（テック記事プラットフォーム）を構築しながら主要機能を習得し、Part 3 で既存プロジェクト（ProConnect）をcloneしてPart 2で学んだ機能を使い実務タスク（バグ修正・機能開発・リファクタリング）を実践する。Part 2 で「見極める力」3つの観点（正しさ・品質・安全性）の型を導入し、Part 3 で実践的に鍛える
      - Part 4 の学習アプローチ — ツール・技術の継続的キャッチアップの習慣を身につける
    - Claude Code 機能の学習マップ — どの機能をどの Chapter で学ぶかを把握する
      - 機能対応マップ一覧表（主要機能(Part 2) / Part 3 で追加する機能 の2区分）
      - 見極める力は機能マップとは別テーブルで、Part 2 での扱い（型の導入）と Part 3 での扱い（実践）を対比する
      - Part 3/4 は設計中のため、確定している情報のみ記載し「詳細は Part 3/4 の冒頭で改めて説明します」と補足する
    - 学習の進め方 — 推奨順序と選択学習の方針を理解する
      - 推奨順序（Part 1 → 2 → 3 → 4）と依存関係
      - 経験に応じた選択学習
  - 公式ドキュメント: [Features Overview](https://code.claude.com/docs/en/features-overview)

## Chapter 1-2 学習を始める前に（2セクション）

**ゴール**: Claude Code を使い始めるために必要な確認事項とプラン選択を理解する。

- **1-2-1 AI ツールを使う前の確認事項**
  - ゴール: 所属組織の AI ツール利用ポリシーを確認し、責任を持って使い始められる
  - 記述パターン: 概念
  - Subsection:
    - 企業・チームの AI ツール利用ポリシー — 所属組織のルールを確認する重要性を理解する
      - 確認すべきポリシー項目
      - Managed settings による組織的なポリシー強制の仕組み
    - コードと機密情報の取り扱い — 機密情報の漏洩リスクと対策を理解する
      - Consumer（Free/Pro/Max）と Commercial（Team/Enterprise/API）のデータ訓練ポリシーの違い
      - データ保持期間（Consumer 訓練許可あり: 5年 / なし: 30日 / Commercial: 30日）
      - テレメトリ・エラーレポートのオプトアウト方法
      - MCP サーバー経由のデータ漏洩リスク（Anthropic は MCP サーバーの管理・監査を行わない）。MCP の詳細は Part 2 Chapter 2-8 で学ぶ
    - AI 生成コードに対する責任 — 生成コードの責任が自分にあることを認識する
      - 見極める力の3観点（正しさ・品質・安全性）の導入。1-1-1 で紹介した3つの能力のうち「見極める力」をここで具体化する
      - Part 2 以降で各観点を実践的に学ぶことへの接続
  - 公式ドキュメント: [Security](https://code.claude.com/docs/en/security), [Data Usage](https://code.claude.com/docs/en/data-usage)

- **1-2-2 プランの選択と費用**
  - ゴール: 自分に適したプランを選択できる
  - 記述パターン: 概念
  - Subsection:
    - プランの種類と機能差 — 各プランの内容と Claude Code で使える機能の違いを理解する
      - Claude のプラン体系（Free プランでは Claude Code 利用不可）
      - 対応プラン: Pro / Max / Teams / Enterprise / Console（API）。サードパーティプロバイダー（Amazon Bedrock / Google Vertex AI / Microsoft Foundry）経由の利用にも触れる
      - 各プランの料金と Claude Code での機能差
    - プランの選び方とコスト管理 — 自分に適したプランを選び、契約して使用量を管理する方法を知る
      - 最低条件（Pro）と推奨プラン（Max）
      - プランの契約手順（claude.ai でのサブスクリプション登録・支払い方法）
      - コストの目安と使用量の管理
  - 公式ドキュメント: [Pricing](https://claude.com/pricing), [Setup](https://code.claude.com/docs/en/setup)
