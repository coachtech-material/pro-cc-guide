## この教材は誰のためのものか

前のセクションで、AI コーディングツールを使いこなすことがジュニアエンジニアの武器になるという話をしました。では、具体的にどんな人がこの教材の対象なのでしょうか。

この教材は、**COACHTECH の Pro生** を主な対象として書かれています。Pro生とは、COACHTECH の選抜試験を経てフリーランスエージェントに所属し、企業に紹介されて副業・フリーランス・転職などの形で Laravel を使った実務開発に携わるジュニアエンジニアのことです。

もちろん、Pro生以外の方でも、以下のような方であれば十分に活用できます。

- Web アプリケーション開発の基礎を学び終え、実務に入ろうとしている方
- Laravel でのバックエンド開発経験がある（学習レベルでも可）方
- AI コーディングツールを実務に取り入れたいが、どう使えばいいかわからない方

逆に、以下のような方には合わない可能性があります。

- プログラミング自体がまったくの初めてという方（まずは言語の基礎学習が先です）
- すでに AI コーディングツールを日常的に使いこなしており、上級テクニックを求めている方

---

## 🧠 先輩エンジニアの思考プロセス

実務に出たばかりの頃、技術書を買っては「自分にはまだ早いかも」と途中で閉じることがよくありました。逆に、基礎的すぎて読む意味がなかったこともあります。教材選びで大事なのは、「自分の現在地」を正直に把握することです。

この教材は、「基礎力はあるが実務はこれから」という人に合わせて設計しています。Laravel の MVC パターンは知っているけど、実際のプロジェクトで 10万行のコードベースを読んだ経験はない。Git でブランチを切ってプルリクエストを出したことはあるけど、コードレビューで厳しい指摘を受けた経験は少ない。そんなあなたが、Claude Code という相棒を得て、実務で戦えるようになる。それがこの教材のゴールです。

## 前提知識チェックリスト

以下は、この教材を進める上で前提とする知識・スキルです。すべてに「できる」と言える必要はありませんが、大半が「聞いたことがある」「少しやったことがある」レベルであることを想定しています。

### Laravel / PHP

- [ ] MVC パターンの役割（Model、View、Controller がそれぞれ何をするか）を説明できる
- [ ] Eloquent ORM でリレーション（hasMany、belongsTo 等）を定義したことがある
- [ ] 認証（Fortify 等）や認可（Policy）の仕組みを使ったことがある
- [ ] FormRequest を使ったバリデーションを実装したことがある
- [ ] RESTful なルーティング設計を理解している
- [ ] PHPUnit でテストを書いたことがある（Feature テスト、Unit テスト）

### フロントエンド

- [ ] HTML / CSS の基本的な構造を理解している
- [ ] Tailwind CSS のユーティリティクラスを使ったことがある

### データベース

- [ ] テーブル設計（正規化、ER 図）の基本を理解している
- [ ] SQL で JOIN やサブクエリを書いたことがある
- [ ] マイグレーションファイルを作成・実行したことがある

### 開発プロセス

- [ ] 要件定義 → 設計 → 実装 → テストという開発の流れを経験したことがある

### 開発環境・ツール

- [ ] ターミナル（コマンドライン）の基本操作ができる（`cd`、`ls`、`mkdir` 等）
- [ ] Docker Compose で開発環境を構築したことがある
- [ ] Git でブランチを作成し、コミット、プッシュ、プルリクエストの作成ができる
- [ ] GitHub 上でコードレビュー（レビューを受ける側）を経験したことがある

### AI 活用

- [ ] ChatGPT などのチャット AI を使ったことがある

> 💡 AI コーディングツールの経験は問いません。「ChatGPT に質問したことがある」程度で十分です。Claude Code の使い方は Part 2 でゼロから丁寧に解説します。

## 必要な環境

Claude Code を動かすには、以下の環境が必要です。Part 2 のセットアップセクションで詳しく手順を説明しますので、ここでは「自分の環境が対応しているか」の確認だけで大丈夫です。

### OS（オペレーティングシステム）

| OS | 対応バージョン |
|---|---|
| macOS | 13.0（Ventura）以降 |
| Windows | 10（バージョン 1809）以降、または Windows Server 2019 以降 |
| Linux | Ubuntu 20.04+、Debian 10+、Alpine Linux 3.19+ |

> 💡 Windows の場合、Git for Windows が必要です。また、WSL（Windows Subsystem for Linux）環境でも利用できます。WSL 1、WSL 2 のどちらにも対応しています。

### ハードウェア

- **メモリ（RAM）**: 4 GB 以上
- **ネットワーク**: インターネット接続が必要（Claude Code はクラウド上の AI モデルと通信します）

### その他

- **シェル**: Bash、Zsh、PowerShell、CMD のいずれか
- **Anthropic のサービス対応国**: 日本は対応しています

> ⚠️ **よくある確認漏れ**: 企業の開発用 PC を使っている場合、ネットワークの制限（プロキシ、ファイアウォール）で Claude Code が通信できないことがあります。社内ネットワーク環境で使う予定がある場合は、事前に IT 部門に確認しておくと安心です。

## 「足りない」と感じたら

チェックリストを確認して、「いくつか自信がない項目がある」と感じても、すぐに諦める必要はありません。

### 前提知識が不安な場合

前提知識のチェックリストで半分以上が「聞いたことがない」という場合は、まず COACHTECH のカリキュラムや Laravel の公式ドキュメントで基礎を固めることをおすすめします。一方、「やったことはあるけど自信がない」という項目が数個ある程度なら、問題ありません。Claude Code を使いながら復習することもできます。

### 環境が整っていない場合

OS やスペックの要件を満たしていない場合は、Part 2 に進む前に環境を整えてください。特に Windows ユーザーの方は、Git for Windows のインストールが必要です。セットアップ手順は Part 2 の Chapter 1 で詳しく解説します。

### Claude Code の有料プランが必要

Claude Code を利用するには、有料の Anthropic アカウントが必要です。無料プランでは Claude Code は利用できません。プランの選択については、次の Chapter（1-2-2）で詳しく説明します。
