# COACHTECH Pro生向け Claude Code 教材

> ジュニアエンジニア（Pro生）が Claude Code を活用し、実務開発の生産性と品質を高めるための教材。

## 対象読者（ペルソナ）

プログラミングスクール COACHTECH を優秀な成績で卒業し、選抜試験を経てフリーランスエージェントに所属しているジュニアエンジニア（「Pro生」）。企業に紹介され、副業・フリーランス・転職などの形でLaravelを使った実務開発に携わる。

COACHTECH で要件定義からテストまでの開発フローを一通り経験しており、基礎力はある。しかし実務は未経験、またはごく初期の段階にある。「自分のスキルが現場で通用するか」という不安と、「学んだことを実務で活かしたい」という意欲の両方を持って教材に向き合っている。

### 技術スキル

Pro生は以下の技術レベルを有している:

- **ターミナル操作**: 基本コマンド（cd, ls, mkdir, mv, rm等）を使ったファイル操作ができる
- **Git/GitHub**: ブランチ運用、PR作成、コンフリクト解決、Issue駆動開発、コードレビューができる
- **Docker**: Docker Composeを使った開発環境の構築・管理ができる
- **HTML/CSS**: レスポンシブデザインを含むWebページを作成できる。Tailwind CSSの基礎がある
- **PHP**: 基本文法、関数、OOPの基礎（クラス、コンストラクタ）を理解している
- **Laravel**: MVC、Eloquent ORM、認証（Fortify）、認可（Policy）、バリデーション、ミドルウェア、RESTful API開発、PHPUnitによるテストを実装できる
- **データベース**: RDB設計（正規化、ER図）、SQL（JOIN、サブクエリ、集計）を扱える
- **開発プロセス**: 要件定義→DB/API設計→実装→テストの一連の開発フローを経験している
- **AI活用**: ChatGPT を少し触った程度。AI コーディングツール（GitHub Copilot・Cursor 等）の使用経験は問わない

### 前提環境

- **OS**: macOS および Windows（WSL経由）の両方を対象とする
- **エディタ**: VS Code を前提とする
- **ブラウザ**: 認証フローに必要

## 背景・目的（コンセプト）

AIの台頭により、企業はジュニアエンジニアの採用に慎重になりつつある。AIを使いこなし、従来ミドル層が担っていた領域の一部をカバーできるかどうかが、ジュニアエンジニアの市場競争力を左右する。

この教材は、Claude Codeを活用してPro生の**生産性の向上**と**品質の担保**を実現する。**コードリーディング**を前提に、既存プロジェクトのコーディング規約と設計方針に従いながら、以下の**実務タスク**をClaude Codeと協働して**高速かつ高品質に**遂行できるようになることを目指す:

- **バグ修正**: 既存コードの不具合を特定し、修正する
- **機能開発**: 新規機能の実装や既存機能の改修を行う
- **リファクタリング**: コードの品質・可読性・パフォーマンスを改善する

## 目標（ゴール）

実務タスクをClaude Codeと協働して**高速かつ高品質に**進めるために、以下の能力を養う。これらの能力は個人での開発に限らず、チーム開発の文脈（PR連携、`CLAUDE.md` の共有、企業ポリシーへの対応等）でも求められる。

### 1. 使いこなす力（Input）

Claude Codeの機能を理解し、既存コードの文脈を踏まえて的確な指示を設計することで、開発の生産性を高める力。

1. **ツールの活用**: Claude Codeの機能・設定・制約を把握し、目的に応じた機能選択とプロンプト設計で的確に活用する

### 2. 見極める力（Output）

Claude Codeの出力を検証・判断し、その判断を自分の言葉で説明できる力。**説明責任**を含め、品質と安全性に責任を持つ。AIの出力をそのまま受け入れる思考の放棄に陥らず、自分で理解・判断する習慣を保つことが重要である。

1. **コードレビュー**: 生成されたコードを自ら読み、理解した上で適用する
2. **テスト**: テスト内容が十分かを判断し、実行して動作を検証する
3. **セキュリティチェック**: 脆弱性が含まれていないことを確認する。特に環境変数や認証情報などの機密情報がコードやコミットに混入しないよう注意する

### 3. 学び続ける力（Time）

ツール・技術の進化を追い、自ら試し、実務に適応し続ける力。

1. **情報収集**: 公式ドキュメント・変更履歴などの一次情報に加え、X をはじめとする SNS やコミュニティで最新動向をキャッチアップし、破壊的変更や非推奨化にも速やかに追従する
2. **新機能の検証**: 新しい機能やツールを自ら手を動かして試し、動作・制約・既存ワークフローとの互換性を確認する
3. **実務への適用判断**: 検証した機能やツールを自分の開発フローに取り入れるべきかを評価・判断する

## 語りかけ方

この教材を読むPro生は、COACHTECH で力を証明し、選抜を勝ち抜いた人たちである。基礎力はある。でも実務はこれからで、「自分は現場でやっていけるだろうか」という不安と、「学んだことを実務で活かしたい」という意欲が同居している。AIが開発現場を変えつつある今、Claude Codeを武器にできれば従来のジュニアの枠を超えた活躍ができる — そのことを、この教材の一行一行で実感してもらいたい。

私たちはPro生にとっての**メンター**である。技術ドキュメントを渡すのではなく、隣で一緒に画面を見ながら「次はこうしてみよう」と語りかける存在として書く。機能の説明で終わらせず、「なぜこれを知ることが皆さんの仕事に役立つのか」を伝える。コマンド例やハンズオンの題材はPro生のドメイン（PHP / Laravel / Web開発）に寄せ、「自分の仕事で使える」と感じてもらう。

## 教材構成（カリキュラム）

| Part | 対応する能力 | 概要 | ハンズオン |
|---|---|---|---|
| Part 1: はじめに | - | 教材の導入、Claude Code の概要 | なし |
| Part 2: Claude Code の基礎 | 使いこなす力（主）+ 見極める力（型の導入） | 新規プロジェクト（テック記事プラットフォーム「mini Zenn」）を構築しながら主要機能を習得 | テック記事プラットフォームを段階的に構築 |
| Part 3: Claude Code の実践 | 見極める力（主）+ 使いこなす力（総合演習） | 既存プロジェクト（ProConnect）で Part 2 の機能を総合的に使いながら実務タスクを遂行 | 提供プロジェクトで実践 |
| Part 4: 継続的な学習 | 学び続ける力 | 情報収集・新機能検証・実務適用の習慣化 | なし |

教材は Part > Chapter > Section の3層構造で構成する。各 Part は学習フェーズに対応し、複数の Chapter を含む。各 Chapter は1つのゴールで構成する（Chapter 間・Part 間の依存関係はあってよい）。教材本体は `curriculums/` に格納し、各 Part の設計詳細は `outline/` を参照。

### カリキュラムの前提

- **Laravel**: バージョン10（`laravel:^10.0`）
- **Claudeアカウント**: Pro（20ドル）プランを最低条件とし、Maxプランを推奨する。企業に所属している場合は企業のプラン・ポリシーを確認すること
- **Node.js**: 不要。Claude Code はネイティブインストール（curl / Homebrew / WinGet）が推奨され、npm インストールは非推奨（deprecated）

## 表記ルール

- 教材は日本語で執筆する
- 機能やコマンドの説明は公式ドキュメント（https://code.claude.com/docs/en/）を唯一の正確性の拠り所とする。コード例・コマンド例は公式ドキュメントと照合する
- 文体は敬体（です・ます調）で統一する
- コマンド例・ハンズオンの題材は PHP / Laravel / Web 開発のコンテキストを優先する
- 絵文字は ⚠️（注意事項・よくあるエラー）、💡（TIP・補足・実務知見）、👀（確認ポイント・見極める力の実践）の3種のみ使用する
- エラーが起きやすい箇所は以下のフォーマットで先回りして説明する:

> ⚠️ よくあるエラー: [エラーの概要]
>
> ```
> [エラーメッセージ例]
> ```
>
> **原因**: [なぜ発生するか]
>
> **対処法**: [具体的な解決手順]

### 用語

| 表記 | 備考 |
|---|---|
| WSL | Windows Subsystem for Linux。初出時に正式名称を記載 |
| MCP | Model Context Protocol。初出時に正式名称を記載 |
| Pro生 | 教材本文では各 Chapter の初出時に「Pro生」と定義し、以降は「皆さん」を使う |
| セクション | 教材本文では Section をカタカナ「セクション」で表記する。outline やスキル定義など内部ドキュメントでは英語表記のまま |

Claude Code の機能名は公式ドキュメントの表記に従い、英語のまま使用する:
Hooks, Skills, Sub-agents, Agent Teams, Plugins, Fast Mode, Headless Mode, Remote Control, Scheduled Tasks, Checkpointing, Plan Mode, Extended Thinking, Worktree, GitHub Actions

## 運用ルール

Chapter の執筆・レビュー・更新・構成変更は Skills で実行する。

| Skill | 用途 |
|---|---|
| `/write-chapter` | Chapter の新規執筆・更新 |
| `/review-chapter` | Chapter の品質レビュー（横断整合性チェック含む） |
| `/outline` | outline の作成・更新・検証（再構成・MECE 検証・公式ドキュメント照合） |
| `/check-updates` | 公式ドキュメントとの鮮度チェック |

### フォルダ構造・命名規則

```
pro-cc-guide/
├── CLAUDE.md              # 教材の哲学・語りかけ方・表記ルール・運用ルール
├── outline/               # カリキュラム設計（依存関係マップ・Part/Chapter/Section 設計）
│   ├── CLAUDE.md
│   └── part-01.md ~ part-04.md
├── .claude/skills/        # 作業手順（執筆・レビュー・更新・構成変更）
├── curriculums/           # 教材本体
│   └── part-XX_タイトル/chapter-XX_タイトル/X-X-X_タイトル.md
├── assets/                # 画像（assets/part-XX/chapter-XX/）
└── memo/                  # 作業メモ置き場（教材の執筆プロセスでは参照しない）
```

- Part ディレクトリ: `part-XX_タイトル/`（01始まり、ゼロパディング）
- Chapter ディレクトリ: `chapter-XX_タイトル/`（Part 内で01始まり、ゼロパディング）
- Section ファイル: `X-X-X_タイトル.md`（Part番号-Chapter番号-Section番号、ゼロパディングなし）
- 画像ファイル名: 内容がわかる英語名（例: `install-confirmation.png`）
- Part 設計ファイル: `outline/part-XX.md`（ゼロパディング）
