# 1-1-3 教材の構成と学習の進め方

## 概要

ここまでで、なぜ Claude Code を学ぶのか、そしてこの教材が皆さんのために書かれたものであることを確認しました。このセクションでは、教材全体の構成を俯瞰し、どの Part で何を学ぶのか、Claude Code のどの機能をどこで扱うのかを把握します。全体像を掴むことで、自分に合った学習計画を立てられるようになります。

## 全体構成と各 Part の概要

この教材は4つの Part で構成されています。

| Part | 対応する能力 | 概要 | ハンズオン |
|---|---|---|---|
| Part 1: はじめに | — | 教材の導入、Claude Code の概要 | なし |
| Part 2: Claude Code の基礎 | 使いこなす力（主）+ 見極める力（型の導入） | 新規プロジェクトを構築しながら主要機能を習得 | テック記事プラットフォームを段階的に構築 |
| Part 3: Claude Code の実践 | 見極める力（主）+ 使いこなす力（総合演習） | 既存プロジェクトで Part 2 の機能を総合的に使いながら実務タスクを遂行 | 提供プロジェクトで実践 |
| Part 4: 継続的な学習 | 学び続ける力 | 情報収集・新機能検証・実務適用の習慣化 | なし |

### Part 1: はじめに（今読んでいる Part）

Claude Code を学ぶ意義と教材の全体像を理解し、学習を開始する準備を整えます。AI ツールを使う前に確認すべきポリシーやプランの選択についても説明します。

### Part 2: Claude Code の基礎

Claude Code の主要機能を、手を動かしながら習得する Part です。**テック記事プラットフォーム「mini Zenn」** を新規プロジェクトとしてゼロから構築していきます。皆さんが日常的に使う Zenn や Qiita のようなプラットフォームを、Claude Code と協働して作り上げます。

「新規プロジェクト」をハンズオンの題材にしているのは、Claude Code の機能に集中するためです。既存のコードベースを読み解く負担がない分、ツールの使い方そのものに注力できます。インストールから始まり、プロジェクトの初期設定、機能の実装、テスト、Git 連携、MCP・Sub-agents・Plugins による拡張、PR 作成まで、開発フローに沿って Claude Code の機能を一通り体験します。

この Part では「見極める力」——コードレビュー、テスト、セキュリティチェック——の**型を導入**します。各ハンズオンの末尾に「👀 確認ポイント」を設け、Claude Code が生成したコードを自分の目で確認する習慣の土台を作ります。深い実践は Part 3 で行います。

### Part 3: Claude Code の実践

Part 2 で習得した機能を、実務に近い環境で**総合的に使いこなす** Part です。**ProConnect** という既存の Laravel プロジェクトを clone し、実際の開発タスク（バグ修正・機能開発・リファクタリング）に取り組みます。

「既存プロジェクト」を題材にしているのは、実務では既にあるコードベースに手を入れる作業がほとんどだからです。他の人が書いたコードを読み、プロジェクトの設計方針を理解し、それに沿った変更を加える——この一連のプロセスを Claude Code と協働して進めます。

この Part の主役は**「見極める力」**です。Part 2 で型を導入した3つの観点（コードレビュー・テスト・セキュリティ）を、実務タスクの中で実践的に鍛えます。各タスクで重点を変えながら（バグ修正ではセキュリティ、機能開発ではコードレビュー、リファクタリングではテスト）、AI の出力を自ら検証・判断する力を養います。

### Part 4: 継続的な学習

エンジニアとして大切な、ツール・技術の継続的なキャッチアップの習慣を身につける Part です。

AI コーディングツールは日々進化しています。新しい機能が追加されたり、ベストプラクティスが変わったりすることは珍しくありません。公式ドキュメントや変更履歴の確認方法、SNS やコミュニティでの情報収集、新機能の検証と実務への適用判断——これらの習慣を身につけることで、教材を終えた後も自走し続けられるようになります。

## Claude Code 機能の学習マップ

Claude Code にはさまざまな機能がありますが、すべてを一度に覚える必要はありません。以下のマップで、どの機能をどの Part で学ぶかを確認しておきましょう。

### Part 3 で追加する機能

Part 3 では Part 2 で学んだ機能を実務タスクで総合的に使いこなします。新たに導入する機能は以下のみです。

| 機能 | 概要 | Chapter |
|---|---|---|
| GitHub Actions | CI/CD パイプラインでの Claude Code 活用と自動コードレビュー | 3-5 |

### 見極める力（Part 2 で型を導入 → Part 3 で実践）

| 観点 | 概要 | Part 2 での扱い | Part 3 での扱い |
|---|---|---|---|
| コードレビュー | 生成されたコードを自ら読み、理解した上で適用する | 👀 確認ポイントとして毎回確認 | 既存設計との整合性を判断 |
| テスト | テスト内容が十分かを判断し、実行して動作を検証する | テストの十分性を意識する | リファクタリング前後の動作保証 |
| セキュリティチェック | 脆弱性や機密情報の混入がないことを確認する | `/security-review` の結果を確認する | 認可の漏れを自分で発見・修正 |

💡 この機能マップは現時点の教材構成に基づいています。Claude Code は活発に開発が進んでおり、新機能が追加されることがあります。Part 4 で学ぶ「学び続ける力」を活かして、最新の機能もキャッチアップしていきましょう。

## 学習の進め方

### 推奨する学習順序

この教材は **Part 1 → Part 2 → Part 3 → Part 4** の順に進めることを推奨します。

Part 2 の Chapter は原則として順番に進めてください。各 Chapter のハンズオンが前の Chapter の成果物を前提としており、テック記事プラットフォーム「mini Zenn」を段階的に構築していく構成になっています。

Part 3 は Part 2 の完了が前提です。Part 2 で習得した Claude Code の機能を、既存プロジェクトの実務タスクで実践するという位置づけです。

Part 4 は Part 3 までの学習内容を踏まえた上で取り組むのが最も効果的ですが、独立した内容のため、興味があれば先に目を通しても構いません。

### 経験に応じた選択学習

すでに AI コーディングツールの使用経験がある方は、以下を参考にしてください。

- **Claude Code 以外の AI コーディングツール（GitHub Copilot、Cursor 等）を使ったことがある場合**: Part 1 の確認事項を読んだ後、Part 2 から始めてください。他のツールとの違いを意識しながら進めると、Claude Code の特性がより深く理解できます
- **Claude Code を使ったことがある場合**: Part 1 を流し読みし、Part 2 で知らない機能がないか確認してください。基本機能を把握できていれば、Part 3 から始めても構いません

迷ったら、Part 1 から順に進めるのが確実です。「知っていること」の復習は短時間で終わりますが、「知らないこと」を飛ばしてしまうと後で戻る必要が出てきます。

## まとめ

- 教材は4つの Part で構成されている。Part 2 で新規プロジェクト（mini Zenn）を通じて全主要機能を習得し、Part 3 で既存プロジェクト（ProConnect）を通じて実践力と見極める力を鍛える
- Part 2 で Claude Code の機能を一通り学び、Part 3 ではそれらを実務タスクで総合的に使いこなす。「見極める力」は Part 2 で型を導入し、Part 3 で実践する
- 推奨順序は Part 1 → 2 → 3 → 4。Part 2 の Chapter は順番に進めること
- 経験に応じた選択学習も可能だが、迷ったら Part 1 から順に進めるのが確実

次の Chapter「学習を始める前に」では、AI ツールを使う前に確認すべき事項とプランの選択について説明します。
