# 1-1-1 なぜ Claude Code を使うのか

## 概要

この教材は、COACHTECH の選抜を経てフリーランスエージェントに所属するジュニアエンジニア——「Pro生」の皆さんが、Claude Code を武器にして実務開発で活躍するためのガイドです。

「COACHTECH で学んだことが現場で通用するだろうか」——そんな不安を感じている人もいるかもしれません。実は今、AI の急速な進化によって開発現場そのものが変わりつつあり、ジュニアエンジニアに求められる力も変化しています。この変化を正しく理解し、AI コーディングツールを使いこなせるかどうかが、皆さんのキャリアを大きく左右します。

この セクションでは、なぜ今 Claude Code を学ぶ必要があるのか、この教材で何を目指すのか、そして Claude Code がどのようなツールなのかを説明します。

## AI 時代のジュニアエンジニアの現在地

### 市場の変化

AI コーディングツールの登場により、開発現場は大きな転換期を迎えています。かつてジュニアエンジニアが担っていた定型的な実装作業——簡単な CRUD の実装、テストコードの作成、ドキュメントの整備など——を AI が高速にこなせるようになりました。その結果、企業がジュニアエンジニアに期待する水準は上がり、「コードが書ける」だけでは差別化が難しくなっています。

しかし、これは悲観すべき状況ではありません。AI を使いこなすことで、従来はミドルレベルのエンジニアが担っていた領域の一部——設計判断を踏まえた実装、既存コードベースの文脈を理解した上での修正、品質を担保するテスト戦略の実行——をジュニアのうちからカバーできるようになるからです。

つまり、AI を味方につけられるジュニアエンジニアは、従来のジュニアの枠を超えた価値を発揮できます。

### 求められる力の変化

では、AI を味方にするとは具体的にどういうことでしょうか。

これまでジュニアエンジニアに求められていたのは、「指示された機能を正しく実装できる」ことでした。しかし今は、それだけでは AI と差別化できません。求められているのは、AI を使いこなし、従来ミドルレベル以上のエンジニアが担っていた領域——既存コードベースの設計意図を読み取る力、プロジェクトの規約に沿った判断、品質を担保する検証——の一部をカバーできることです。

「AI に仕事を奪われる」のではなく「AI と協働して、一人では到達できなかった成果を出す」。この教材は、そのための力を身につけるガイドです。

### コーディングエージェントの進化

AI コーディングツールは、ここ数年で急速に進化してきました。

**補完型ツールの登場（2021年〜）**——GitHub Copilot に代表される AI コード補完ツールが登場し、エディタ上でコードの候補をリアルタイムに提示するスタイルが広まりました。開発者がコードを書く速度は向上しましたが、AI の役割はあくまで「次の数行を予測して提案する」補助的なものでした。

**AI ネイティブエディタの台頭（2024年〜）**——Cursor をはじめとする AI ネイティブなエディタが登場し、コード補完だけでなく、ファイル全体の編集やチャットベースの対話を統合しました。AI との連携がエディタの中核機能として組み込まれ、開発体験が大きく変わりました。

**エージェント型ツールの登場（2025年〜）**——そして Claude Code に代表されるエージェント型のツールが登場します。エージェント型ツールは、単にコードを提案するのではなく、コードベース全体を読み取り、ファイルを編集し、コマンドを実行し、外部ツールと連携しながら、開発タスクを自律的に遂行します。開発者は「何をしたいか」を伝えるだけで、AI がその実現方法を計画し、実行してくれるのです。

## この教材のコンセプト

この教材は、Claude Code を活用して皆さんの**生産性の向上**と**品質の担保**を実現します。

**コードリーディング**を前提に、既存プロジェクトのコーディング規約と設計方針に従いながら、以下の実務タスクを Claude Code と協働して**高速かつ高品質に**遂行できるようになることが、この教材の目指すところです。

- **バグ修正**: 既存コードの不具合を特定し、修正する
- **機能開発**: 新規機能の実装や既存機能の改修を行う
- **リファクタリング**: コードの品質・可読性・パフォーマンスを改善する

「AI に丸投げして終わり」ではありません。コードを読み、プロジェクトの文脈を理解し、Claude Code に的確な指示を出し、出力された結果を自分の目で検証する。この一連のプロセスこそが、皆さんの実務スキルを確実に高めていきます。

## この教材で養う3つの能力

実務タスクを Claude Code と協働して高速かつ高品質に進めるために、教材全体を通じて以下の3つの能力を養います。

**使いこなす力（Input）**——Claude Code の機能を理解し、既存コードの文脈を踏まえて的確な指示を設計する力です。同じツールでも、使い方次第で結果は大きく変わります。

**見極める力（Output）**——Claude Code の出力を検証・判断し、その判断を自分の言葉で説明できる力です。コードレビュー、テスト、セキュリティチェックの3つの観点で品質に責任を持ちます。AI の出力をそのまま受け入れる「思考の放棄」に陥らず、自分で理解・判断する習慣を保つことが重要です。

**学び続ける力（Time）**——ツール・技術の進化を追い、自ら試し、実務に適応し続ける力です。AI コーディングツールは日々進化しています。今日学んだ知識が半年後には古くなっている可能性もあります。だからこそ、学び続ける習慣そのものが最大の武器になります。

## Claude Code の特徴

### 補完型とエージェント型の違い

AI コーディングツールには、大きく分けて2つのアプローチがあります。

**補完型**は、開発者がコードを書いている最中に次の数行を予測して提案するアプローチです。GitHub Copilot が代表例で、エディタ上でリアルタイムにコードの候補が表示されます。あくまで開発者が主体で、AI は「次に書きそうなコード」を先回りして見せてくれるアシスタントです。

**エージェント型**は、開発者が「何をしたいか」を伝えると、AI が自律的にタスクを遂行するアプローチです。コードベース全体を読み取り、必要なファイルを編集し、コマンドを実行し、結果を確認しながら作業を進めます。開発者は指示を出し、結果を検証する役割を担います。

| | 補完型 | エージェント型 |
|---|---|---|
| AI の役割 | 次の数行を予測・提案 | タスクを自律的に計画・実行 |
| 操作の主体 | 開発者がコードを書く | AI が作業し、開発者が検証する |
| 作業範囲 | カーソル周辺の数行 | コードベース全体 |
| 代表例 | GitHub Copilot | Claude Code |

💡 補完型とエージェント型は二者択一ではありません。補完型ツールでも近年はエージェント的な機能が追加されており、境界は曖昧になりつつあります。ここで大切なのは「アプローチの違い」を理解しておくことで、ツールの特性に合った使い方ができるようになることです。

### Claude Code の特徴と強み

Claude Code は、Anthropic が開発したエージェント型の AI コーディングツールです。公式ドキュメントでは次のように説明されています。

> Claude Code is an agentic coding tool that reads your codebase, edits files, runs commands, and integrates with your development tools.
>
> （Claude Code は、コードベースを読み取り、ファイルを編集し、コマンドを実行し、開発ツールと統合するエージェント型コーディングツールです。）

この説明に、Claude Code の本質が凝縮されています。それぞれの能力を見てみましょう。

- **コードベースの読み取り**: プロジェクト全体のファイル構成やコードの内容を理解します。1つのファイルだけでなく、関連するファイル間の依存関係や設計パターンも把握した上で作業を行います
- **ファイルの編集**: 必要なファイルを直接作成・編集します。複数ファイルにまたがる変更も一度に行えます
- **コマンドの実行**: テストの実行、Git 操作、パッケージのインストールなど、ターミナルで行う操作を実行します。コマンドの結果を確認し、エラーがあれば自ら修正を試みます
- **ツールとの統合**: Git、GitHub、データベース、外部 API など、開発で使うさまざまなツールと連携します。MCP（Model Context Protocol）という仕組みを通じて、連携先をさらに拡張することもできます

Claude Code は、さまざまな環境で利用できます。

- **ターミナル（CLI）**: ターミナルから直接起動して使う、Claude Code の基本的な利用形態です。この教材では主にターミナルでの操作を扱います
- **VS Code / JetBrains**: エディタの拡張機能として利用できます。コードの差分表示やファイルの参照など、エディタならではの機能と連携します
- **デスクトップアプリ**: IDE やターミナルとは独立したスタンドアロンアプリです。差分のビジュアル表示や複数セッションの並行実行ができます
- **Web / モバイル**: ブラウザや iOS アプリから利用できます。ローカル環境のセットアップなしで、長時間実行するタスクを開始して後から確認するといった使い方ができます

💡 この教材ではターミナル（CLI）での操作を中心に進めますが、VS Code 拡張機能との連携についても要所で触れていきます。

## まとめ

- AI の進化により、ジュニアエンジニアには AI を使いこなしてミドル・シニアの領域をカバーする力が求められている
- この教材のコンセプトは「生産性の向上」と「品質の担保」。コードリーディングを前提に、既存プロジェクトの規約と設計に従いながら実務タスクを Claude Code と高速かつ高品質に遂行できるようになることを目指す
- そのために「使いこなす力」「見極める力」「学び続ける力」の3つの能力を養う
- Claude Code はエージェント型の AI コーディングツールであり、コードベースの読み取り・ファイル編集・コマンド実行・ツール統合を自律的に行う
- 補完型ツールが「次の数行を提案する」のに対し、エージェント型ツールは「タスクを自律的に遂行する」という根本的なアプローチの違いがある
