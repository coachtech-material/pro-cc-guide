# 1-1-2 対象読者と前提条件

## 概要

この セクションでは、この教材が想定する読者像と、学習を始めるために必要な技術スキル・開発環境を確認します。自分のスキルレベルと照らし合わせながら読み進めてください。「ほとんど当てはまる」と感じたなら、この教材は皆さんのために書かれたものです。

## この教材の対象読者

この教材は、プログラミングスクール COACHTECH を卒業し、フリーランスエージェントを通じて実務開発に携わる（またはこれから携わる）ジュニアエンジニア——「Pro生」の皆さんを対象としています。

皆さんは COACHTECH で要件定義からテストまでの開発フローを一通り経験しており、基礎力があります。一方で、実務経験はまだ少なく、「現場で通用するだろうか」という不安を感じることもあるでしょう。この教材は、そんな皆さんが Claude Code を武器にして、自信を持って実務に臨めるようになることを目指しています。

副業・フリーランス・転職など、実務に入る形態はさまざまですが、いずれの場合も Laravel を使った Web 開発が中心になることを想定しています。

## 前提となる技術スキル

この教材を効果的に活用するために、以下の技術スキルを前提としています。すべてを完璧に習得している必要はありませんが、「やったことがある」「調べながらならできる」というレベルが目安です。

### ターミナル操作

基本的なコマンド（`cd`、`ls`、`mkdir`、`mv`、`rm` 等）を使ったファイル操作ができること。Claude Code はターミナル上で動作するため、ターミナルの基本操作に慣れていることが重要です。

### Git / GitHub

ブランチの作成・切り替え、コミット、プッシュ、プルリクエストの作成、コンフリクトの解決、Issue を起点とした開発フローを理解していること。Claude Code は Git と密接に連携するため、Git の基本操作を理解していることが前提になります。

### Docker

Docker Compose を使った開発環境の構築・管理ができること。この教材のハンズオンでは、Docker を使って Laravel の開発環境を構築します。

### HTML / CSS

レスポンシブデザインを含む Web ページの作成ができること。Tailwind CSS の基礎知識があると、ハンズオンがスムーズに進みます。

### PHP

基本文法、関数、オブジェクト指向プログラミングの基礎（クラス、コンストラクタ等）を理解していること。

### Laravel

MVC アーキテクチャ、Eloquent ORM、認証（Fortify）、認可（Policy）、バリデーション、ミドルウェア、RESTful API 開発、PHPUnit によるテストを実装できること。この教材の中核となる技術スタックです。

### データベース

リレーショナルデータベースの設計（正規化、ER 図）、SQL（JOIN、サブクエリ、集計関数）を扱えること。

### 開発プロセス

要件定義 → データベース / API 設計 → 実装 → テストという一連の開発フローを経験していること。

### AI ツールの利用経験

ChatGPT を少し触った程度で十分です。GitHub Copilot や Cursor などの AI コーディングツールの使用経験は問いません。Claude Code の使い方はこの教材でゼロから学びます。

## 前提となる開発環境

この教材を進めるために、以下の環境が必要です。

| 項目 | 要件 |
|---|---|
| OS | macOS または Windows（WSL（Windows Subsystem for Linux）経由） |
| エディタ | VS Code |
| ブラウザ | Chrome 等のモダンブラウザ（認証フローで使用） |
| Docker | Docker Desktop がインストール済みであること |
| Node.js | **不要** |

💡 Claude Code は以前は npm 経由でインストールしていましたが、現在はネイティブインストール（curl / Homebrew / WinGet）が推奨されており、npm でのインストールは非推奨（deprecated）です。そのため、Node.js のインストールは必要ありません。Claude Code のインストール手順は Part 2 で詳しく説明します。

> ⚠️ よくあるエラー: WSL 環境での Docker 連携
>
> Windows で WSL を使用する場合、Docker Desktop の設定で「Use the WSL 2 based engine」が有効になっていることを確認してください。この設定が無効だと、WSL 内から Docker コマンドが使えません。
>
> ```
> error during connect: This error may indicate that the docker daemon is not running.
> ```
>
> **原因**: Docker Desktop が WSL 2 統合モードで動作していない
>
> **対処法**: Docker Desktop の Settings → General → 「Use the WSL 2 based engine」にチェックを入れ、Settings → Resources → WSL Integration で使用する WSL ディストリビューションを有効にする

## まとめ

- この教材は COACHTECH を卒業し、Laravel を使った実務開発に携わるジュニアエンジニア（Pro生）を対象としている

- AI コーディングツールの事前経験は不要。Claude Code の使い方はこの教材でゼロから学ぶ
- 開発環境として macOS または Windows（WSL）、VS Code、Docker Desktop が必要。Node.js は不要
