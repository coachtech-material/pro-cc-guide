# 1-2-1 AI ツールを使う前の確認事項

## 概要

前の Chapter では、Claude Code を学ぶ意義と教材の全体像を確認しました。「早く使ってみたい」という気持ちがあるかもしれませんが、ツールをインストールする前に確認しておくべきことがあります。

Pro生の皆さんは、フリーランスエージェント経由で企業に紹介され、実務に携わる立場です。企業ごとに AI ツールの利用ポリシーは異なり、「知らなかった」では済まされません。機密情報の漏洩やポリシー違反は、せっかく掴んだ実務の機会を失うリスクに直結します。

このセクションでは、AI ツールを責任を持って使い始めるために確認すべき3つの観点 — **組織のポリシー**、**データの取り扱い**、**生成コードへの責任** — を整理します。

## 企業・チームの AI ツール利用ポリシー

### なぜポリシー確認が必要なのか

企業によって AI ツールの利用方針は大きく異なります。全面的に推奨している企業もあれば、特定のツールやモデルに限定している企業、あるいは AI ツールの利用自体を禁止している企業もあります。実務に入る前に、所属する組織のルールを必ず確認してください。

### 確認すべきポリシー項目

案件に参画する際、以下の項目を確認しましょう。不明な場合は、プロジェクトマネージャーやチームリーダーに直接聞くのが最も確実です。

- **AI ツールの利用可否**: そもそも AI コーディングツールを使ってよいか
- **使用可能なツール・モデルの指定**: Claude Code、GitHub Copilot、Cursor など、利用が許可されているツールの範囲
- **コード入力の可否**: 業務コードを AI に入力してよいか。入力できるコードの範囲に制限はないか
- **ローカル実行・クラウド実行の制限**: ローカルマシンでの実行のみ許可か、クラウド実行（Claude Code on the web）も利用可能か
- **生成コードの取り扱い**: AI が生成したコードをそのままコミットしてよいか。レビューやテストの要件はあるか

💡 ポリシーが明文化されていない企業も少なくありません。その場合でも「確認した上で使っている」という姿勢が、プロフェッショナルとしての信頼につながります。

### Managed settings による組織的なポリシー強制

Claude Code には、組織が利用ポリシーをツールの設定レベルで強制する **Managed settings** という仕組みがあります。

Managed settings は、組織の管理者が設定するもので、個々のユーザーが上書きすることはできません。例えば以下のような制御が可能です。

- 特定のコマンドやツールの実行を禁止する
- MCP サーバーの利用を制限する
- テレメトリ（利用状況の送信）の設定を固定する

Managed settings が適用されている環境では、Claude Code が自動的にその制約に従います。皆さんが設定を変更する必要はありませんが、「なぜこの操作ができないのだろう？」と思ったときに、組織のポリシーが適用されている可能性があることを覚えておいてください。

Managed settings の配信方法には、Anthropic のサーバーからの配信、MDM（モバイルデバイス管理）ツール経由の配信、ファイルベースの配信があります。具体的な設定方法は管理者の領域ですが、「組織がツールの挙動をコントロールできる仕組みがある」ということを知っておくことが大切です。

## コードと機密情報の取り扱い

Claude Code を使うと、皆さんのコードやプロンプトは Anthropic のサーバーに送信されます。これは Claude が応答を生成するために必要な処理ですが、「自分のデータがどう扱われるのか」を理解しておくことは重要です。

### Consumer プランと Commercial プランのデータ訓練ポリシー

Anthropic のプランは大きく **Consumer（Free / Pro / Max）** と **Commercial（Team / Enterprise / API）** に分かれ、データの訓練利用に関するポリシーが異なります。

| 区分 | プラン | AI モデルの訓練への利用 |
|---|---|---|
| Consumer | Free / Pro / Max | ユーザーが選択可能 |
| Commercial | Teams / Enterprise / API / サードパーティプロバイダー / Claude Gov | 原則として訓練に使用しない |

Consumer プランでは、claude.ai の設定画面（Settings > Data Privacy Controls）から訓練利用の許可・不許可をいつでも変更できます。

⚠️ Consumer プランで業務コードを扱う場合は、訓練利用の設定を必ず確認してください。意図せずデータが訓練に使われることを防ぐため、claude.ai にログインし、Settings > Data Privacy Controls から設定を確認・変更できます。

### データ保持期間

Anthropic がデータを保持する期間は、プランと設定によって異なります。

| 条件 | 保持期間 |
|---|---|
| Consumer — 訓練利用を許可 | 5年 |
| Consumer — 訓練利用を不許可 | 30日 |
| Commercial（Teams / Enterprise / API） | 30日（Enterprise では Zero Data Retention オプションあり） |

💡 `/feedback` コマンドで送信したフィードバックは、プランや設定にかかわらず5年間保持されます。フィードバック送信時は、会話履歴全体のコピーが Anthropic に送られる点に注意してください。

### テレメトリとエラーレポートのオプトアウト

Claude Code は、利用状況のメトリクス（テレメトリ）とエラーログを外部サービスに送信します。いずれもコードやファイルパスは含まれませんが、オプトアウト（送信停止）することも可能です。

| データの種類 | 送信先 | オプトアウト方法 |
|---|---|---|
| テレメトリ（利用状況） | Statsig | 環境変数 `DISABLE_TELEMETRY=1` |
| エラーログ | Sentry | 環境変数 `DISABLE_ERROR_REPORTING=1` |
| フィードバック | Anthropic | 環境変数 `DISABLE_FEEDBACK_COMMAND=1` |
| セッション品質アンケート | Anthropic | 環境変数 `CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY=1` |

すべての非必須通信を一括で無効にしたい場合は、環境変数 `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` を設定します。

環境変数の設定方法は Part 2 で扱います。ここでは「オプトアウトの手段がある」ということを知っておいてください。

💡 サードパーティプロバイダー（Amazon Bedrock / Google Vertex AI / Microsoft Foundry）経由で Claude Code を利用する場合、テレメトリ、エラーレポート、フィードバックはデフォルトで無効になっています。ただし、セッション品質アンケートはプロバイダーにかかわらずデフォルトで有効です。

### MCP サーバー経由のデータ漏洩リスク

MCP（Model Context Protocol）は、Claude Code を外部サービスと連携させるための仕組みです（詳しくは Part 3 で学びます）。MCP サーバーを利用すると、Claude Code が Google Drive や Jira などの外部データにアクセスできるようになります。

ここで知っておくべき重要な点があります。**Anthropic は MCP サーバーの管理・監査を行いません**。MCP サーバーは誰でも作成・公開できるため、信頼できない MCP サーバーを利用すると、意図しないデータ送信が発生するリスクがあります。

MCP サーバーを利用する際は、以下を心がけてください。

- 自分で作成した MCP サーバー、または信頼できる提供元のものを使う
- チームで利用が許可された MCP サーバーのリストを確認する
- 不明な MCP サーバーは使わない

## AI 生成コードに対する責任

Claude Code が生成したコードは、**皆さん自身のコード**です。コードを理解せずにそのまま採用した結果、バグやセキュリティの問題が発生した場合、その責任は皆さんにあります。

これは Chapter 1-1 で紹介した「見極める力」に直結します。

- **コードレビュー**: 生成されたコードを自ら読み、理解した上で適用する
- **テスト**: テスト内容が十分かを判断し、実行して動作を検証する
- **セキュリティチェック**: 脆弱性が含まれていないことを確認する。特に環境変数や認証情報などの機密情報がコードやコミットに混入しないよう注意する

「AI が書いたから大丈夫だろう」という思考の放棄は、プロフェッショナルとして最も避けるべき姿勢です。Claude Code は強力なパートナーですが、最終的な判断は常に皆さん自身が行います。

## まとめ

- 企業・チームの AI ツール利用ポリシーを必ず確認する。ポリシーが明文化されていない場合も、確認した上で使う姿勢が大切
- Managed settings により、組織はツールの挙動をポリシーレベルで制御できる
- Consumer プラン（Free / Pro / Max）と Commercial プラン（Teams / Enterprise / API 等）ではデータ訓練ポリシーが異なる。Consumer プランでは訓練利用の設定を確認する
- テレメトリやエラーレポートはオプトアウト可能。MCP サーバーは信頼できるもののみ利用する
- AI が生成したコードの責任は自分にある。「見極める力」を常に実践する
