# 2-1-2 インタラクティブモードとワンショットモード

## 概要

前のセクションで Claude Code のインストールと認証が完了しました。ここからは、実際に Claude Code を使ってみましょう。

Claude Code には、大きく分けて2つの使い方があります。対話しながら作業を進める**インタラクティブモード**と、1回の指示で結果を得る**ワンショットモード**です。このセクションでは、この2つのモードの違いを理解し、効率的な入力方法を覚え、ファイル生成と diff 承認を体験します。解説しながら実際に手を動かしていきます。

## 2つのモード

### インタラクティブモード

インタラクティブモードは、Claude Code の最も基本的な使い方です。`claude` コマンドで起動し、対話しながら作業を進めます。

```bash
claude
```

起動すると、プロンプト（入力待ち）が表示されます。ここに指示を入力し、Enter で送信します。Claude が応答を返し、必要に応じてファイルの編集やコマンドの実行を行います。やり取りを重ねながら、1つの作業を段階的に進められるのがインタラクティブモードの特徴です。

<!-- TODO: 画像追加 - インタラクティブモードの画面 -->

初回の指示と一緒に起動することもできます。

```bash
claude "このプロジェクトの構成を教えて"
```

この場合、Claude Code が起動すると同時に指示が送信され、応答後もそのままインタラクティブモードが続きます。

セッションを終了するには、`exit` と入力するか、`Ctrl+D` を押してください。

では、実際にインタラクティブモードを試してみましょう。練習用のディレクトリを作って Claude Code を起動します。

```bash
mkdir -p ~/claude-practice && cd ~/claude-practice
claude
```

起動できたら、何か簡単な質問をしてみてください（例: 「PHPの配列操作のメソッドを3つ教えて」）。応答が返ってきたら、そのまま次に進みます。

### ワンショットモード

ワンショットモード（`-p` フラグ）は、1回の指示で結果を得て終了するモードです。対話は行わず、結果を標準出力に出力します。

```bash
claude -p "PHPの配列操作の主なメソッドを5つ教えて"
```

ワンショットモードは、ちょっとした質問やスクリプトの中での利用に向いています。応答が完了すると、Claude Code は自動的に終了します。

インタラクティブモードを `exit` で終了し、ワンショットモードを試してみましょう。

```bash
claude -p "現在のディレクトリにあるファイルの一覧を教えて"
```

応答が表示された後に自動的に終了するのを確認してください。インタラクティブモードとの違いを体感できたでしょうか。

### Pipe / stdin 連携

ワンショットモードの強力な使い方の1つが、他のコマンドの出力をパイプで渡す連携です。

```bash
cat app/Models/User.php | claude -p "このモデルのリレーションを説明して"
```

この例では、`User.php` の内容を Claude Code に渡して、リレーションの説明を求めています。ファイルの内容やコマンドの出力を直接渡せるので、特定のコードについて素早く質問したいときに便利です。

他にもこんな使い方ができます。

```bash
# Git の差分を渡してレビューを依頼する
git diff | claude -p "この変更をレビューして"

# テストの実行結果を渡してエラーの原因を聞く
php artisan test 2>&1 | claude -p "このテスト結果のエラー原因を教えて"
```

### 2つのモードの使い分け

| | インタラクティブモード | ワンショットモード |
|---|---|---|
| 起動コマンド | `claude` | `claude -p "指示"` |
| 対話 | 複数回やり取りできる | 1回で完結する |
| 終了 | `exit` や `Ctrl+D` で手動終了 | 応答後に自動終了 |
| 主な用途 | 機能開発、バグ修正、設計相談など | 簡単な質問、コードの確認、スクリプト連携 |

この教材では、主にインタラクティブモードを使います。ワンショットモードは、ちょっとした確認や CI/CD での自動化などで活躍します。

## 入力テクニック

Claude Code を効率よく使うために、いくつかの入力テクニックを覚えておきましょう。

### 複数行入力

指示が長くなるとき、改行して複数行で書くと読みやすくなります。改行方法はいくつかあります。

| 方法 | 対応ターミナル |
|---|---|
| `\` + `Enter` | すべてのターミナル |
| `Shift+Enter` | iTerm2, WezTerm, Ghostty, Kitty（その他は `/terminal-setup` で設定） |
| `Option+Enter` | macOS（デフォルト） |

例えば、次のように複数行で指示を書けます。

```
Userモデルに以下のバリデーションを追加してください。\
- name: 必須、最大255文字\
- email: 必須、メール形式、ユニーク
```

### `@` ファイル添付

`@` に続けてファイルパスを入力すると、そのファイルの内容を指示に含めることができます。Tab キーでファイルパスの補完もできます。

```
@app/Models/Article.php このモデルにカテゴリとのリレーションを追加して
```

ファイルの内容をわざわざコピーして貼り付ける必要がなく、Claude Code がファイルを直接読み取ってくれます。

### `!` Bash モード

`!` で始めると、Claude を介さずにシェルコマンドを直接実行できます。実行結果は会話のコンテキストに追加されるので、Claude が結果を踏まえた応答をしてくれます。

```
! php artisan route:list
```

「今のルート一覧を見てから、次の指示を考えたい」といった場面で便利です。

### キーボードショートカット

よく使うショートカットを紹介します。

| ショートカット | 説明 |
|---|---|
| `Ctrl+C` | 現在の入力や生成をキャンセルする |
| `Ctrl+D` | Claude Code を終了する |
| `Ctrl+L` | ターミナル画面をクリアする（会話履歴は残る） |
| `Ctrl+R` | コマンド履歴を逆方向に検索する |
| `Esc+Esc` | 直前の変更を巻き戻す（Rewind。Chapter 2-7 で詳しく学ぶ） |
| `Up` / `Down` | 過去の入力履歴をたどる |
| `?` | 全ショートカット一覧を表示する |

💡 すべてのショートカットを覚える必要はありません。まずは `Ctrl+C`（キャンセル）と `Up`（履歴）だけ覚えておけば十分です。他のショートカットは使いながら少しずつ身につけていきましょう。`?` キーを押せばいつでも一覧を確認できます。

## ファイル生成と diff 承認

### diff の読み方

Claude Code にファイルの作成や編集を指示すると、実際に変更を行う前に **diff（差分）** が表示されます。diff は「何がどう変わるか」を示すもので、皆さんが内容を確認して承認するまで変更は適用されません。

diff の表示形式は Git の diff と同じです。

```diff
+ <?php
+
+ echo "Hello, World!";
```

- `+`（緑）で始まる行: 追加される内容
- `-`（赤）で始まる行: 削除される内容
- 色なしの行: 変更されない既存の内容（文脈の表示）

### 承認と却下

diff が表示されたら、以下の選択肢があります。

| 操作 | 説明 |
|---|---|
| `y`（Yes） | 変更を承認して適用する |
| `n`（No） | 変更を却下する |
| `e`（Edit） | 変更内容を編集してから適用する |

承認すると、ファイルが実際に作成・編集されます。却下すると、変更は適用されず、Claude に別の方法を指示できます。

💡 変更のたびに承認するのが煩わしい場合、`Shift+Tab` で Auto-Accept Mode に切り替えると、すべての変更を自動で承認できます。ただし、まずは1つずつ diff を確認する習慣をつけましょう。権限モードの詳細は Chapter 2-9 で学びます。

これは Claude Code の重要な設計思想です。AI が勝手にコードを変更するのではなく、**皆さんが確認して承認する**というプロセスが組み込まれています。

### 実際にファイルを生成して diff を確認する

では、実際に diff を体験しましょう。先ほどの練習用ディレクトリで Claude Code を起動してください（すでに起動中ならそのまま進めます）。

```bash
cd ~/claude-practice
claude
```

Claude Code に以下の指示を入力してください。

```
Hello Worldを出力するPHPファイルを作ってください
```

Claude Code が `hello.php`（またはそれに類するファイル名）の作成を提案し、diff が表示されるはずです。

1. 表示された diff の内容を読みます。`+` で始まる行が追加されるコードです
2. 内容に問題がなければ `y` を押して承認します
3. ファイルが作成されたことを確認します

<!-- TODO: 画像追加 - diff 承認の画面 -->

Bash モードでファイルの内容と実行結果を確認してみましょう。

```
! cat hello.php
```

```
! php hello.php
```

`Hello, World!`（またはそれに類する出力）が表示されれば成功です。

さらに、先ほど作成したファイルをパイプで渡して、コードの説明を求めてみましょう。`exit` で Claude Code を終了してから実行します。

```bash
cat hello.php | claude -p "このコードを説明して"
```

ファイルの内容が Claude に渡され、コードの解説が返ってきます。

### エージェントループ

Claude Code は、1つの指示に対して複数のステップを自律的に実行します。例えば「Article モデルを作って」と指示すると、Claude Code は以下のような流れで作業を進めます。

1. 既存のコードベースを調べる（コンテキストの収集）
2. 必要なファイルを作成・編集する（アクションの実行）
3. 結果を確認する（結果の検証）

この「調べる → 実行する → 確認する」のサイクルを**エージェントループ**と呼びます。Claude Code が各ステップで何をしているかは、ターミナル上のツール実行ログで確認できます。エージェントループの詳細は Chapter 2-3 で改めて学びます。

今の段階では、「Claude Code は1つの指示に対して複数のツールを使いながら自律的に作業を進め、ファイルの変更時には diff を表示して承認を求める」ということを覚えておいてください。

## まとめ

- **インタラクティブモード**（`claude`）は対話しながら作業を進める基本の使い方。この教材では主にこのモードを使う
- **ワンショットモード**（`claude -p`）は1回の指示で結果を得て終了する。パイプ連携（`cat file | claude -p`）で他のコマンドと組み合わせると強力
- `@` でファイルを添付、`!` でシェルコマンドを直接実行できる
- ファイルの変更は diff として表示され、承認するまで適用されない。**diff を読んで承認する習慣**が、AI との協働で品質を担保する第一歩
- Claude Code は「調べる → 実行する → 確認する」のエージェントループで自律的に作業を進める

これで Claude Code の基本操作を体験できました。次の Chapter では、プロジェクトに `CLAUDE.md` を用意し、Claude Code に記事プラットフォームの土台を作らせていきます。
