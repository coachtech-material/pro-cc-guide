## インストール

### 推奨インストール方法

Claude Code のインストールには、公式が提供するネイティブインストーラーを使います。この方法が推奨される理由は、依存関係が不要で、インストール後にバックグラウンドで自動アップデートされるためです。

**macOS / WSL（Windows Subsystem for Linux）の場合:**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

このコマンドを実行すると、Claude Code のバイナリが `~/.local/bin/` にインストールされます。

**Windows（PowerShell）の場合:**

```powershell
irm https://claude.ai/install.ps1 | iex
```

> ⚠️ よくあるエラー: コマンドが見つからない
>
> ```
> zsh: command not found: claude
> ```
>
> **原因**: インストール直後はシェルが新しいパスを認識していないことがあります。
>
> **対処法**: ターミナルを再起動するか、`source ~/.zshrc`（Zsh の場合）または `source ~/.bashrc`（Bash の場合）を実行してください。

インストールが完了したら、バージョンを確認しましょう。

```bash
claude --version
```

バージョン番号が表示されれば、インストール成功です。

### 代替インストール方法

環境やポリシーに応じて、以下の方法でもインストールできます。

**Homebrew（macOS）:**

```bash
brew install --cask claude-code
```

**WinGet（Windows）:**

```powershell
winget install Anthropic.ClaudeCode
```

Homebrew や WinGet 経由でインストールした場合、自動アップデートは行われません。アップデートするには `brew upgrade claude-code` や `winget upgrade Anthropic.ClaudeCode` を手動で実行する必要があります。

⚠️ npm でのインストール（`npm install -g @anthropic-ai/claude-code`）は**非推奨（deprecated）**です。以前は npm が主なインストール方法でしたが、現在はネイティブインストーラーが推奨されています。npm 版は Node.js 18 以上が必要で、自動アップデートもされません。ネイティブインストーラーを使いましょう。

## 認証

### アカウントの準備

Claude Code を利用するには、Pro 以上のプランを契約した Claude アカウントが必要です（プランの選択については [1-2-2 プランの選択と費用](../../part-01_はじめに/chapter-02_学習を始める前に/1-2-2_プランの選択と費用.md) を参照してください）。

まだアカウントを作成していない場合は、[claude.ai](https://claude.ai) でアカウントを作成し、Pro 以上のプランを契約してください。

### ブラウザ認証

Claude Code を初めて起動すると、ブラウザが自動的に開き、認証画面が表示されます。

```bash
claude
```

### 認証の管理

認証に関連するコマンドを覚えておきましょう。

| コマンド | 説明 |
|---|---|
| `/login` | Claude Code 内で再認証する |
| `/logout` | ログアウトする |
| `claude auth status` | 現在の認証状態やプラン情報を確認する（ターミナルから実行） |

企業に所属して作業する場合、企業が契約している Teams / Enterprise プランやサードパーティプロバイダー（Amazon Bedrock / Google Vertex AI / Microsoft Foundry）のアカウントを使うよう指示されることがあります。その場合は企業の手順に従ってください。

## 環境確認

### `/doctor` による環境診断

Claude Code には、動作環境を診断する `/doctor` コマンドが用意されています。インストールと認証が完了したら、環境に問題がないか確認しましょう。

```bash
claude doctor
```

このコマンドは、Claude Code のインストールと設定の詳細チェックを行い、問題があれば報告してくれます。

### VS Code ターミナルでの利用

この教材では、VS Code のターミナルから Claude Code を使うことを前提としています。VS Code でプロジェクトのディレクトリを開き、統合ターミナル（`` Ctrl+` `` で開閉）から `claude` を起動してください。

Claude Code はカレントディレクトリをプロジェクトのルートとして認識します。VS Code でプロジェクトフォルダを開いた状態でターミナルを使えば、自然にプロジェクトのルートディレクトリで Claude Code が起動します。

### ターミナルの推奨設定

Claude Code を快適に使うために、ターミナルの設定を調整しておくことをお勧めします。

**複数行入力の設定**: Claude Code では、プロンプトを複数行で書くことがよくあります。`Shift+Enter` で改行できると便利ですが、ターミナルによっては設定が必要です。

Claude Code 内で以下のコマンドを実行すると、お使いのターミナルに合わせた設定を自動で行ってくれます。

```
/terminal-setup
```

iTerm2、WezTerm、Ghostty、Kitty では `Shift+Enter` がそのまま使えます。VS Code や Alacritty などでは `/terminal-setup` を実行して設定してください。

💡 どのターミナルでも `\` + `Enter` で改行できます。`Shift+Enter` の設定がうまくいかない場合はこちらを使ってください。

### 利用環境の紹介

💡 この教材ではターミナル（CLI）での操作を中心に進めますが、Claude Code はさまざまな環境で利用できます。ここでは、ターミナル以外の利用環境を簡単に紹介します。

**VS Code 拡張機能**: VS Code のサイドバーから Claude Code を利用できます。コードの差分表示やファイル参照など、エディタならではの機能と連携します。この教材では VS Code のターミナルから CLI を使いますが、拡張機能も併用できます。

**デスクトップアプリ**: IDE やターミナルとは独立したスタンドアロンアプリケーションです。差分のビジュアル表示や複数セッションの並行実行ができます。

**Web / モバイル**: ブラウザや iOS アプリから Claude Code を利用できます。ローカル環境のセットアップなしで、長時間実行するタスクを開始して後から確認するといった使い方ができます。

**JetBrains 連携**: PhpStorm をはじめとする JetBrains 系 IDE のプラグインとして利用できます。

**Remote Control**: 他のアプリケーションから Claude Code セッションをプログラムで制御する仕組みです。自動化やワークフロー統合に活用できます。

これらの環境は必要に応じて試してみてください。この教材の学習にはターミナル（CLI）で十分です。

## まとめ

- Claude Code のインストールにはネイティブインストーラー（`curl` コマンド）を使う。npm は非推奨
- 初回起動時にブラウザ認証が行われる。`/login` `/logout` で認証を管理し、`claude auth status` で状態を確認できる
- `claude doctor` で動作環境を診断できる
- VS Code のターミナルからプロジェクトディレクトリで `claude` を起動するのが基本の使い方
- `/terminal-setup` で複数行入力などのターミナル設定を最適化できる

次のセクションでは、Claude Code の2つの基本的な使い方——インタラクティブモードとワンショットモード——を学び、実際にファイルを生成して diff を確認する体験をします。
