# 1-1-3 教材の構成と学習の進め方

## 概要

前の Section では、この教材の対象読者と前提条件を確認しました。この Section では、教材の全体構成、各 Part で学ぶ内容、Claude Code の機能がどこで登場するかの一覧、そして皆さんに適した学習の進め方を説明します。

教材の全体像を把握し、自分に適した学習計画を立てるために、以下の順に解説します。

- 全体構成と各 Part の概要
- Claude Code 機能の学習マップ
- 学習の進め方（推奨順序と選択学習）

## 全体構成と各 Part の概要

この教材は4つの Part で構成されています。

| Part | 対応する能力 | 概要 | ハンズオン |
|---|---|---|---|
| **Part 1: はじめに** | — | 教材の導入。Claude Code を学ぶ意義、対象読者、教材構成の説明 | なし |
| **Part 2: Claude Code の基礎** | 使いこなす力 + 見極める力 | 新規プロジェクト（タスク管理カンバンアプリ）を構築しながら主要機能を習得 | タスク管理アプリを段階的に構築 |
| **Part 3: Claude Code の実践** | 使いこなす力 + 見極める力 | 既存プロジェクト（ProConnect）で応用機能を学びながら実務タスクを遂行 | 提供プロジェクトで実践 |
| **Part 4: 継続的な学習** | 学び続ける力 | 情報収集・新機能検証・実務適用の習慣化 | なし |

### Part 2 の学習アプローチ

Part 2 では、タスク管理カンバンアプリ（Laravel 10 / Laravel Sail）をゼロから構築しながら、Claude Code の主要機能を習得します。インストールから始めて、`CLAUDE.md` の整備、コード生成、テスト作成、Git 連携、権限設定、コスト管理、PR 作成まで、Chapter を追うごとにアプリが成長していきます。

### Part 3 の学習アプローチ

Part 3 では、既存プロジェクト「ProConnect」（IT エンジニアと開発案件のマッチングプラットフォーム）を clone し、応用機能（Sub-agents、Hooks、Skills 等）を使いながら実務タスクを実践します。バグ修正・機能開発・リファクタリングの3つのタスクを通じて、Claude Code との協働を実務レベルで体験します。

### 見極める力の実践

Part 2 と Part 3 を通じて、「見極める力」の3つの観点を常に実践します。

- **コードレビュー**: 生成されたコードを自ら読み、理解した上で適用する
- **テスト**: テスト内容が十分かを判断し、実行して動作を検証する
- **セキュリティチェック**: 脆弱性が含まれていないことを確認する。特に環境変数や認証情報などの機密情報がコードやコミットに混入しないよう注意する

### Part 4 の学習アプローチ

Part 4 は「学び続ける力」に焦点を当てています。Claude Code に限らず、AI コーディングツール全般の進化は非常に速く、数か月で機能や推奨プラクティスが変わることも珍しくありません。公式ドキュメントや Changelog の読み方、SNS・コミュニティでの情報収集、新機能の検証方法、実務への適用判断の基準を学び、エンジニアとして継続的にキャッチアップする習慣を身につけます。

## 学習の進め方

### 推奨順序と依存関係

この教材は **Part 1 → Part 2 → Part 3 → Part 4** の順に進めることを推奨します。

Part 2 の各 Chapter は基本的に線形に進みます（2-1 → 2-2 → ... → 2-10）。前の Chapter で学んだ機能やハンズオンの成果物を次の Chapter で活用するため、順番通りに進めてください。ただし、Chapter 2-7（権限設定）と Chapter 2-8（モデル選択）は、Chapter 2-2 の完了を前提としつつ、他の Chapter と独立して学習できます。

Part 3 は Part 2 全体の完了が前提です。Part 3 内では、Chapter 3-1（コードリーディング）を最初に完了した後、Chapter 3-2（バグ修正）・3-3（機能開発）・3-4（リファクタリング）に進みます。Chapter 3-5（チーム開発）は 3-2 〜 3-4 の実践経験が前提です。

Part 4 の各 Chapter は独立しており、Part 3 の完了後であれば任意の順序で学習できます。

### 経験に応じた選択学習

皆さんの経験に応じて、以下のように学習をカスタマイズできます。

**Claude Code が初めての方（推奨ルート）**

Part 1 から順番に進めてください。Part 2 でゼロから丁寧に学ぶことで、Part 3 の実践にスムーズに入れます。

**他の AI コーディングツール（GitHub Copilot、Cursor 等）の経験がある方**

Part 1 に目を通した後、Part 2 は各 Chapter の概要を確認し、知らない機能や Claude Code 固有の機能がある Chapter を重点的に学んでください。Part 3 からは全員が同じスタートラインです。

**すでに Claude Code を使っている方**

Part 1 〜 2 は機能の学習マップを参考に、まだ知らない機能を拾い読みしてください。Part 3 の実践と Part 4 の継続学習が最も価値のある部分になるでしょう。

## まとめ

- 教材は Part 1（導入）→ Part 2（基礎・新規アプリ構築）→ Part 3（実践・既存プロジェクト）→ Part 4（継続学習）の4部構成
- Part 2 で基本機能、Part 3 で応用機能を学び、両 Part を通じて「見極める力」を実践する
- 推奨順序は Part 1 → 2 → 3 → 4 の線形だが、経験に応じた選択学習も可能
- 機能の学習マップを参照すれば、どの機能がどの Chapter で登場するかを把握できる

次の Chapter（Chapter 1-2「学習を始める前に」）では、Claude Code を使い始める前に確認すべき事項 — 企業の AI ツール利用ポリシー、データの取り扱い、プランの選択 — を解説します。
