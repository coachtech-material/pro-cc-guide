## インストール

Claude Code をインストールします。推奨されるネイティブインストールと、代替手段を順に説明します。

### インストール手順（macOS / WSL）

Claude Code の推奨インストール方法は、ネイティブインストーラーを使った方法です。ターミナルで以下のコマンドを実行してください。

**macOS / Linux / WSL の場合**

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**Windows（PowerShell）の場合**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**Windows（CMD）の場合**

```cmd
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

⚠️ Windows ネイティブで利用する場合は、事前に **Git for Windows** のインストールが必要です。WSL（Windows Subsystem for Linux）を使う場合は不要です（WSL 2 の利用を推奨します。WSL 2 ではサンドボックス機能が利用できます）。

インストールが完了したら、バージョンを確認します。

```bash
claude --version
```

バージョン番号が表示されればインストール成功です。

ネイティブインストーラーでインストールした場合、Claude Code は**バックグラウンドで自動アップデート**されます。常に最新バージョンが利用できるため、手動での更新作業は基本的に不要です。

### 代替インストール方法（Homebrew / WinGet）

Homebrew や WinGet を使ったインストールも可能です。ただし、これらの方法では自動アップデートが行われないため、手動でのアップグレードが必要になります。

**Homebrew（macOS）**

```bash
brew install --cask claude-code
```

アップグレード: `brew upgrade claude-code`

**WinGet（Windows）**

```powershell
winget install Anthropic.ClaudeCode
```

アップグレード: `winget upgrade Anthropic.ClaudeCode`

⚠️ **npm を使ったインストールは非推奨（deprecated）です。** Claude Code のインストールに Node.js は不要です（1-1-2 で確認した通りです）。ネイティブインストーラーは高速で、依存関係がなく、自動アップデートにも対応しています。npm でインストール済みの場合は、ネイティブインストーラーに移行してください。

```bash
# npm から移行する場合
curl -fsSL https://claude.ai/install.sh | bash
npm uninstall -g @anthropic-ai/claude-code
```

## 認証

インストールが完了したら認証を行います。

### アカウント登録手順

1-2-2 でアカウント作成とプラン契約は完了している前提です。ここでは Claude Code 側の認証フローに進みます。

### 認証フロー

ターミナルで `claude` コマンドを実行すると、初回起動時に認証が求められます。

```bash
claude
```

1. `claude` を実行すると、ブラウザが自動的に開きます
2. ブラウザで Claude のアカウントにログインします
3. 認証が完了すると、ターミナルに戻って Claude Code が利用可能になります

ブラウザが自動で開かない場合は、`c` キーを押すとログイン URL がクリップボードにコピーされます。コピーした URL をブラウザに貼り付けてログインしてください。

認証に関連するコマンドは以下の通りです。

| コマンド | 説明 |
|---|---|
| `/login` | 再認証（アカウントの切り替え等） |
| `/logout` | ログアウト |
| `claude auth status` | 認証状態の確認（どのアカウント・プランで接続しているか）。Claude Code の外から実行する |

認証が完了したら、`claude auth status` で正しいアカウント・プランで接続されているか確認しておきましょう。

⚠️ 認証がうまくいかない場合は、以下を確認してください。

- ブラウザで [claude.ai](https://claude.ai) にログインできるか
- Pro 以上のプランが有効になっているか（Free プランでは Claude Code を利用できません）
- ネットワーク環境に問題がないか（企業のプロキシやファイアウォールがブロックしている場合があります）
- `claude auth status` コマンドで認証状態を確認する（`--text` フラグを付けると人間が読みやすい形式で表示されます）

## 環境確認

インストールと認証が完了したら環境を確認します。

### `/doctor` による環境診断

`claude doctor` コマンドを実行すると、インストール状態や設定に問題がないかを診断できます。

```bash
claude doctor
```

問題が検出された場合は、その内容と対処方法が表示されます。インストール直後に一度実行しておくと安心です。

### VS Code ターミナルでの利用

Claude Code はターミナルで動作するツールです。VS Code のターミナル（`Ctrl+`` で開く）内でも `claude` コマンドを実行して利用できます。この教材では、VS Code のターミナル内での利用を想定しています。

Claude Code を起動した状態で `/terminal-setup` コマンドを実行すると、**Shift+Enter** で改行できるようにターミナルを設定できます。VS Code をお使いの場合は、ここで設定しておきましょう。

```
/terminal-setup
```

💡 iTerm2、WezTerm、Ghostty、Kitty では Shift+Enter がデフォルトで利用可能なため、`/terminal-setup` は表示されません。

Claude Code では、プロンプトの入力中に改行する方法が複数あります。

| 方法 | 説明 |
|---|---|
| `\` + Enter | すべてのターミナルで利用可能 |
| Shift+Enter | `/terminal-setup` 設定後、または対応ターミナルで利用可能 |

### クラウドプロバイダ認証の存在紹介

💡 ここまで説明した認証フローは、Anthropic の公式アカウント（Pro / Max / Team / Enterprise）を使う方法です。これ以外に、クラウドプロバイダ経由で Claude Code を利用する方法もあります。

- **Amazon Bedrock**
- **Google Vertex AI**
- **Microsoft Foundry**

クラウドプロバイダ経由の場合は、ブラウザでの認証は不要で、環境変数を設定する方式になります。企業が AWS や Google Cloud を利用している場合に使われることがあります。詳細は本教材の範囲外ですが、このような選択肢があることを知っておいてください。

## まとめ

- Claude Code の推奨インストール方法はネイティブインストーラー（`curl`）で、自動アップデートに対応している
- Homebrew / WinGet でもインストールできるが、自動アップデートは行われない
- npm でのインストールは非推奨（deprecated）。Node.js は不要
- `claude` コマンドの初回実行でブラウザ認証が行われる。ブラウザが開かない場合は `c` キーで URL をコピーできる
- `/login` で再認証、`/logout` でログアウト、`claude auth status` で認証状態を確認できる
- `claude doctor` で環境診断、`/terminal-setup` で Shift+Enter の設定ができる
- クラウドプロバイダ（Amazon Bedrock / Google Vertex AI / Microsoft Foundry）経由でも利用可能
