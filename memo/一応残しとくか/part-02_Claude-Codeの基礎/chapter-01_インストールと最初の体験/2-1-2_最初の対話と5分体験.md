## 2つのモードと入力テクニック

Claude Code には2つの主要なモードがあります。まずモードの違いと入力テクニックを理解しましょう。

### インタラクティブモード / ワンショットモード

**インタラクティブモード**

`claude` コマンドで起動し、対話的にやり取りするモードです。質問を投げかけ、回答を受け取り、さらに続けて指示を出す — という対話を繰り返せます。

```bash
claude
```

初期プロンプト付きで起動することもできます。この場合、指定したプロンプトで対話が始まり、そのまま対話を続けられます。

```bash
claude "このプロジェクトの構成を説明して"
```

**ワンショットモード**

`-p`（print）フラグを付けると、1回きりの実行で結果を出力して終了します。

```bash
claude -p "Git の最新コミットメッセージを教えて"
```

パイプと組み合わせることで、ファイルの内容やコマンドの出力を Claude Code に渡すこともできます。

```bash
cat README.md | claude -p "この内容を要約して"
```

**使い分けの指針**

| モード | 用途 |
|---|---|
| インタラクティブモード（`claude`） | 対話しながら作業を進めたいとき、複数のステップがあるタスク |
| ワンショットモード（`claude -p`） | 1回の質問で完結する確認、スクリプトやパイプとの組み合わせ |

💡 上記はコマンドの紹介です。この後のハンズオンで実際に実行します。

### 入力テクニック

Claude Code にはプロンプト入力を便利にするテクニックがあります。

**複数行入力**

長い指示を書くときは、複数行に分けて入力できます。

| 方法 | 説明 |
|---|---|
| `\` + Enter | すべてのターミナルで利用可能（2-1-1 で確認済み） |
| Shift+Enter | `/terminal-setup` 設定後、または対応ターミナルで利用可能（2-1-1 で設定済み） |

**`@` ファイル添付**

プロンプト内で `@` に続けてファイルパスを入力すると、オートコンプリートが表示されます。ファイルの内容をコンテキストとして渡したいときに便利です。

```
@src/app/page.tsx のこの部分を修正して
```

**`!` Bash モード**

プロンプトの先頭に `!` を付けると、シェルコマンドを直接実行できます。実行結果は会話のコンテキストに追加されます。

```
! git status
! npm test
```

**キーボードショートカット**

| ショートカット | 説明 |
|---|---|
| Ctrl+C | 現在の入力や生成をキャンセル |
| Ctrl+L | ターミナル画面をクリア（会話履歴は維持） |
| ↑ / ↓ | コマンド履歴の移動 |
| Ctrl+D | Claude Code を終了（EOF シグナル） |
| Shift+Tab | 権限モードの切り替え（後の Chapter で詳しく学びます） |

`/help` コマンドを実行すると、利用可能なコマンドの一覧を確認できます。Claude Code を終了するには、`exit` と入力するか Ctrl+D を押します。

💡 上記はコマンドの紹介です。この後のハンズオンで実際に実行します。

## ファイル生成と diff 承認

Claude Code にファイル生成を指示すると、変更内容を diff 形式で表示して承認を求めます。ここではその仕組みを理解しましょう。

### ファイル生成と diff 承認・却下

Claude Code に「ファイルを作成して」と指示すると、以下の流れで処理が進みます。

1. Claude Code が適切なファイルを特定（または新規作成を判断）する
2. 変更内容が **diff 形式**で表示される
3. 皆さんが内容を確認し、**承認（Accept）または却下（Reject）**を選択する
4. 承認した場合のみ、変更がファイルに反映される

diff 表示では、追加される行が `+` で、削除される行が `-` で示されます。この表示を確認することで、Claude Code が何を変更しようとしているかを把握できます。

承認時の操作は以下の通りです。

| 操作 | 説明 |
|---|---|
| Accept（承認） | 表示された変更を適用する |
| Reject（却下） | 変更を適用せず、やり直しを指示できる |

セッション中に毎回確認するのが煩わしい場合は、**Shift+Tab** で権限モードを切り替えることで、ファイル編集を自動承認するモードに変更できます。ただし、この教材の学習中は一つ一つ確認する習慣をつけることを推奨します。

### エージェントループの簡潔な紹介

Claude Code は指示を受けると、自律的に以下のサイクルを回します。

1. **コンテキスト収集**: ファイルを読み取り、コードベースを理解する
2. **アクション実行**: ファイルを編集したり、コマンドを実行する
3. **結果確認**: テストの実行結果やエラーを確認する

このサイクルを**エージェントループ**と呼びます。Claude Code が作業している間、画面にはツールの実行状況（どのファイルを読んだか、何を編集したか等）がリアルタイムで表示されます。今は「Claude Code はこのように動いている」と眺める程度で十分です。エージェントループの詳細は Chapter 2-3 で詳しく扱います。

> **検証: diff の確認**
>
> ファイル生成の diff が表示されたとき、以下を確認してみましょう。
>
> **確認ポイント**:
> - 変更内容が指示した内容と一致しているか
> - 意図しないファイルが変更されていないか

## ハンズオン: 簡単なファイルを生成し diff を確認して承認する

ここからは実際に手を動かします。以下のステップに沿って進めてください。

### Step 1: 作業用ディレクトリを作成し Claude Code を起動する

まず、作業用のディレクトリを作成し、その中で Claude Code を起動します。

```bash
mkdir ~/cc-first-try
cd ~/cc-first-try
claude
```

Claude Code が起動し、プロンプトが表示されたら準備完了です。

### Step 2: 最初の質問をする

まずは簡単な質問をしてみましょう。以下のように入力して Enter を押します。

```
Claude Code で何ができるか、3つ教えて
```

Claude Code が回答を返します。このようにインタラクティブモードでは、自然言語で質問や指示を入力するだけで対話できます。

### Step 3: ファイル生成を指示し diff を確認する

次に、ファイルの生成を指示します。

```
hello.txt に「はじめまして、Claude Code を使い始めました」という内容で自己紹介文を書いて
```

Claude Code がファイルの作成を提案し、diff が表示されます。

**確認してみましょう**:
- diff に表示されている内容が、指示した内容と一致しているか
- ファイル名が `hello.txt` になっているか

問題なければ承認してください。生成されたファイルを確認するには、`!` Bash モードを使います。

```
! cat hello.txt
```

### Step 4: ワンショットモードを試す

一度 `exit` で Claude Code を終了し、ワンショットモードを試します。

```
exit
```

ワンショットモードで、先ほど作成したファイルの内容を説明させてみましょう。

```bash
claude -p "hello.txt の内容を説明して"
```

回答が表示された後、自動的にプロンプトに戻ります。インタラクティブモードとは異なり、1回の応答で終了するのがワンショットモードの特徴です。

### Step 5: 終了

ワンショットモードは自動で終了するため、特に操作は不要です。

💡 この作業用ディレクトリ（`cc-first-try`）は Chapter 2-2 では使いません。Chapter 2-2 では新しく Laravel プロジェクトを作成します。不要になったら削除して構いません。

## まとめ

- Claude Code には**インタラクティブモード**（`claude` で起動、対話的にやり取り）と**ワンショットモード**（`claude -p` で1回きりの実行）がある
- `claude "query"` はインタラクティブモードを初期プロンプト付きで開始する（`-p` とは異なり、対話を続けられる）
- 入力テクニック: `@` でファイル添付、`!` で Bash コマンド実行、`\` + Enter や Shift+Enter で複数行入力
- ファイル生成時は diff 形式で変更内容が表示され、承認（Accept）/ 却下（Reject）を選択する
- Claude Code は「コンテキスト収集 → アクション実行 → 結果確認」のエージェントループで動作する（詳細は Chapter 2-3）
