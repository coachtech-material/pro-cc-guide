# 1-2-1 AI ツールを使う前の確認事項

## 概要

前の Chapter では、Claude Code を学ぶ意義と教材の全体像を確認しました。いよいよ Claude Code を使い始めたいところですが、その前に実務で AI ツールを利用する際に確認すべき事項を押さえておきましょう。

この Section では、以下の順に解説します。

- 所属する企業・チームの AI ツール利用ポリシーの確認
- コードや機密情報がどのように扱われるかの理解
- AI が生成したコードに対する責任の考え方

## 企業・チームの AI ツール利用ポリシー

皆さん（Pro生）は企業に紹介され、副業・フリーランス・転職などの形で実務開発に携わります。AI ツールを業務で使い始める前に、**所属する企業・チームの AI ツール利用ポリシーを必ず確認してください**。ポリシーを確認せずに AI ツールを使用すると、契約違反やセキュリティインシデントにつながる可能性があります。

### 確認すべきポリシー項目

企業やチームによってポリシーの範囲や厳しさは異なります。以下の項目を確認しましょう。

- **AI ツールの利用許可**: そもそも業務での AI ツール利用が許可されているか
- **利用可能なツール・モデルの制限**: Claude Code に限らず、使用してよいツールやモデルが指定されているか
- **コード送信の範囲**: 社内コードを AI サービスに送信してよいか。送信可能な場合、対象リポジトリやファイルに制限はあるか
- **機密情報の取り扱い**: API キー、環境変数、顧客データなどの機密情報を AI ツールに入力してよいか
- **生成コードの利用条件**: AI が生成したコードの利用に関するルール（ライセンス確認、レビュー要件など）があるか
- **利用するプラン**: 個人の Pro / Max プランを使うのか、企業の Team / Enterprise プランを使うのか

⚠️ ポリシーが明文化されていない場合でも、「確認していないから使っていい」とはなりません。上長やチームリーダーに確認してから利用を開始してください。

### Managed settings による組織的なポリシー強制の仕組み

企業によっては、Claude Code の設定をポリシーとして一元管理し、個人が変更できないようにしている場合があります。この仕組みを **managed settings** と呼びます。

Claude Code の設定には優先度があり、managed settings は最も優先度が高く、ユーザーが上書きすることはできません。

| 優先度 | スコープ | 設定場所 | 影響範囲 |
|---|---|---|---|
| 1（最高） | Managed | サーバー管理、MDM/OS レベル、`managed-settings.json` | マシン上の全ユーザー |
| 2 | コマンドライン引数 | 起動時のフラグ | そのセッションのみ |
| 3 | Local | `.claude/settings.local.json` | 自分のみ（リポジトリ内） |
| 4 | Project | `.claude/settings.json` | 全コラボレーター |
| 5（最低） | User | `~/.claude/settings.json` | 自分の全プロジェクト |

managed settings では、以下のような制御が可能です。

- 権限バイパスモードの禁止
- 許可する MCP サーバーの制限
- ネットワークアクセス先の制限
- ファイルアクセス範囲の制限
- Hooks の制限

💡 managed settings が適用されている環境では、皆さんが個人設定で変更しようとしても反映されません。「設定が反映されない」と感じたら、まず managed settings が適用されていないか確認しましょう。設定の詳細は Part 2（Chapter 2-9）で学びます。

## コードと機密情報の取り扱い

Claude Code を使うと、皆さんのコードやプロンプトが Anthropic のサーバーに送信されます。このデータがどのように扱われるかは、利用しているプランによって異なります。

### Consumer と Commercial のデータ訓練ポリシーの違い

| 区分 | 対象プラン | AI モデルの訓練にデータが使われるか |
|---|---|---|
| Consumer | Free / Pro / Max | ユーザーが許可した場合のみ使われる（設定で選択可能） |
| Commercial | Team / Enterprise / API | **使われない**（顧客が明示的にオプトインした場合を除く） |

Consumer プランでは、[claude.ai/settings/data-privacy-controls](https://claude.ai/settings/data-privacy-controls) でデータ訓練への利用を許可するかどうかを設定できます。この設定はいつでも変更可能です。

Commercial プランでは、Anthropic の商用利用規約により、顧客のコードやプロンプトを AI モデルの訓練に使用することは明確に禁止されています。

⚠️ 企業の Team / Enterprise プランを利用している場合は、企業のデータが訓練に使われないことが保証されます。個人の Pro / Max プランで業務コードを扱う場合は、訓練許可の設定に注意してください。

### データ保持期間

送信したデータがサーバーにどの程度の期間保持されるかも、プランと設定によって異なります。

| 区分 | 条件 | 保持期間 |
|---|---|---|
| Consumer | 訓練許可あり | 5年間（モデル開発・安全性向上のため） |
| Consumer | 訓練許可なし | 30日間 |
| Commercial | 標準 | 30日間 |
| Commercial | Zero Data Retention（ZDR）適用 | 保持なし（Enterprise で利用可能） |

💡 `/bug` コマンドでフィードバックを送信した場合、会話の全履歴が Anthropic に送信され、5年間保持されます。機密情報を含むセッションでは注意が必要です。

### テレメトリ・エラーレポートのオプトアウト方法

Claude Code はデフォルトで、操作のメトリクス（レイテンシ、信頼性、使用パターン等）やエラー情報を外部サービスに送信します。コードやファイルパスは含まれませんが、オプトアウトしたい場合は以下の環境変数を設定します。

| 環境変数 | 効果 |
|---|---|
| `DISABLE_TELEMETRY=1` | メトリクス送信（Statsig）を無効化 |
| `DISABLE_ERROR_REPORTING=1` | エラーレポート（Sentry）を無効化 |
| `DISABLE_BUG_COMMAND=1` | `/bug` コマンドによるレポート送信を無効化 |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1` | 上記すべて＋セッション品質調査を一括無効化 |

💡 Amazon Bedrock / Google Vertex AI / Microsoft Foundry 経由で利用する場合、テレメトリとエラーレポートはデフォルトで無効です。

### MCP サーバー経由のデータ漏洩リスク

Claude Code は MCP（Model Context Protocol）を通じて外部ツール（データベース、API、プロジェクト管理ツールなど）と連携できます。MCP サーバーについて、以下の点を理解しておくことが重要です。

- **Anthropic は MCP サーバーの管理・監査を行いません**。MCP サーバーを通じてやり取りされるデータの安全性は、そのサーバーの提供元に依存します
- MCP サーバーを利用する場合は、**信頼できるプロバイダーのサーバーを使用する**か、**自分で作成したサーバーを使用する**ことが推奨されます
- 初めて利用する MCP サーバーには信頼性の確認（trust verification）が求められます

⚠️ MCP サーバーは強力な拡張機能ですが、信頼できないサーバーを接続すると、プロジェクトのコードや機密情報が意図せず外部に送信されるリスクがあります。MCP の詳細は Part 3 で学びます。

## AI 生成コードに対する責任

Claude Code が生成したコードであっても、皆さんがレビューして承認し、プロジェクトに適用した時点で、そのコードの責任は皆さんにあります。これは Claude Code に限らず、あらゆる AI ツールに共通する原則です。

「AI が書いたコードだから自分の責任ではない」という主張は通用しません。実務では以下のことが求められます。

- 生成されたコードの内容を**自分で理解する**
- コードが既存のコーディング規約や設計方針に**適合しているか確認する**
- セキュリティ上の問題（脆弱性、機密情報の混入等）が**ないか検証する**
- テストを実行して**動作を確認する**

これは Chapter 1-1 で紹介した「見極める力」そのものです。AI の出力をそのまま受け入れる「思考の放棄」に陥らず、自分で判断する習慣を持つことが、プロのエンジニアとしての責任です。

## まとめ

- 業務で AI ツールを使う前に、所属する企業・チームの利用ポリシーを必ず確認する
- Managed settings により、企業が Claude Code の設定を一元管理し、個人による変更を制限できる
- Consumer プラン（Free / Pro / Max）ではデータ訓練への利用を設定で制御できる。Commercial プラン（Team / Enterprise / API）ではデータが訓練に使われない
- データ保持期間はプランと設定によって 30日〜5年まで異なる
- テレメトリやエラーレポートは環境変数でオプトアウトできる
- MCP サーバーは Anthropic が管理・監査しないため、信頼できるサーバーのみを使用する
- AI が生成したコードであっても、承認・適用した時点で責任は自分にある
