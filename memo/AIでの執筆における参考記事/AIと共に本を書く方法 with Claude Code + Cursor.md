# **執筆における全体の流れ**

まずは全体の流れに触れておく。と言っても、通常、本を書くときの流れはおおよそ変わらないだろう。書く部分がかなりAIに委譲されている部分が異なる。

- ペルソナを作る
- 本のコンセプトを作る
- 目次を作る
- Claude Codeを使って全体を描く
- 叩きをCursorで書き換えていく
- 手で最終調整する

# **ペルソナ・コンセプト・目次が超大事**

目次を書くところまでは通常と同じだろう。実はこの部分をしっかりやっておくことでかなりの部分をAIに任せることができる。というのも、**「本の読者と存在意義」を決めるとAIがいい感じに内容を想像してくれるようになる**からだ。

### **ペルソナ**

ペルソナでは「誰に届けるのか」を定義する。

- どんな人に届けたいのか
- その人がこの本を読むモチベーション
- 1名だけではなく2-3名描いておくと読者の幅が取れる

### **コンセプト**

コンセプトでは「なぜその本が存在するのか」を定義する。

- この本を書く著者のモチベーション
- ペルソナがこの本を読むとどんな学びを得るのか
- ペルソナがこの本を読んだあとにどのような行動変容が起きるのか
- 行動変容を生むために「何を語り、何を語らない本」なのか

### **目次**

目次では「この本に書かれるべき内容」を定義する。

- 本の全体構成(目次)
- 各部/各章で書かれる内容の詳細

**目次だけでなく、各章で書かれる内容の詳細まで踏み込んで準備が必要なのはそれがAIにとってガードレールになるからだ。**伝えたい内容を通常の目次レベルよりも1段階深く掘って作っておくことでAIが文章生成する際のヒントになってくれる。

ちなみにこれらもすべて生成AIで作成可能であるが、この部分が要件定義になるため、かなりこだわって作ることをおすすめする。

# **Claude Codeを使って全体を描く**

それらが決まったら早速、執筆を始めよう。

### [**CLAUDE.md](https://claude.md/) は「本を貫通して守ってほしいこと」を書いておく**

[**Claudeのメモリを管理する - Anthropic***異なるメモリ場所とベストプラクティスを使用して、セッション間でClaude Codeのメモリを管理する方法を学びます。docs.anthropic.com*](https://docs.anthropic.com/ja/docs/claude-code/memory)

CLAUDE.mdは簡単に言えば「Claude Codeとの約束事を書いておくファイル」と呼べる。ここに書いてあることは基本的に読み取って守ってくれる。

CLAUDE.mdには「本を貫通して守ってほしいこと」を書いておくと良い。また、1度書いたらあまり変更しない部分を集めておくと管理が楽である。おすすめはCLAUDE.mdに以下を書いておくことだ。

- ペルソナ
- コンセプト
- 執筆の際に気をつけてほしいこと・執筆ルール
- プロジェクト内のフォルダ構造とそれぞれの役割
- 本に書くこと・書かないこと

### **目次は別ファイルに分けておくと便利**

次は目次を実際に置いておこう。ここでは table-content.md という名前で作ろう。ファイルを分ける理由としては、目次は執筆の最中に微調整を行いたくなるケースが多いからだ。また Cursorなどのエディタ内チャットでファイルメンションできるのも楽である。

- 目次
- 目次に伴う執筆内容
- 各章に対応するファイル名

### **既存の資料をかき集めて有効活用しよう**

**生成AIはゼロからの生成よりも、有りものを組み合わせ変換する方が良いアウトプットが出てくる事が多い。**このため、本の主題に近いアウトプットをすでにしているならばそれらを素材として使ったほうがはるかに楽に良いアウトプットを得ることができる。

また、**素材を渡すだけではなく「コンセプトやルール」を渡すほうがより意図に近い物が出てくる。**これはすでに前章で紹介したからそれを使えば良い。

たとえば、[会社員のゲームルール](https://zenn.dev/shuzon/books/119ce690fab13d)を執筆した際は自身の日記やnoteをすべてknowledgeフォルダに置いていた。「knowledgeフォルダに有る記事を読み、当章の内容を考えてほしい」と指示することでほぼ変換を行うだけで執筆が済んだ。

すでに考えてアウトプットしたことはAI執筆における知識資産だから有効に使いたい。

### **実際に執筆させる**

ここまで来るとほとんど自動で文章を書いてくれる。よほどWeb上にないドメイン(業界)知識や専門知識を含まない限りはかなり筋の良い内容を提案してくれる。

また文脈が足りていない・知識が足りていない場合はdeep researchした内容を共に渡してあげると良い。世に出回っている情報を使って補足するならば十分役に立つ。

# **叩きをCursorで書き換えていく**

Claude Codeが出してきた記事をそのまま出すわけには行かない。細かいニュアンスや伝えたいこととのズレ、冗長さを編纂していく。

### **行指向で編集できるCursorは便利**

この時に助けになるのは CursorなどのEditor型 AIである。理由は以下2点。

- Claude Codeで細かい編集はあまり向いていない
- Cursorは行指向で範囲選択しながら対話的にプロンプティング可能

### **Inline Promptを利用しよう**

CursorなどEditor型AIでは行選択した箇所にのみ適用できるプロンプト( inline prompt) を利用できる。思ったよりも冗長になっている箇所や、表現や伝えたいことが異なる段落を選択し「こんな感じで行きたい」とピンポイントで修正できるため便利である。

範囲選択部分のみに有効なプロンプトが使える

# **手で最終調整する**

と、ここまではAIを利用した執筆について書いてきた。しかし、**どこまで行っても最後は手で直していくしかない。こればっかりは楽できない。**

AI執筆した書籍は自分自身で書いたものではないため、細部の表現やニュアンスが何となくしっくり来ないことも多い。そこは手で変えていくほかない。また、自分で書いたわけではないから目が滑りやすくなる。ここも1行1行に責任感を持ってレビューすることが大切だ。

レビュー管理はGitとGitHubが個人的にはおすすめであるが、使い方がわからない人は無理して使わなくていいだろう。

# **人間の仕事は「品質保証」と「行き先へのガイド」**

この記事におけるAI執筆の大前提を最後に伝える。

**AI執筆において人間の仕事は2つある。それは「品質保証」と「行き先へのガイド」である。**

品質保証はAIが生成してきた文章に対して発信の品質責任を負うことである。誤ったことや誤解を招く表現はレビューによって修正していく。成果物のAuthorizeが仕事になる。

行き先へのガイドは、つまり執筆者として本がたどり着くべきゴールへ行くようにAIをしっかりとナビゲートする役目である。AI執筆は執筆における運転席をAIに渡す瞬間が数多く登場する。しかし、著者は自分であり、AIの手綱を握るべき役目も自分である。AIのハンドルを握り、行きたいところへ連れて行くのは人間の仕事だ。

これを見て、1人でもほんの執筆をしてみたい人が増えると冥利に尽きる。
