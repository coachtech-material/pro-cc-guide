# **執筆方針とスタイルガイド**

### **文体とトーン**

**基本方針**

- **トーン**: 親しみやすく、初心者に優しい。教科書的な堅苦しさを避け、会話的で読みやすい文体
- **文末**: です・ます調で統一
- **読者への呼びかけ**: 「あなた」を使用（例：「あなたの執筆プロジェクト」）
- **視点**: 読者と一緒に学ぶ姿勢（例：「一緒に見ていきましょう」）

**文章の構成**

- **1文の長さ**: 60文字以内を目安。長い場合は2文に分割
- **段落の長さ**: 3-5文を目安。長すぎる段落は視覚的に圧迫感を与えるため分割
- **箇条書きの活用**: 3つ以上の項目は箇条書きで表現
- **強調**: 重要な概念は**太字**、専門用語は初出時に定義

### **専門用語の扱い**

**初出時の定義ルール**

専門用語は必ず初出時に定義し、括弧で英語表記を併記する。

**例**:

`Claude CodeのSubagent（サブエージェント、専門化されたAIアシスタント）を使うと...`**copy**

**略語の使用**

- 初出時はフルスペル + 略語を併記
- 2回目以降は略語のみ使用可

**例**:

`Model Context Protocol（MCP）は... [初出]
MCPを使うと... [2回目以降]`**copy**

### **コードブロックとコマンド例**

**コードブロックの書式**

- 必ず言語タグ付きfenced code blocksを使用
- コマンド実行例はbash、設定ファイルはjsonやyamlなど

**例**:

````bash
claude code --thinking
```

```json
{
  "model": "claude-opus-4-5"
}
````**copy**

**コマンド出力の表示**

- 実行コマンドと出力を明確に区別
- コメントで説明を追加

**例**:

`# ファイル一覧を表示
ls -la

# 出力:
# total 16
# drwxr-xr-x  4 user  staff  128 Jan 24 10:00 .
# ...`**copy**

### **見出し構造**

**見出しレベルの使い分け**

- **# (h1)**: 章タイトルのみ
- **## (h2)**: 主要セクション
- **### (h3)**: サブセクション
- **#### (h4)**: 詳細項目（多用しない）

**見出しの命名規則**

- 明確で具体的なタイトル
- 「〜について」「〜とは」などの冗長表現を避ける
- 動詞を使って行動を示す（例：「Skillを作成する」）

### **例示と実践**

**具体例の必須化**

- 抽象的な説明だけでなく、必ず具体例を含める
- 実際に動くコード例を提供
- 「悪い例」「良い例」の対比も効果的

**実践演習の構成**

各章の最後に以下の構成で実践演習を配置：

1. **学習目標** (3-5個の箇条書き)
2. **実践演習** (具体的な課題、1-3個)
