# outline の作成・更新・検証

**入力**: $ARGUMENTS

## モード判定

| モード | トリガー | 内容 |
|---|---|---|
| **再構成** | 「追加」「削除」「並べ替え」「統合」「再構成」 | Part/Chapter の構成変更 + 影響ファイル更新 |
| **検証** | 「MECE」「整理」「検証」 | MECE 構造の検証・改善 |
| **照合** | 「公式ドキュメント」「最新化」「照合」 | 公式ドキュメントとの差分検出 |

スコープ: `Part番号` / `Part番号-Chapter番号` / `Part番号-Chapter番号-Section番号`

---

## 再構成モード

1. **変更計画**: 変更内容・影響範囲・番号変更マッピングを提示し承認を得る
2. **outline 更新**: `outline/part-XX.md` と `outline/CLAUDE.md` の依存関係マップを更新
3. **影響ファイル更新**: root `CLAUDE.md`、`curriculums/` 配下で構成を参照している Section
4. **整合性検証**: 番号の抜け・重複、Grep で旧番号の残留を検索

---

## 検証モード

設計原則に基づき MECE 構造を検証する。

**設計原則**:

| # | 原則 |
|---|---|
| 1 | **階層的 MECE 分解**: 各レベルは上位を MECE に分解する |
| 2 | **抽象度の一貫性**: 兄弟要素は同じ抽象度 |
| 3 | **各レベル 2〜4 個**: 5個以上ならグルーピング不足 |
| 4 | **分解軸の単一性**: 兄弟が同じ軸上にあること |

問題を検出したら再構成案を提示し、承認を得て適用する。

---

## 照合モード

1. **URL 抽出**: outline から公式ドキュメント URL を抽出し WebFetch する
2. **差分検出**: 🔴 誤り・廃止（修正必須）/ 🟡 追加が望ましい に分類
3. **レポート提示**: 差分一覧をユーザーに提示し、承認を得て適用
